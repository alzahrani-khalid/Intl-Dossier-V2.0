## 5. E2E Flow Evidence

**Flow A — Hub-driven creation (UX-01):**
User → `/dossiers/create` → `CreateDossierHub` (8 cards) → click card →
`/dossiers/{type}/create` → `CreateWizardShell` composes steps → submit → dossier row.
Evidence: `31-VERIFICATION.md` goals #1, #2, #3, #4 (all PASS); `CreateDossierHub.tsx`
route-maps via `TYPED_LIST_TO_WIZARD` constant.

**Flow B — Typed-list FAB (UX-04):**
On `/dossiers/countries` (or any typed list), FAB / Create button → direct to typed
wizard, bypassing hub. Evidence: `useContextAwareFAB.ts:36-43` defines the route map;
`DossierListPage.tsx:511` ternary routes direct-when-filtered vs. hub; confirmed for
countries, engagements, persons, elected-officials
(`31-VERIFICATION.md` goal #4).

**Flow C — Guidance banner (UX-02):**
`StepGuidanceBanner.tsx:37-38` uses key pattern
`dossier-wizard:guidance:{type}:{stepId}`; dismiss state persists in localStorage
(Phase 31). 55 `guidance` string occurrences across 19 i18n files (EN+AR). All 8
wizard configs carry a `guidanceKey` per step.

**Flow D — Person identity (PBI-02, PBI-06):**
New person wizard renders honorific + split names + nationality + DOB + gender.
List pages render `formatPersonLabel` output + nationality flag badge
(`32-VERIFICATION.md` Observable Truth #6: `formatPersonLabel` used 6× in
persons list, 3× in `ElectedOfficialListTable`). Vitest `person-display.test.ts`:
17/17 pass.
