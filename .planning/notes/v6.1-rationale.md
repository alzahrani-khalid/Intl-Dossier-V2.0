# v6.1 Hardening & Reconciliation — Rationale

## Decision

Run a small surgical milestone (v6.1) to close v6.0 tech debt before scoping v7.0.

## Why before v7.0

v6.0 audit (`.planning/milestones/v6.0-MILESTONE-AUDIT.md:122, 240`) shipped with `audit_status: tech_debt`. No critical blockers, all 52 reqs satisfied at code level, but real CI/documentation/data-layer gaps remain:

- 6 phases without explicit per-phase VERIFICATION.md (33, 34, 36, 37, 39, 40)
- `size-limit` budget gate broken (2.42 MB measured vs 815 KB ceiling — chunk-glob drift)
- 14 visual-regression baselines deferred to operator action across Phases 38/40/41
- Schema/seed gaps: `intelligence_digest` table missing, VIP person ISO join missing in RPC, `060-dashboard-demo.sql` not applied
- 5 Phase 43 anti-patterns (WR-02..WR-06) — `aria-label` shadowing visible text, dead `??` branches, invalid `hsl(var(--*))` wrapping a hex
- 1 deferred plan (33-08 storybook)
- Stale `[x]` checkboxes in REQUIREMENTS.md and ROADMAP progress table

These items are cheap to close as a batch and unblock confidence for any v7.0 work that builds on the design system. Running a feature milestone (Intelligence Engine candidate) on top of a broken bundle gate, missing schema, and unverified visual baselines compounds the debt.

## Scope (3 phases, ~8-12 plans)

| Phase | Scope                                                                                                                                                             | Plans est. |
| ----- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------- |
| 44    | Docs & toolchain: VERIFICATION.md backfill (6 phases), REQUIREMENTS+ROADMAP sync, size-limit chunk-glob repair, WR-02..06 fixes, 33-08 storybook                  | 4-5        |
| 45    | Schema & seed: `intelligence_digest` table + hook, VIP person ISO join, apply `060-dashboard-demo.sql`, close 4 BLOCKED-BY-SEED specs                             | 3          |
| 46    | Visual baselines: regenerate Phase 38/40/41 baselines via `doppler run -- pnpm --filter frontend exec playwright test ... --update-snapshots`; human-eyeball pass | 1-2        |

## Risk

Low. All items pre-scoped by the v6.0 audit. No design decisions outstanding. Audit recommended exactly this surgical phase as `early in v7.0` — promoting it to its own micro-milestone (v6.1) gives cleaner provenance and a defensible ship line before the next feature push.

## Deferred

**Intelligence Engine** (signals model, structured digest, briefings beyond one-shot, alerting) — captured as seed `v7.0-intelligence-engine.md`. Trigger: v6.1 ships.
