<objective>
Phase 40 Gap-Closure AUTH-FIX: unblock all 6 Phase 40 list-pages Playwright spec files (68 tests total) by fixing the auth helper's selector mismatch. Currently `page.fill('[name="email"], input[type="email"]')` times out at 30s because the live `LoginPageAceternity.tsx` renders the email input with `id="email"` (set by `<FormInputAceternity name="email">` which forwards `id={name}`) but its top-level HTML `name` attribute lives inside react-hook-form's spread — Playwright's CSS selector doesn't reliably match against rhf-injected attributes in this codebase, hence the timeout.

The fix: switch to id-based selectors (`#email`, `#password`) which are guaranteed by `FormInputAceternity.tsx:124` (`id={name}`) and the inline `id="password"` on the password input at `LoginPageAceternity.tsx:96`.

Purpose: make `loginForListPages()` actually log in, so the existing G1–G8 code-level closures (plans 40-12..40-19) can be E2E-verified in a single sweep. Also enables the live render verification of G9, G10, G11 (plans 40-20..40-22) once those land.

Output: a single 2-line change in the auth helper + a single-spec smoke test confirming the helper now works.
</objective>
