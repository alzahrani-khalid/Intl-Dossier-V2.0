    Locate the file that initializes i18next and renders `I18nextProvider`. Add a `useEffect` that syncs `<html lang>` to `i18n.language` on every language change.

    If `frontend/src/providers/I18nProvider.tsx` exists:

    ```tsx
    import { useEffect } from 'react'
    import { useTranslation } from 'react-i18next'

    // inside component body:
    const { i18n } = useTranslation()
    useEffect(() =&gt; {
      const lang = i18n.language === 'ar' ? 'ar' : 'en'
      const dir = i18n.language === 'ar' ? 'rtl' : 'ltr'
      document.documentElement.setAttribute('lang', lang)
      document.documentElement.setAttribute('dir', dir)
    }, [i18n.language])
    ```

    If no such provider file exists, ADD this hook to `frontend/src/App.tsx` (or whichever component sits at the top of the protected route tree). The selection logic must use `i18n.language === 'ar'` exactly — do not use locale strings like `ar-SA`.

    If `<html lang>` is currently set ONLY in `index.html` (static `<html lang="en">`), this useEffect overrides it correctly at runtime. Leave the static value as `en` (default load).
