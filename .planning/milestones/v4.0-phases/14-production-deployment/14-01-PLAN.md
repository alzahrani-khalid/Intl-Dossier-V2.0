**Fix 3 — Certbot certificate path mounting (per D-01, Pitfall 2):**

In `deploy/docker-compose.prod.yml`, update the nginx service volumes to mount certbot's live directory directly instead of the stale `./nginx/ssl` copy:

Change nginx volumes from:

```yaml
volumes:
  - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
  - ./nginx/ssl:/etc/nginx/ssl:ro
  - ./certbot/www:/var/www/certbot:ro
```

to:

```yaml
volumes:
  - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
  - ./certbot/conf:/etc/letsencrypt:ro
  - ./certbot/www:/var/www/certbot:ro
```

Then update `deploy/nginx/nginx.prod.conf` SSL certificate paths from:

```
ssl_certificate /etc/nginx/ssl/fullchain.pem;
ssl_certificate_key /etc/nginx/ssl/privkey.pem;
```

to:

```
ssl_certificate /etc/letsencrypt/live/${DOMAIN}/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/${DOMAIN}/privkey.pem;
```

**IMPORTANT:** Since nginx.conf is a static file and cannot use env vars, use the literal domain name placeholder `_DOMAIN_` in the cert paths. The `setup-ssl.sh` script (or a deploy step) will `sed` replace `_DOMAIN_` with the actual domain. Update the comment in nginx.prod.conf to document this:

```nginx
# SSL Configuration — _DOMAIN_ is replaced by setup-ssl.sh with the actual domain
ssl_certificate /etc/letsencrypt/live/_DOMAIN_/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/_DOMAIN_/privkey.pem;
```

**Fix 4 — Add certbot post-renewal reload:**

In `deploy/docker-compose.prod.yml`, update the certbot entrypoint to reload nginx after renewal:

```yaml
entrypoint: '/bin/sh -c ''trap exit TERM; while :; do certbot renew --deploy-hook "docker exec intl-dossier-nginx nginx -s reload" ; sleep 12h & wait $${!}; done;'''
```

Note: The deploy-hook requires the certbot container to have access to the Docker socket OR use a different reload mechanism. Since certbot runs in its own container, a simpler approach is to add a cron on the host that reloads nginx after certbot renewal. However, the simplest fix is to just have nginx reload on a schedule. Update the nginx healthcheck to also serve as a periodic config test. The certbot renewal writes to the shared volume, and nginx will pick up new certs on reload.

Actually, the cleanest approach: add a `post-renewal` hook script. Create `deploy/certbot/renewal-hooks/deploy/reload-nginx.sh`:

```bash
#!/bin/sh
# Certbot calls this after successful renewal
# nginx reads certs from the shared /etc/letsencrypt volume
# A simple 'nginx -s reload' inside the nginx container picks up new certs
# This is handled by the host cron or docker exec
echo "Certificate renewed, nginx will pick up on next reload"
```

And update the certbot entrypoint to:

```yaml
entrypoint: "/bin/sh -c 'trap exit TERM; while :; do certbot renew; sleep 12h & wait $${!}; done;'"
```

(Keep the existing entrypoint — certbot renewal writes to the shared volume, and we add a simple nginx reload cron on the host side as part of the deploy setup.)

**Fix 5 — Create verification script `deploy/verify-deployment.sh`:**

```bash
#!/bin/bash
# Deployment verification script for Phase 14
# Usage: DOMAIN=yourdomain.com bash deploy/verify-deployment.sh

set -euo pipefail

DOMAIN="${DOMAIN:-localhost}"
PROTO="${PROTO:-https}"
BASE_URL="${PROTO}://${DOMAIN}"
ERRORS=0

echo "=== Deployment Verification ==="
echo "Target: ${BASE_URL}"
echo ""

# 1. Health check
echo -n "[1/5] Health check... "
if curl -sf "${BASE_URL}/health" > /dev/null 2>&1; then
  echo "PASS"
else
  echo "FAIL"
  ERRORS=$((ERRORS + 1))
fi

# 2. API proxy (preserves /api/ prefix)
echo -n "[2/5] API proxy... "
API_STATUS=$(curl -sf -o /dev/null -w "%{http_code}" "${BASE_URL}/api/health" 2>/dev/null || echo "000")
if [ "$API_STATUS" = "200" ]; then
  echo "PASS (HTTP $API_STATUS)"
else
  echo "FAIL (HTTP $API_STATUS)"
  ERRORS=$((ERRORS + 1))
fi

# 3. TLS certificate (skip if localhost)
if [ "$DOMAIN" != "localhost" ] && [ "$PROTO" = "https" ]; then
  echo -n "[3/5] TLS certificate... "
  if curl -vI "https://${DOMAIN}" 2>&1 | grep -q "SSL certificate verify ok"; then
    echo "PASS"
  else
    echo "FAIL"
    ERRORS=$((ERRORS + 1))
  fi
else
  echo "[3/5] TLS certificate... SKIP (localhost)"
fi

# 4. HSTS header
if [ "$PROTO" = "https" ]; then
  echo -n "[4/5] HSTS header... "
  if curl -sI "${BASE_URL}/" 2>/dev/null | grep -qi "strict-transport-security"; then
    echo "PASS"
  else
    echo "FAIL"
    ERRORS=$((ERRORS + 1))
  fi
else
  echo "[4/5] HSTS header... SKIP (HTTP mode)"
fi

# 5. HTTP->HTTPS redirect
if [ "$PROTO" = "https" ] && [ "$DOMAIN" != "localhost" ]; then
  echo -n "[5/5] HTTP->HTTPS redirect... "
  REDIRECT=$(curl -sI -o /dev/null -w "%{http_code}" "http://${DOMAIN}/" 2>/dev/null || echo "000")
  if [ "$REDIRECT" = "301" ]; then
    echo "PASS (HTTP $REDIRECT)"
  else
    echo "FAIL (HTTP $REDIRECT)"
    ERRORS=$((ERRORS + 1))
  fi
else
  echo "[5/5] HTTP->HTTPS redirect... SKIP"
fi

echo ""
if [ "$ERRORS" -eq 0 ]; then
  echo "Result: ALL CHECKS PASSED"
  exit 0
else
  echo "Result: $ERRORS CHECK(S) FAILED"
  exit 1
fi
```
