### Step 6 — Invoke `populate_diplomatic_seed()` as admin

**Action:** Log into the staging app in a browser as the admin user from Step 1 (NOT the E2E admin). Open DevTools → Application → Local Storage → copy the Supabase access token (`sb-...-auth-token`).

Alternatively (SQL editor path per `17-UAT.md` Test 4):

```sql

set local request.jwt.claims = '{"sub":"<admin-uuid>","role":"authenticated"}';
select public.populate_diplomatic_seed();
```

**Expected result:** Function returns success. Seed rows inserted with `is_seed_data = true`.

**Verification:**

```sql

  (select count(*) from dossiers where is_seed_data = true) as seed_dossiers,
  (select count(*) from countries where is_seed_data = true) as seed_countries,
  (select count(*) from organizations where is_seed_data = true) as seed_orgs,
  (select count(*) from engagements where is_seed_data = true) as seed_engagements;
```

Rows should match SEED-01 (5–10 countries, orgs, forums, engagements), SEED-02 (dossier relationships across strategic/operational/informational tiers), SEED-03 (bilingual names EN + AR present).

```sql
-- Bilingual sanity check

```

**Human checkpoint:** Counts + bilingual names confirmed. Output pasted to `evidence/20-01/05-seed-counts.txt`. — Closes **success criterion 3** and `17-UAT.md` Test 4 (SEED-04).
