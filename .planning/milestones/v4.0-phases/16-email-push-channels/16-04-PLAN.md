    **Step 3: Create PushOptInBanner.tsx**

    Component logic (per D-10, D-11, D-12):
    1. Uses `usePushSubscription()` hook from Plan 03 for `isSupported`, `permission`, `isSubscribed`, `subscribe`, `isLoading`
    2. Uses `useTranslation('push-notifications')` for bilingual strings
    3. Fetches `push_prompt_dismissed_at` from user_preferences via API on mount
    4. Visibility conditions (ALL must be true to show):
       - `isSupported === true` (browser supports push)
       - `permission !== 'granted'` (not already granted)
       - `isSubscribed === false`
       - `push_prompt_dismissed_at === null` (not dismissed)
       - `hasActionableNotification === true` (prop from parent, per D-10)
    5. "Enable notifications" button onClick:
       - Set isLoading
       - Call `subscribe()` from hook
       - If success: show toast `t('toast.enabled')`, banner auto-dismisses
       - If denied: show toast `t('toast.denied')`, call dismiss API, banner hides
       - If error: show toast `t('toast.failed')`
    6. "Not now" button onClick:
       - Call API: `PATCH /api/user-preferences` with `{ push_prompt_dismissed_at: new Date().toISOString() }`
       - Banner hides immediately (optimistic UI)
