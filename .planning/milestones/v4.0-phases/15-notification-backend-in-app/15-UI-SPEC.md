# Phase 15 -- UI Design Contract

> Visual and interaction contract for the notification backend and in-app delivery phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase nature:** This is primarily a backend integration phase. The notification UI (bell icon, panel, category tabs, mark-as-read, preferences, Realtime subscription) already exists. This contract documents the existing visual contract for verification, plus the one new UI surface: Sonner toast notifications for real-time arrivals.

## New UI Surface: Toast Notifications

The only new frontend UI work is ensuring Sonner toasts fire correctly when Supabase Realtime delivers a new notification.

### Toast Specification

| Property    | Value                                                    |
| ----------- | -------------------------------------------------------- |
| Library     | Sonner (already installed)                               |
| Position    | `top-end` (RTL-aware: top-right in LTR, top-left in RTL) |
| Duration    | 5000ms (auto-dismiss)                                    |
| Max visible | 3 stacked toasts                                         |
| Animation   | Sonner default slide-in                                  |

### Toast Variants by Category

| Category                  | Icon                     | Color Accent            | Toast Type          |
| ------------------------- | ------------------------ | ----------------------- | ------------------- |
| `assignments`             | `ClipboardList` (lucide) | `var(--heroui-accent)`  | `toast()` (default) |
| `deadlines` (approaching) | `Clock` (lucide)         | `var(--heroui-warning)` | `toast.warning()`   |
| `deadlines` (overdue)     | `Clock` (lucide)         | `var(--heroui-danger)`  | `toast.error()`     |
| `workflow` (lifecycle)    | `GitBranch` (lucide)     | `var(--heroui-accent)`  | `toast()` (default) |

### Toast Interaction

- Clicking the toast navigates to `notification.action_url` and marks the notification as read.
- Swipe-to-dismiss on mobile (Sonner built-in).
- Close button visible on hover (desktop).

Source: CONTEXT.md discretion area (toast styling), RESEARCH.md (Sonner already in use).

## Interaction States

### Notification Panel

| State                     | Visual                                                                                             |
| ------------------------- | -------------------------------------------------------------------------------------------------- |
| Loading                   | Skeleton placeholders (3 rows) using existing `Skeleton` component                                 |
| Empty (no notifications)  | Centered illustration-free layout: `Bell` icon (48px, `text-muted-foreground`), heading, body text |
| Empty (filtered category) | Same pattern with category-specific message                                                        |
| Error                     | Error text with retry link styled as `text-primary` underline                                      |
| Unread item               | `font-semibold` title + 8px accent dot on start edge                                               |
| Read item                 | `font-normal` title, no dot, `text-muted-foreground` on message                                    |
| Hover (desktop)           | `bg-muted/50` background transition 150ms                                                          |
| Active/pressed            | `bg-muted` background                                                                              |

### Bell Icon

| State      | Visual                                                                            |
| ---------- | --------------------------------------------------------------------------------- |
| No unread  | `Bell` icon at 20px, `text-muted-foreground`                                      |
| Has unread | `Bell` icon at 20px, `text-foreground` + red badge with count (max display "99+") |
| Panel open | `Bell` icon at 20px, `text-primary`                                               |

Source: Existing `NotificationPanel.tsx` and `NotificationBadge.tsx` (verification contract).

## RTL Considerations

| Element                  | RTL Behavior                                                                                                         |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------- |
| Notification item layout | `flexDirection: row` -- avatar/icon FIRST (rightmost), content SECOND, timestamp/actions LAST (leftmost)             |
| Unread dot               | Positioned at `start` edge (right in RTL, left in LTR)                                                               |
| Category tabs            | Natural scroll direction, no `.reverse()`                                                                            |
| Toast position           | `top-end` resolves to top-left in RTL                                                                                |
| Chevron icons            | Settings navigation uses `chevron-left` for drill-in (correct for RTL per CLAUDE.md Rule 5)                          |
| Text alignment           | `writingDirection: "rtl"` on Arabic text, no `textAlign: "right"` (CLAUDE.md Rule 3 -- web equivalent: `text-start`) |

Source: CLAUDE.md RTL rules, project convention.
