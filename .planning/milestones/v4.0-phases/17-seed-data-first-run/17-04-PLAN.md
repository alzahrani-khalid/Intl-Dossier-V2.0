<task type="auto">
  <name>Task 2: Build FirstRunModal component</name>
  <files>frontend/src/components/FirstRun/FirstRunModal.tsx</files>
  <action>
    Read `frontend/src/components/ui/heroui-modal.tsx` first to confirm the exact compound API exports (HeroUIModal, Modal.Header/Body/Footer or similar).

    Create `FirstRunModal.tsx` with this contract:

    ```ts
    interface FirstRunModalProps {
      open: boolean
      onOpenChange: (open: boolean) => void
      canSeed: boolean   // admin variant when true
    }
    ```

    Implementation requirements:
    - `import { useTranslation } from 'react-i18next'` with `useTranslation('sample-data')`.
    - `const { i18n } = useTranslation(); const isRTL = i18n.language === 'ar';`
    - Use the HeroUI v3 Modal wrapper from `@/components/ui/heroui-modal`.
    - `useMutation` from `@tanstack/react-query` calling `supabase.rpc('populate_diplomatic_seed')`.
    - Import the supabase client from the project's existing client module (e.g., `@/lib/supabase` — verify path).
    - Use `useQueryClient()` and on success invalidate these query key prefixes (broad invalidation per RESEARCH §5; note `['tasks']` NOT `['work-items']`):
      - `['tasks']`
      - `['dossiers']`
      - `['dossiers-for-brief']`
      - `['dashboard-success-metrics']`
      - `['dashboard-trends']`
      - `['countries']`, `['organizations']`, `['forums']`, `['engagements']`, `['persons']`
    - Mutation result handling — branch on `data.status`:
      - `'seeded'` → success toast with `successBody` interpolation from `data.counts` (pass `{ dossiers: counts.dossiers, tasks: counts.tasks, persons: counts.persons }`), then `onOpenChange(false)`.
      - `'already_seeded'` → info toast with `alreadySeededBody`, close modal.
      - `'forbidden'` → error toast with `forbiddenBody`.
      - thrown error → error toast with `errorBody`.
    - Use the project's existing toast helper (likely `sonner` `toast.success/info/error` — check `frontend/src/main.tsx` or existing components).
    - **Layout:** mobile-first. Centered modal. Body uses `flex flex-col gap-4 p-4 sm:p-6`. Buttons use `min-h-11 min-w-11`. ALL spacing uses logical properties (`ms-*`, `me-*`, `ps-*`, `pe-*`, `text-start`, `text-end`).
    - Set `dir={isRTL ? 'rtl' : 'ltr'}` on the modal content root.
    - Buttons rendered in JSX in **Arabic reading order** when RTL: primary action first (rightmost), secondary action last (leftmost). NOTE: this is **web CSS** behavior — `dir="rtl"` on the container reverses `flex-direction: row` visually so the first DOM child renders rightmost. Same end result as React Native `forceRTL`, but the mechanism is CSS, not RN layout. HeroUI Button with `variant="primary"` for Populate, `variant="ghost"` (or equivalent) for Skip.
    - **Variant logic:**
      - If `canSeed === true`: render `adminBody`, plus Populate + Skip buttons.
      - If `canSeed === false`: render `nonAdminBody` + Close button only. No populate button at all.
    - While mutation `isPending`, disable both buttons and show `loading` text on the populate button.
    - No `textAlign: right` anywhere — use `text-start` (logical).
