**Shared Cards (all dossier types use these):**

**SharedSummaryStatsCard.tsx:**

- Props: `{ dossierId: string }`
- Uses `useDossierOverview(dossierId)` for stats data
- Shows key metrics in a compact stat grid: total engagements, open tasks, linked dossiers, recent activity count
- Layout: `bg-card rounded-lg border p-4 sm:p-6`
- Title: `text-base font-semibold leading-tight text-start` — "Summary"
- Stats: 2x2 grid of stat items, each with label (`text-sm text-muted-foreground`) and value (`text-base font-semibold`)
- Empty state: "No data available" centered message if all stats are 0

**SharedRecentActivityCard.tsx:**

- Props: `{ dossierId: string, maxItems?: number }`
- Uses `useDossierActivityTimeline(dossierId)` for activity data
- Shows last 5 activities in a compact list
- Each item: icon + description + relative timestamp
- Layout: same card container
- Title: "Recent Activity"
- Empty state: "No recent activity"

**Country Cards (per D-09):**

**BilateralSummaryCard.tsx:**

- Props: `{ dossierId: string }`
- Shows bilateral relationship overview: diplomatic level, key agreements count, last high-level meeting date
- Extract data patterns from `CountryDossierDetail.tsx` (read it first to understand what data is available)
- Layout: card container with key-value pairs using `text-sm text-muted-foreground` labels

**KeyContactsCard.tsx:**

- Props: `{ dossierId: string }`
- Shows up to 5 key contacts linked to this country dossier
- Each contact: name (bilingual), title/role, organization
- "View all" link if more than 5 exist
- Contact items: clickable, navigate to person detail page

**EngagementsByStageCard.tsx:**

- Props: `{ dossierId: string }`
- Groups engagements linked to this dossier by lifecycle stage
- Each stage: name + count chip using lifecycle color coding
- Stage colors: match Phase 9 lifecycle stage colors (intake=blue, preparation=yellow, briefing=purple, execution=green, follow_up=orange, closed=gray)
- Compact: horizontal chip row or small grouped list

**Organization Cards (per D-09):**

**MembershipStructureCard.tsx:**

- Props: `{ dossierId: string }`
- Shows organization's membership tiers or departments
- Extract from `OrganizationDossierDetail.tsx` patterns

**KeyRepresentativesCard.tsx:**

- Props: `{ dossierId: string }`
- Similar to KeyContactsCard but for organization representatives
- Shows role/position within the organization

**MoUStatusCard.tsx:**

- Props: `{ dossierId: string }`
- Shows MoU/agreement status summary: count by status (active/expired/pending)
- Status badges: active=`bg-success/10 text-success`, expired=`bg-destructive/10 text-destructive`, pending=`bg-warning/10 text-warning`
- Per D-11: summary-level card, not full CRUD
- "View MoUs" action button linking to MoUs tab
