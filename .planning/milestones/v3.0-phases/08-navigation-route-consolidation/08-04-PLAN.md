**Step 1: Create `frontend/src/hooks/useRecentNavigation.ts`** per D-13:

```typescript
const STORAGE_KEY = 'recent_navigation'
const MAX_ITEMS = 10

interface RecentItem {
  path: string
  title: string
  type: 'page' | 'dossier' | 'work-item'
  icon?: string // lucide icon name for serialization
  timestamp: number
}

export function useRecentNavigation(): {
  recentItems: RecentItem[]
  addRecent: (item: Omit<RecentItem, 'timestamp'>) => void
  clearRecents: () => void
}
```

Implementation:

1. Read from `localStorage` on mount (with try/catch for SSR safety).
2. `addRecent`: Add item to front of list, deduplicate by path, trim to `MAX_ITEMS`, write to localStorage.
3. `clearRecents`: Clear localStorage key and reset state.
4. Use `useSyncExternalStore` or `useState` with localStorage sync. Keep it simple — `useState` initialized from localStorage, with `addRecent` writing to both state and localStorage.
5. **Track navigation events**: Use `useLocation()` from TanStack Router in a `useEffect` to auto-track page visits. Map current pathname to a title using the navigation config or page title. Only track meaningful navigations (skip redirects, skip consecutive same-path).

**Step 2: Enhance `CommandPalette.tsx`** per D-12, D-13, D-14, D-15:

This is an enhancement of the EXISTING CommandPalette, not a rewrite. Preserve all existing functionality (quick actions, shortcuts, keyboard navigation). Add the following:

**A. Recent items group (D-13)**:

- When command palette opens and search input is empty, show "Recents" group first with last 5-10 items from `useRecentNavigation()`.
- Each recent item shows: icon (based on type), title, path as subtitle.
- On select, navigate to the path and close palette.

**B. Grouped results (D-14)**:

- When user types, show results in 5 groups (VS Code/Raycast style):
  1. **Recents** — filter recent items matching query (top 3)
  2. **Dossiers** — from `useQuickSwitcherSearch` results filtered to dossier types (top 5)
  3. **Pages** — navigation pages matching query, derived from `createNavigationGroups` flat items list (top 5)
  4. **Work Items** — from `useQuickSwitcherSearch` results filtered to work items (top 3)
  5. **Commands** — existing quick actions (create engagement, switch language, toggle theme) matching query (top 3)
- Use `CommandGroup` with `heading` for each non-empty group.
- Use `CommandSeparator` between groups.

**C. Hybrid search (D-15)**:

- The existing `useQuickSwitcherSearch` already searches the API with debounce.
- Add cache-first behavior: Before the API search fires (during debounce delay), search TanStack Query cache using `queryClient.getQueriesData()` for any cached dossier/work-item data matching the query. Show cache results immediately with a "searching..." indicator for API results.
- Once API results arrive, merge with cache results (deduplicate by ID).

**D. Mobile full-screen overlay (D-16)**:

- On mobile (`useResponsive().isMobile`), the `CommandDialog` should render as a full-screen overlay instead of a centered modal. Apply conditional className: on mobile, override dialog positioning to `fixed inset-0 h-full max-h-full rounded-none` instead of the default centered positioning.
- The `CommandInput` on mobile should have a larger font and touch-friendly padding.

**E. Empty state**:

- When palette opens with empty input: show Recents (if any), then show navigation pages as "Quick Navigation".
- When query returns no results: show helpful message with suggestion to try different keywords.

**F. Navigation on select**:

- All items use `useNavigate()` from TanStack Router on select.
- Close palette after navigation.
- Add selected item to recent navigation via `addRecent`.

RTL: All text alignment via `text-start`. Icon margins via `ms-*` / `me-*`. CommandInput placeholder text is i18n-translated.

Code style: no semicolons, single quotes, trailing commas, explicit return types, no `any`.

IMPORTANT: Keep the existing CommandPalette exports intact (`CommandPalette`, `ShortcutHint`, `ShortcutGuide`). Do not break the import in `__root.tsx`.

Bundle consideration: The CommandPalette is already in `__root.tsx` (always loaded). Keep search hooks lazy — `useQuickSwitcherSearch` is already debounced. Recent items use lightweight localStorage. No new heavy imports.
