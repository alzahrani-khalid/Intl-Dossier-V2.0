**1. Create Supabase migration** `supabase/migrations/20260330000001_operations_hub_rpcs.sql` with 4 RPC functions:

**`get_attention_items(p_user_id UUID DEFAULT NULL, p_threshold_hours INTEGER DEFAULT 48)`**
Returns TABLE with columns: `id UUID, item_type TEXT, severity TEXT, title TEXT, title_ar TEXT, entity_id UUID, entity_type TEXT, deadline TIMESTAMPTZ, days_overdue NUMERIC, days_in_stage NUMERIC, lifecycle_stage TEXT, engagement_name TEXT, engagement_name_ar TEXT`.
4 RETURN QUERY blocks:

- Overdue work items (red): `FROM unified_work_items_view wi WHERE wi.is_overdue = true AND wi.status NOT IN ('completed','cancelled') AND (p_user_id IS NULL OR wi.assignee_id = p_user_id)`
- Due-soon work items (yellow): same view, `wi.is_overdue = false AND wi.deadline IS NOT NULL AND wi.deadline <= NOW() + (p_threshold_hours || ' hours')::INTERVAL AND wi.status NOT IN ('completed','cancelled')`
- SLA-at-risk intake tickets (yellow): `FROM intake_tickets it WHERE it.tracking_type = 'sla' AND it.sla_deadline IS NOT NULL AND it.sla_deadline <= NOW() + (p_threshold_hours || ' hours')::INTERVAL AND it.status NOT IN ('resolved','closed','cancelled')`
- Stalled engagements (orange): `FROM engagement_dossiers ed JOIN dossiers d ON d.id = ed.id LEFT JOIN LATERAL (SELECT transitioned_at FROM lifecycle_transitions WHERE engagement_id = ed.id ORDER BY transitioned_at DESC LIMIT 1) lt ON true WHERE ed.lifecycle_stage != 'closed' AND d.status = 'active' AND lt.transitioned_at IS NOT NULL AND lt.transitioned_at <= NOW() - INTERVAL '14 days'`
  Use `LANGUAGE plpgsql SECURITY DEFINER`.

**`get_upcoming_events(p_user_id UUID DEFAULT NULL, p_days_ahead INTEGER DEFAULT 14)`**
Returns TABLE: `id UUID, title TEXT, title_ar TEXT, start_date TIMESTAMPTZ, end_date TIMESTAMPTZ, event_type TEXT, engagement_id UUID, engagement_name TEXT, engagement_name_ar TEXT, lifecycle_stage TEXT`.
Query `engagement_dossiers ed JOIN dossiers d` for engagements with `start_date` within p_days_ahead, plus `calendar_entries` for additional events. Order by `start_date ASC`.

**`get_engagement_stage_counts(p_user_id UUID DEFAULT NULL)`**
Returns TABLE: `stage TEXT, stage_count INTEGER, top_engagements JSONB`.
Group `engagement_dossiers` by `lifecycle_stage`, count per stage, include top 5 engagements per stage as JSONB array with `{id, name_en, name_ar, engagement_type, updated_at}`. Filter to `status = 'active'` dossiers only.

**`get_dashboard_stats(p_user_id UUID DEFAULT NULL)`**
Returns TABLE: `active_engagements INTEGER, open_tasks INTEGER, sla_at_risk INTEGER, upcoming_week INTEGER`.
Single row with: count of active engagement_dossiers, count of unified_work_items_view with status not completed/cancelled, count of intake_tickets with tracking_type='sla' approaching deadline, count of events in next 7 days.

All functions accept optional `p_user_id` — when NULL return all items, when set filter to user's assigned items per D-11.
