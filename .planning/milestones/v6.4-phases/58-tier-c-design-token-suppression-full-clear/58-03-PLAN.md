## Token mapping rules

Apply the token mapping table at `58-01-PLAN.md:110-127` verbatim (Pattern 1). Apply the Wave 2 additions in `58-02-PLAN.md` lines 80–110 verbatim (text/border/background swaps including `bg-blue-50 dark:bg-blue-900/20 → bg-info/10` for informational banners and `bg-blue-100 → bg-primary/10` for selected rows).

Wave-3-specific decisions:

- **Status banners** (info / success / warning / danger): `border border-{tone}/30 bg-{tone}/10 text-{tone}` — no `dark:` chains.
- **Selected list row** (EntitySearchDialog, BulkActionPreviewDialog rows): `bg-primary/10` (selected) / `bg-muted` (hover).
- **3-way diff** (ConflictResolutionDialog): same → `bg-muted text-ink-mute`; conflict → `bg-danger/10 text-danger border-danger/30`; merged → `bg-success/10 text-success border-success/30`.
- **Blue-purple collisions** (6 files): For each `text-blue-*` site, decide by semantic context — `text-accent` for link/CTA (per D-58-02-EXTRA-03), `text-info` for informational icons, `text-primary` for selected outline. For `purple` sites (rare in dialogs), follow Wave 2 WGMemberSuggestions precedent: `purple → accent`.
- **Type badges in EntitySearchDialog**: honor `dossierTypeColors` map from `frontend/src/lib/semantic-colors.ts` (import the map; do not redefine the 8 dossier-type → ColorSet mapping inline).
- **Drawer surface** (`CommitmentDetailDrawer.tsx`): `bg-surface-raised` for the drawer body if `bg-white dark:bg-gray-900` is the current literal; `border-line` for divider rules.
