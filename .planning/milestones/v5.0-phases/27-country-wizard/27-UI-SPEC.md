## Typography

Weights: 2 (`400 normal`, `600 semibold`). Sizes: 4 (`12px`, `13px`, `14px`, `18px`).

| Role           | Size             | Weight         | Line Height | CSS Variable |
| -------------- | ---------------- | -------------- | ----------- | ------------ |
| Label          | 12px (0.75rem)   | 400 (normal)   | 1.375       | --text-sm    |
| Review label   | 12px (0.75rem)   | 400 (normal)   | 1.375       | --text-sm    |
| Body           | 13px (0.8125rem) | 400 (normal)   | 1.5         | --text-base  |
| Review value   | 13px (0.8125rem) | 400 (normal)   | 1.5         | --text-base  |
| Step title     | 14px (0.875rem)  | 600 (semibold) | 1.25        | --text-lg    |
| Review heading | 14px (0.875rem)  | 600 (semibold) | 1.25        | --text-lg    |
| Wizard heading | 18px (1.125rem)  | 600 (semibold) | 1.25        | --text-2xl   |

Label and Review label use weight 400 (normal) -- the label/value distinction is carried by size (12px vs 13px) and color (`text-muted-foreground` vs `text-foreground`), making a separate weight unnecessary.

Source: Phase 26 UI-SPEC (inherited) + review-specific roles added. Weight 500 removed per checker feedback.

## Interaction Contracts

### Auto-Fill Flow

| Interaction         | Behavior                                                                                                          |
| ------------------- | ----------------------------------------------------------------------------------------------------------------- |
| Trigger             | `name_en` field value changes with length >= 3 characters, debounced 500ms                                        |
| Lookup              | `GET /api/countries?search={name_en}&limit=1` via TanStack Query (5 min staleTime)                                |
| Auto-fill condition | Only fill fields that are still empty (default value). Never overwrite user-entered data.                         |
| Fields auto-filled  | `iso_code_2`, `iso_code_3`, `region`, `capital_en`, `capital_ar` (if available in reference data)                 |
| Visual feedback     | Brief 200ms `bg-accent/10` flash on auto-filled fields + toast notification                                       |
| No match            | Silent -- no error, no toast. User fills fields manually.                                                         |
| User override       | User can edit any auto-filled field. Once edited, that field is no longer eligible for auto-fill on next trigger. |
| Fallback            | If API unavailable, use `country-codes.ts` for ISO code matching only (no region/capital).                        |

### Review Step Navigation

| Interaction        | Behavior                                                                               |
| ------------------ | -------------------------------------------------------------------------------------- |
| Edit button click  | Calls `wizard.setCurrentStep(targetStepIndex)`. Form state preserved.                  |
| Return to review   | User clicks "Continue" through steps normally, or clicks step 3 in progress indicator. |
| Data freshness     | Review step uses `form.watch()` for live values -- never stale after edits.            |
| Submit from review | "Create Country" button triggers `form.handleSubmit()` with full schema validation.    |

### ISO Code Input Behavior

| Interaction   | Behavior                                                                       |
| ------------- | ------------------------------------------------------------------------------ |
| Typing        | Auto-uppercase via CSS `uppercase` class + `toUpperCase()` transform on change |
| Max length    | ISO-2: 2 characters, ISO-3: 3 characters (enforced by `maxLength` attribute)   |
| Validation    | Zod schema: `^[A-Z]{2}$` for ISO-2, `^[A-Z]{3}$` for ISO-3                     |
| Error display | Inline field error below input per Phase 26 convention                         |

### Region Dropdown Behavior

| Interaction        | Behavior                                                         |
| ------------------ | ---------------------------------------------------------------- |
| Options            | 6 predefined: asia, africa, europe, americas, oceania, antarctic |
| Labels             | Bilingual via i18n keys (`form-wizard:regions.{value}`)          |
| Pre-selection      | If auto-fill provides region, pre-select it. User can change.    |
| No selection state | Placeholder "Select a region" shown in muted text                |

### Step Validation Rules

| Step            | Required Fields                      | Optional Fields                                      |
| --------------- | ------------------------------------ | ---------------------------------------------------- |
| Basic Info      | name_en, name_ar                     | abbreviation, description, tags, status, sensitivity |
| Country Details | iso_code_2, iso_code_3, region       | capital_en, capital_ar                               |
| Review          | No additional validation (read-only) | --                                                   |
