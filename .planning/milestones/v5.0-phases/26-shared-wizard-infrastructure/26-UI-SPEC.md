## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                           |
| ----- | ----- | ----------------------------------------------- |
| xs    | 4px   | Icon gaps, inline padding                       |
| sm    | 8px   | Compact element spacing, field-to-label gap     |
| md    | 16px  | Default element spacing, step content padding   |
| lg    | 24px  | Section padding, step-to-step gap               |
| xl    | 32px  | Layout gaps, wizard container padding (desktop) |
| 2xl   | 48px  | Major section breaks                            |
| 3xl   | 64px  | Page-level spacing                              |

Exceptions: 44px minimum touch targets on all wizard navigation buttons (Back, Next, Submit, Save Draft).

## Component Inventory

Phase 26 produces shared building blocks, not standalone pages. The components below define the visual contract downstream wizards inherit.

### CreateWizardShell

Wraps `FormWizard` with dossier-specific orchestration. Visual additions over FormWizard:

| Element         | Specification                                                                                |
| --------------- | -------------------------------------------------------------------------------------------- |
| Container       | `max-w-2xl mx-auto px-4 sm:px-6 lg:px-8` (centered, responsive padding)                      |
| Step progress   | Reuse existing `FormWizard` step indicator (numbered dots + titles)                          |
| Draft indicator | Small `Save` icon + "Draft saved" text at top-end corner, fades in/out with 250ms transition |
| Action bar      | `actionBarMode: 'auto'` -- sticky on mobile, inline on desktop                               |
| Animation       | Existing Framer Motion step transitions from `FormWizard` (no new animations)                |
| Bilingual       | Step titles from i18n, direction from `useDirection()`                                       |

### Step Layout Convention

Each wizard step (built in Phases 27-31) follows this layout contract:

| Element                | Specification                                                                                                                                         |
| ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| Step container         | `space-y-6` (24px vertical gap between field groups)                                                                                                  |
| Field group            | `space-y-2` (8px gap between label and input)                                                                                                         |
| Field label            | `--text-sm` (12px), weight 500, color `--base-500`                                                                                                    |
| Field input            | HeroUI form wrappers (`heroui-forms.tsx`), min-height 44px touch target                                                                               |
| Bilingual field pairs  | Two inputs side by side on `sm:` breakpoint, stacked on mobile. English start, Arabic end (flips in RTL). Use `grid grid-cols-1 sm:grid-cols-2 gap-4` |
| Classification section | Collapsible via `details/summary` or Accordion. Collapsed by default. Label: "Classification" / "\u0627\u0644\u062a\u0635\u0646\u064a\u0641"          |
| Required indicator     | Red asterisk `*` after label text, color `--heroui-danger`                                                                                            |

### Draft Migration (INFRA-07)

No visual component. Silent migration runs in `useCreateDossierWizard` on mount. No toast, no UI indicator per D-03.

## Interaction Contracts

### Step Navigation

| Interaction          | Behavior                                                                                                                                 |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| Next button          | Validates current step fields via `form.trigger()`. If valid, advance. If invalid, show field-level errors and validation summary toast. |
| Back button          | Always enabled (no validation needed). Preserves form state.                                                                             |
| Step indicator click | Allowed only for completed (previously visited) steps. Validates current step before jumping.                                            |
| Keyboard             | `Enter` on last field of step triggers Next. `Escape` on wizard triggers cancel confirmation if form is dirty.                           |

### Draft Persistence

| Interaction         | Behavior                                                                                                  |
| ------------------- | --------------------------------------------------------------------------------------------------------- |
| Auto-save trigger   | Debounced 2 seconds after last field change                                                               |
| Storage key         | `dossier-create-{type}` per INFRA-05                                                                      |
| Draft restore       | On mount, if draft exists for type, restore and show toast                                                |
| Draft discard       | Inline confirmation (see Copywriting), then `localStorage.removeItem()`                                   |
| Old draft migration | Silent per D-03. Old `dossier-create-draft` key parsed, split by type, saved to new keys, old key deleted |

### Form Submission

| Interaction         | Behavior                                                                                              |
| ------------------- | ----------------------------------------------------------------------------------------------------- |
| Submit button       | Disabled while `isLoading`. Shows `Loader2` spinner icon replacing text.                              |
| Success             | Toast "Dossier created", navigate to `getDossierDetailPath()`, clear draft                            |
| Error               | Toast with submission error copy, button re-enabled, form state preserved                             |
| Duplicate detection | If API returns duplicate warning, show inline alert below submit button with link to existing dossier |

## RTL Contract

All wizard infrastructure follows project RTL rules from CLAUDE.md:

| Rule                     | Implementation                                                                                                |
| ------------------------ | ------------------------------------------------------------------------------------------------------------- |
| Logical properties only  | `ms-*`, `me-*`, `ps-*`, `pe-*`, `text-start`, `text-end`                                                      |
| Direction detection      | `useDirection()` hook, never prop-drilled                                                                     |
| Directional icons        | Back = `ChevronRight` in RTL (points right = "return"), Next = `ChevronLeft` in RTL (points left = "forward") |
| Text alignment           | `text-start` for form labels, `text-center` for step titles. No `text-right`.                                 |
| Bilingual field ordering | In RTL: Arabic field renders first (start), English field second (end)                                        |
