Planner implicitly reconciled this; flagging for visibility only.

## Findings

### Blocking (must fix before execution)

None.

### Warnings (should address but not blocking)

1. **SPEC-to-migration column count discrepancy (11 vs 10)** — SPEC says "11 new columns"; migration adds 10 (photo_url pre-exists). Add a one-line comment in the migration header clarifying this to prevent a future auditor from flagging it as a miss.
2. **`javascript:` URL in `<img src>`** — Photo URL preview renders whatever the user typed. While `<img src=javascript:...>` does not execute in modern browsers, consider a lightweight `new URL(photo_url).protocol` allowlist (`http:`, `https:`, `data:`) in 32-03 IdentityCard and 32-02 schema. Non-blocking; deferrable.
3. **32-03 unused `isRTL` local** — computed from `i18n.language === 'ar'` but only used for the nationality label; fine but trim during review.
4. **DossierPicker E2E flakiness** — 32-04 T4 spec picks the first matching country option; ensure the test seed guarantees Saudi Arabia country dossier exists, or tighten to a specific UUID via a `beforeAll` SQL probe.

### Suggestions (optional)

- Add a Playwright accessibility sweep (`@axe-core/playwright`) on Step 1 to catch missing labels/`aria-*` attributes early.
- Consider adding a tiny Vitest case in 32-02 T8 for single-word AR name (`last_name_ar: 'زيد'`, `first_name_ar: null`) to lock the D-07 behavior.
