## Spacing Scale

Declared values (multiples of 4). Already defined in `modern-nav-tokens.css` as `--space-*`.

| Token | Value | Usage                                |
| ----- | ----- | ------------------------------------ |
| xs    | 4px   | Icon gaps, inline padding            |
| sm    | 8px   | Compact element spacing, mobile gaps |
| md    | 16px  | Default element spacing              |
| lg    | 24px  | Section padding, card inner padding  |
| xl    | 32px  | Layout gaps between major sections   |
| 2xl   | 48px  | Major section breaks                 |
| 3xl   | 64px  | Page-level vertical spacing          |

Exceptions:

- Touch targets: 44px minimum (`--icon-button-touch-target: 44px` already tokenized)
- Icon rail width: 56px (fixed, not from spacing scale)
- Expanded panel width: 240px (fixed navigation width)

Source: modern-nav-tokens.css `--space-*` tokens

## Typography

Already tokenized in `index.css` as `--text-*` and `--font-*`. This phase does NOT change the type scale -- it ensures the existing scale is applied responsively.

Two weights only: 400 (normal) and 600 (semibold).

| Role       | Size (mobile)        | Size (desktop)       | Weight                  | Line Height              |
| ---------- | -------------------- | -------------------- | ----------------------- | ------------------------ |
| Body       | 13px (`--text-base`) | 13px (`--text-base`) | 400 (`--font-normal`)   | 1.5 (`--leading-normal`) |
| Label      | 11px (`--text-xs`)   | 11px (`--text-xs`)   | 400 (`--font-normal`)   | 1.375 (`--leading-snug`) |
| Section    | 14px (`--text-lg`)   | 16px (`--text-xl`)   | 600 (`--font-semibold`) | 1.25 (`--leading-tight`) |
| Page title | 18px (`--text-2xl`)  | 20px (`--text-3xl`)  | 600 (`--font-semibold`) | 1.25 (`--leading-tight`) |

Responsive rule: Section headers and page titles step down one size on viewports under 640px. Body and label sizes remain constant across all breakpoints (WCAG minimum compliance).

Source: index.css `--text-*`, `--font-*`, `--leading-*` tokens

## Breakpoint Contract

This is the core contract for Phase 5. All pages must conform to these rules.

### Breakpoint Definitions

| Name    | Min-width | Tailwind prefix | Layout behavior                     |
| ------- | --------- | --------------- | ----------------------------------- |
| Mobile  | 0px       | (base)          | Single column, hamburger nav, cards |
| Small   | 640px     | `sm:`           | Wider cards, 2-col where useful     |
| Tablet  | 768px     | `md:`           | Icon rail nav, 2-col grids          |
| Desktop | 1024px    | `lg:`           | Full sidebar nav, 3-col grids       |
| Wide    | 1280px    | `xl:`           | Max content width, wider tables     |

Source: CONTEXT.md D-01, CLAUDE.md breakpoints

### Navigation Behavior (D-01, D-02)

| Viewport   | Navigation pattern                                                                               |
| ---------- | ------------------------------------------------------------------------------------------------ |
| < 768px    | Hamburger button in header, full-screen drawer overlay                                           |
| 768-1023px | Icon rail only (56px), no expanded panel, tooltips on hover for each icon (using HeroUI Tooltip) |
| >= 1024px  | Icon rail (56px) + expanded panel (240px), full sidebar                                          |

Component: `NavigationShell` replaces `AppSidebar` + `Header` in `_protected.tsx`.

### Data Table Behavior (D-03, D-04)

| Viewport | Table pattern                                                  |
| -------- | -------------------------------------------------------------- |
| < 768px  | Card view (stacked cards, one per row). Hide column headers.   |
| >= 768px | Standard table with horizontal scroll if columns exceed width. |

Card view uses existing `DataTable` pattern: `enableViewToggle`, `mobileCardColumns`, `cardTitleColumn` props. Extend to `AdvancedDataTable` and `SelectableDataTable`.

Bulk actions (SelectableDataTable): Sticky bottom bar via `mobile-action-bar.tsx` on viewports < 768px. Desktop keeps inline toolbar.

### Modal/Form Behavior (D-07, D-08)

| Viewport | Pattern                                               |
| -------- | ----------------------------------------------------- |
| < 768px  | Bottom sheet (Vaul) with drag-to-dismiss, snap points |
| >= 768px | Centered dialog (HeroUI Modal)                        |

Forms: Single scrollable layout, all fields stacked vertically, sticky save button at bottom on mobile. No stepped wizards.

### Grid Layouts

| Viewport   | Grid columns                 |
| ---------- | ---------------------------- |
| < 640px    | 1 column (`grid-cols-1`)     |
| 640-767px  | 2 columns (`sm:grid-cols-2`) |
| 768-1023px | 2 columns (`md:grid-cols-2`) |
| >= 1024px  | 3 columns (`lg:grid-cols-3`) |

Dashboard metric cards, dossier lists, and settings grids follow this pattern.

## Touch Target Contract (RESP-02)

Every interactive element must meet these minimums:

| Element type        | Minimum size | Implementation                             |
| ------------------- | ------------ | ------------------------------------------ |
| Buttons             | 44x44px      | `min-h-11 min-w-11` (Tailwind h-11 = 44px) |
| Icon buttons        | 44x44px      | Use `touch-target.tsx` wrapper             |
| Links (standalone)  | 44x44px      | `min-h-11 inline-flex items-center`        |
| Links (inline text) | No minimum   | Inherits line height                       |
| Form inputs         | 44px height  | `min-h-11`                                 |
| Checkboxes/Radios   | 44x44px      | Expand hit area via padding                |
| Table rows (mobile) | 48px height  | Card view with `min-h-12 p-4`              |

Gap between adjacent touch targets: minimum 8px (`gap-2`).

Source: CLAUDE.md touch targets, REQUIREMENTS.md RESP-02

## Responsive Interaction Patterns

### Sidebar Drawer (Mobile < 768px)

- Opens from start edge (right in RTL, left in LTR)
- Overlay backdrop with `opacity-50 bg-black`
- Closes on: backdrop tap, swipe toward edge, close button, Escape key
- Trap focus inside drawer when open
- Z-index: `--z-modal` (1050)

### Icon Rail Tooltips (Tablet 768-1023px)

- Each icon rail item displays a HeroUI Tooltip on hover/focus
- Tooltip placement: `end` (appears to the left in RTL, right in LTR)
- Tooltip delay: 0ms (immediate)
- Tooltip content: the full navigation label text (same as expanded panel label)
- Visually hidden `<span>` with the label text inside each icon button for screen readers (`sr-only` class)

### Bottom Sheet (Mobile Modals)

- Snap points: small (30vh), medium (50vh), large (75vh), full (100vh)
- Drag handle: 32px wide, 4px tall, centered, `bg-muted-foreground/30`
- Keyboard avoidance: sheet pushes up when virtual keyboard appears
- Uses existing `bottom-sheet.tsx` (Vaul-based)

### Sticky Bottom Action Bar

- Height: 56px + safe area inset bottom
- Background: `bg-background/95 backdrop-blur-sm`
- Border top: `border-t border-border`
- Z-index: `--z-sticky` (1020)
- Uses existing `mobile-action-bar.tsx`

### Scroll Behavior

- Tables on tablet: horizontal scroll with `-webkit-overflow-scrolling: touch`
- Modals: vertical scroll within body, fixed header/footer
- Page content: natural scroll, no scroll snap

## Implementation Constraints

### RTL Compliance (from Phase 4)

- Use `useDirection()` hook -- never re-derive `isRTL` from `i18n.language`
- All spacing uses logical properties: `ms-*`, `me-*`, `ps-*`, `pe-*`
- No `text-left`, `text-right` -- use `text-start`, `text-end`
- Drawer opens from `inset-inline-start` (start edge in both RTL and LTR)
- Directional icons: `className={isRTL ? 'rotate-180' : ''}` for chevrons

### Mobile-First CSS Order

All responsive styles MUST be written mobile-first:

```
base styles (mobile) -> sm: -> md: -> lg: -> xl:
```

Never write desktop-first and then override down.
