## Trust Boundaries

| Boundary                   | Description                                                                                                                                   |
| -------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| eval judge → on-prem gemma | `judgeRubric` contacts ONLY `getCopilotModel().url` (on-prem vLLM/gemma). No cloud judge — zero-egress, the same invariant the copilot holds. |
| CI runner → judge endpoint | When the endpoint env is unset, live scoring is skipped (deploy gate) so CI never depends on the GPU host or a secret.                        |

## STRIDE Threat Register

| Threat ID  | Category               | Component               | Disposition | Mitigation Plan                                                                                                                                                |
| ---------- | ---------------------- | ----------------------- | ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-74-09-01 | Information disclosure | zero-egress invariant   | mitigate    | The judge uses `getCopilotModel()` (on-prem) only; no fixture content or candidate is sent to a cloud LLM. Fixtures are synthetic — no real classified briefs. |
| T-74-09-02 | Tampering              | judge output            | mitigate    | Validate the judge response through `RubricScoreSchema.parse` — a malformed/over-range score throws rather than silently passing the gate.                     |
| T-74-09-03 | Denial of service      | judge unavailable in CI | mitigate    | `isJudgeConfigured()` gates live scoring; absent endpoint → `it.skip`, so a missing GPU host does not fail or hang CI (D6).                                    |
| T-74-09-04 | Repudiation            | regression detection    | mitigate    | Each rubric's `_degraded.json` proves the gate rejects a below-threshold output; the structural/positive-failure path runs even without the judge.             |
| T-74-09-05 | Spoofing               | self-grading bias       | accept      | D4 accepts self-grading bias (regression-delta gate, not an absolute oracle); the judge is on-prem and consistent across runs.                                 |
