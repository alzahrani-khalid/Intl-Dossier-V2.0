<objective>
Re-home country-intelligence generation (`intelligence-refresh`, `intelligence-refresh-v2`, `intelligence-batch-update`) from AnythingLLM workspace-chat to the on-prem vLLM OpenAI-compatible `/v1/chat/completions` endpoint (D3). This is **net-new generation** — there is no existing on-prem path for these fns — and the single largest EVAL-04 task (~1766 lines across three fns).

The current path calls `${ANYTHINGLLM_URL}/api/v1/workspace/country-intelligence/chat` after `ensureWorkspaceExists` + an `/api/health` probe, producing structured JSON that lands in `intelligence_reports` (`title`, `title_ar`, `content`, `content_ar`, `confidence_level`, `data_sources_metadata`). The replacement uses the SAME on-prem endpoint the copilot binds to (`VLLM_BASE_URL` → `http://vllm:8000`, model `gemma-4-12b`, JSON mode) and preserves the write contract exactly so existing `intelligence-get`/UI rows are unchanged.

Purpose: Remove AnythingLLM from the last runtime generator on the critical path while keeping country-intel generation working on-prem (EVAL-04). Output: a shared on-prem chat helper + three re-homed edge fns; zero AnythingLLM reference in them.
</objective>
