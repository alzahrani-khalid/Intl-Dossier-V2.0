## Interaction Contracts

### Breadcrumb Behavior

- Breadcrumb trail renders on all 8 dossier type list pages: countries, organizations, forums, engagements, topics, working-groups, persons, elected-officials
- Current page segment is non-clickable, styled with `font-semibold` and accent color
- All ancestor segments are clickable `Link` components
- RTL: breadcrumb separator order handled by logical flex direction (no `.reverse()`)
- Mobile: breadcrumb collapses to show only current + parent; ellipsis for deeper levels

### Skeleton Loading

- Each dossier list page shows its type-specific skeleton from `DossierLoadingSkeletons` during data fetch
- Skeleton uses `animate-pulse` with `bg-muted` base (existing `Skeleton` component)
- Skeleton layout must match the final rendered layout structure (same grid columns, same card heights)
- Duration: skeleton shows for minimum 0ms (instant swap when data arrives, no artificial delay)

### Pagination URL State (D-33, D-34)

- Page number persisted in URL: `?page=2`
- Browser refresh preserves current page
- Page size default: 20 items (not configurable in URL for this phase)
- Out-of-range page numbers clamp to valid range silently

### Kanban Filter URL State (D-34)

- Filter state serialized to URL search params: `?status=in_progress&priority=high`
- Clearing all filters removes search params (clean URL)
- Shareable: pasting a filtered URL applies those filters on load

### Version Conflict (D-41)

- On save attempt, compare local `updated_at` with server `updated_at`
- If mismatch: show destructive-colored banner with "Review Changes" CTA
- CTA opens a diff view or prompts user to reload before editing
- No silent overwrite permitted

### Wizard Decomposition (C-12)

- Current monolithic `DossierCreateWizard` splits into step components
- Each step component is under 300 LOC
- Step navigation via stepper bar (already exists in project patterns)
- "Discard Draft" is a destructive action requiring confirmation dialog
