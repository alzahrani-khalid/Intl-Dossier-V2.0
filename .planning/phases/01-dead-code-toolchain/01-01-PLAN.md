**Step 2: Count remaining unfixable violations.**


**Step 3: Fix remaining violations manually.** The most common unfixable violations will be:

- `@typescript-eslint/explicit-function-return-type`: Add return type annotations to functions missing them. For React components, add `: React.ReactNode` or `: JSX.Element`. For hooks, add the specific return type. For event handlers inside components (arrow functions passed as props), these are allowed by `allowExpressions: true`.
- `@typescript-eslint/no-explicit-any`: Replace `any` with proper types. Use `unknown` when the type is genuinely unknown, then narrow with type guards. For third-party library callbacks where the type is complex, use the library's exported types.
- `@typescript-eslint/strict-boolean-expressions`: Replace truthy checks like `if (value)` with explicit checks like `if (value !== null && value !== undefined)` or `if (value !== '')` or `if (value.length > 0)` depending on the type.
- `@typescript-eslint/no-floating-promises`: Add `void` prefix to fire-and-forget promises: `void someAsyncFn()`. Or add `.catch()` handlers. For event handlers, wrap in `void`: `onClick={() => void handleAsync()}`.
- `no-console`: Replace `console.log` with `console.warn` or `console.error`, or remove debug logging entirely.
- `no-restricted-syntax` (RTL violations in frontend): Replace physical CSS classes with logical equivalents per CLAUDE.md RTL table: `ml-*` to `ms-*`, `mr-*` to `me-*`, `pl-*` to `ps-*`, `pr-*` to `pe-*`, `text-left` to `text-start`, `text-right` to `text-end`, `left-*` to `start-*`, `right-*` to `end-*`, `rounded-l-*` to `rounded-s-*`, `rounded-r-*` to `rounded-e-*`, `border-l-*` to `border-s-*`, `border-r-*` to `border-e-*`.

**Step 4: Re-run ESLint to confirm zero errors.**

```bash
pnpm exec eslint frontend/src backend/src --max-warnings 0 --no-warn-ignored
```

**IMPORTANT:** If the violation count is extremely large (500+), prioritize by category:

1. First fix all auto-fixable (Step 1)
2. Then RTL violations (small count, mechanical replacement)
3. Then `no-console` (remove or convert)
4. Then `no-floating-promises` (add `void` prefix)
5. Then `explicit-function-return-type` (add annotations)
6. Then `no-explicit-any` (replace with proper types)
7. Then `strict-boolean-expressions` (explicit null/undefined checks)

For files with excessive violations that cannot be fixed without changing behavior, add targeted eslint-disable comments with the specific rule name and a TODO referencing the appropriate future phase:

```typescript

```
