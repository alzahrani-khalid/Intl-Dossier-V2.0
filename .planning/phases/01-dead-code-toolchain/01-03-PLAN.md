**Step 1: Update lint-staged config in `package.json` (per D-12, D-14).**

Replace the existing `"lint-staged"` section in root `package.json` with:

```json
{
  "lint-staged": {
    "frontend/**/*.{ts,tsx}": ["eslint --fix --no-warn-ignored", "prettier --write"],
    "backend/**/*.ts": ["eslint --fix --no-warn-ignored", "prettier --write"],
    "*.{json,md,yml,yaml,css}": ["prettier --write"]
  }
}
```

This runs ESLint (with auto-fix) and Prettier on staged TS/TSX files only (per D-14: scoped to staged files).

**Step 2: Update `.husky/pre-commit` script (per D-12, D-13).**

Write this exact content to `.husky/pre-commit`:

```bash
#!/usr/bin/env sh

# Run lint-staged for ESLint + Prettier on staged files
npx lint-staged

# Run full-project checks (cannot be scoped to staged files)
# --incremental caches type info in .tsbuildinfo for faster subsequent runs
pnpm exec tsc --noEmit --incremental
pnpm exec knip --no-progress
```

Note: `tsc --noEmit` and `knip` must check the full project because they cannot operate on individual files (per RESEARCH.md Pattern 3 note). The `--incremental` flag on tsc caches type info in `.tsbuildinfo` files, making subsequent checks 2-5s instead of 15-30s (per Pitfall 5).

**Step 3: Add `.tsbuildinfo` to `.gitignore`.**

Append to `.gitignore` if not already present:

```
# TypeScript incremental build cache (used by pre-commit tsc --incremental)
*.tsbuildinfo
```

**Step 4: Ensure the husky hook is executable.**

```bash
chmod +x .husky/pre-commit
```

**Per D-09 and TOOL-05:** Update all dependencies to latest stable versions. Per D-11: keep `^` semver ranges.

**Step 1: Check for outdated packages.**

```bash
pnpm outdated --recursive 2>&1
```

**Step 2: Update all packages to latest within semver range.**

```bash
pnpm update --recursive --latest
```

The `--latest` flag updates to the latest version, updating the `^` range in package.json. This is what D-09 requires: latest stable, not just within current range.

**Step 3: Verify no breaking changes.**

After update, run the full verification suite:

```bash
# TypeScript compilation
pnpm exec tsc --noEmit --project frontend/tsconfig.json
pnpm exec tsc --noEmit --project backend/tsconfig.json

# ESLint
pnpm exec eslint frontend/src backend/src --max-warnings 0 --no-warn-ignored

# Knip
pnpm exec knip --no-progress

# Build
pnpm build
```

**Step 4: If any check fails after update, investigate and fix.**

Common issues after major dependency updates:

- Type errors from stricter types in updated packages: fix the types
- API changes in updated packages: check the changelog and update usage
- Peer dependency conflicts: resolve by aligning versions

If a specific package update causes breaking changes that are too large to fix in this plan, pin that package to its current version with a TODO comment:

```json
"problematic-package": "^current.version.here"
// TODO: Update to latest after resolving breaking changes
```

**Step 5: Verify `pnpm outdated` shows minimal or zero outdated packages.**

```bash
pnpm outdated --recursive 2>&1 | head -20
```

Some packages may have newer major versions that are intentionally not adopted (e.g., ESLint 10 due to react-hooks plugin). This is acceptable.

**Step 6: Run Prettier check to ensure formatting is still clean.**

