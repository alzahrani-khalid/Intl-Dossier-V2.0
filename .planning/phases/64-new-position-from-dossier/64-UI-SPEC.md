# Phase 64 — UI Design Contract

> Visual and interaction contract for the New Position quick-create dialog (POSNEW-01/POSNEW-02). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope:** one dialog (`NewPositionDialog`, extracted from the broken `PositionDialog` in `AddToDossierDialogs.tsx`) plus the D-13 rewire of the `DossierPositionsTab` header actions. This is a fix inside an established design system — NOT a design-system change. Out of scope per REQUIREMENTS.md: net-new design tokens or direction variants.

## Spacing Scale

Declared values come from the existing Bureau tokens (`colors_and_type.css` `--space-*`; all multiples of 4). Do not invent new spacing.

| Token                              | Value | Usage in this phase                                                                                                   |
| ---------------------------------- | ----- | --------------------------------------------------------------------------------------------------------------------- |
| `--space-1` / `gap-1`              | 4px   | Icon-to-text gap inside translate buttons                                                                             |
| `--space-2` / `space-y-2`, `gap-2` | 8px   | Label-to-input gap (sibling precedent); audience checkbox grid gap                                                    |
| `--space-3`                        | 12px  | Checkbox-to-label inline gap upper bound (use 8px default)                                                            |
| `--space-4` / `space-y-4`, `--gap` | 16px  | Between form fields (sibling precedent); `DialogContent` internal grid gap (`gap-[var(--gap)]`, already in primitive) |
| `--space-5` / `--pad`              | 20px  | `DialogContent` padding (`p-[var(--pad)]`, already in primitive)                                                      |
| `--space-6` / `mt-6`               | 24px  | Form-body-to-`DialogFooter` gap (sibling precedent)                                                                   |
| `--space-8`                        | 32px  | not used in this dialog                                                                                               |
| `--space-12`                       | 48px  | not used in this dialog                                                                                               |

Exceptions:

- `min-h-11` (44px) on all inputs, selects, textarea triggers irrelevant, and footer buttons — sibling precedent and project touch-target floor. The `Button` size variant already resolves to `min-h-11 sm:min-h-9`; siblings force `className="min-h-11"` — match them.
- Translate assist buttons use `Button` `size="sm"` (`min-h-11 sm:min-h-8`) — small on desktop, floor-compliant below `sm`.

## Typography

All sizes are what the existing primitives already render — declare them, do not restyle. Exactly two weights: 400 and 600.

| Role                                                         | Size                      | Weight                                        | Line Height                        | Source                                                                                 |
| ------------------------------------------------------------ | ------------------------- | --------------------------------------------- | ---------------------------------- | -------------------------------------------------------------------------------------- |
| Dialog title                                                 | 22px                      | 600                                           | tight (≈1.25, `leading-tight`)     | `DialogTitle` primitive (`drawer-title text-[22px]`)                                   |
| Dialog description / meta / validation messages              | 12px                      | 400                                           | 1.5                                | `DialogDescription` (`card-sub`); validation text `text-xs`                            |
| Body / input / textarea / select / checkbox labels / buttons | 13px (md+), 16px below md | 400 (600 inside `.btn-primary` per prototype) | 1.5 (`--line-h-body`)              | `Input`/`Textarea` primitives (`md:text-[13px]`), `--t-body`                           |
| Field labels                                                 | 11px                      | 600                                           | none (single line, `leading-none`) | `Label` primitive → `.label` (uppercase, `letter-spacing: 0.08em`, `var(--ink-faint)`) |

Rules:

- Arabic text renders in Tajawal automatically via the `html[dir="rtl"]` cascade; the `title_ar`/`content_ar` inputs must also render Tajawal in EN locale (see Bilingual & RTL Contract).
- No new font sizes, no `font-bold`, no Tailwind arbitrary type values beyond what the primitives ship.

## Color

All colors via `var(--*)` tokens or the `@theme`-mapped utilities (`bg-surface`, `text-ink`, `border-line`, `bg-accent`). **No raw hex. No Tailwind color literals.** Values below are the Bureau resolution for reference only — code references tokens.

| Role            | Value                                                                                    | Usage                                                                                                              |
| --------------- | ---------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| Dominant (60%)  | `--surface` (#ffffff) over `--bg` (#f7f6f4)                                              | Dialog surface (already in `DialogContent` primitive); input/select/checkbox backgrounds                           |
| Secondary (30%) | `--line` (#e8e4dc), `--line-soft` (#efece3), ink ramp `--ink`/`--ink-mute`/`--ink-faint` | 1px borders on dialog, inputs, outline buttons; ghost-button hover fill; label/description/placeholder text        |
| Accent (10%)    | `--accent` oklch(58% 0.14 32)                                                            | Reserved list below                                                                                                |
| Destructive     | `--danger` oklch(52% 0.18 25)                                                            | Validation only (no destructive actions this phase): `aria-invalid` field borders + inline validation message text |
| Warning         | `--warn` oklch(62% 0.14 75)                                                              | Partial-failure warning toast accent (sonner `toast.warning`) only                                                 |

Accent reserved for (explicit list — nothing else):

1. Primary submit button "Create position" (`Button` default variant → `.btn-primary`: `bg: var(--accent)`, text `var(--accent-fg)`)
2. Checked state of audience-group checkboxes (existing `Checkbox` primitive checked fill)
3. Focused field border + focus-visible outline (`focus-visible:border-[var(--accent)]` / `outline-[var(--accent)]` — already in primitives)
4. Text-selection highlight inside inputs (already in `Input` primitive)
5. The tab's primary "Create position" button (same `.btn-primary`)

NOT accent: translate buttons (ghost — `var(--ink)` text, `var(--line-soft)` hover), Cancel button (outline — `var(--surface)` bg, `var(--line)` border), labels, the `DossierContextBadge` (keeps its existing styling), the secondary "Attach existing position" button (outline).

Prohibited (project CLAUDE.md): card drop-shadows (dialog keeps its primitive `var(--shadow)` only), gradients, new button variants.

### Dialog layout (top to bottom, single column)

```
DialogContent  sm:max-w-lg  (sibling parity, D-01)
├─ DialogHeader
│   ├─ DialogTitle      [MessageSquare h-5 w-5] + title (gap-2)
│   └─ DialogDescription
├─ form  (body scrolls: max-h-[calc(85vh-10rem)] overflow-y-auto if needed at 1024px)
│   ├─ DossierContextBadge
│   ├─ Position type        Select, min-h-11, default = "Standard Position" by name-match (D-05)
│   ├─ Title (English) *    label row: Label(start) + ghost translate btn(end) │ Input dir="ltr"
│   ├─ Title (Arabic) *     label row: Label(start) + ghost translate btn(end) │ Input dir="rtl" font-arabic
│   ├─ Content (English)    label row + ghost translate btn │ Textarea rows=3 dir="ltr"
│   ├─ Content (Arabic)     label row + ghost translate btn │ Textarea rows=3 dir="rtl" font-arabic
│   └─ Audience groups *    fieldset/legend; grid grid-cols-1 sm:grid-cols-2 gap-2;
│                           each item = Checkbox + 13px label, whole row clickable (htmlFor);
│                           "All Staff" pre-checked by name-match (D-06)
└─ DialogFooter  mt-6
    ├─ Button outline  min-h-11  → Cancel
    └─ Button default  min-h-11  → Create position  (Loader2 me-2 while pending)
```

- Field stack: `space-y-4`; label-to-control: `space-y-2` (sibling rhythm).
- Label rows with translate buttons: `flex items-center justify-between` (logical — start/end, never left/right).
- Translate buttons: `Button variant="ghost" size="sm"`, content = `Languages` icon (h-4 w-4) + 12px text label, `gap-1`. Icon swaps to `Loader2 animate-spin` while in flight. Icon is non-directional — no RTL flip.
- The translate button lives on the SOURCE field and fills the COUNTERPART field (D-07): button on Title (English) is "Translate to Arabic" → fills `title_ar`.

### Positions tab header (D-13)

Header actions area (existing `flex … sm:justify-between` row) holds two buttons, primary closest to reading start:

- `Create position` — `Button` default (`.btn-primary`) → opens `NewPositionDialog`
- `Attach existing position` — `Button variant="outline"` → opens existing `AttachPositionDialog`

Decision (discretion area, recorded): side-by-side buttons, not a dropdown menu — two actions don't justify a menu, and the mislabel bug came from hiding the distinction.

## Interaction States

| #   | State                           | Contract                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| --- | ------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Lookups loading                 | Select trigger and checkbox group render disabled; submit disabled. No skeletons — lookups are two tiny cached queries (`staleTime` 30 min)                                                                                                                                                                                                                                                                                                                                                                                |
| 2   | Lookup failure                  | Inline error message (12px, `var(--danger)`) replaces the affected control group; submit stays disabled. Copy per Copywriting Contract                                                                                                                                                                                                                                                                                                                                                                                     |
| 3   | Validation (D-04)               | RHF + zodResolver, `mode: 'onTouched'`. Inline message under the field (12px, `var(--danger)`), `aria-invalid` border (`var(--danger)`, already in primitives), message rendered via `t(errors.field.message)`. Submit disabled until `formState.isValid`                                                                                                                                                                                                                                                                  |
| 4   | Translate in-flight (D-07/D-08) | The clicked button: `Loader2` spinner + disabled; its paired button also disabled (no concurrent EN↔AR race). Disable-while-in-flight IS the debounce (decision recorded). Source and target fields stay editable. `aria-live="polite"` announces "Translating…"                                                                                                                                                                                                                                                           |
| 5   | Translate success               | Counterpart field filled via `setValue(…, { shouldValidate: true, shouldDirty: true })` — an editable draft, focus NOT stolen. Confidence score: omitted (decision recorded — keeps quick-create quick; deployed edge currently 503s anyway)                                                                                                                                                                                                                                                                               |
| 6   | Translate failure               | Small error toast (copy above); target field untouched; submission never depends on AI (D-07). Never paste placeholder text like `[ترجمة مطلوبة]` into fields (RESEARCH Pitfall 4)                                                                                                                                                                                                                                                                                                                                         |
| 7   | Submit in-flight                | Submit button: `Loader2 me-2 animate-spin` + disabled (sibling parity — other controls stay enabled)                                                                                                                                                                                                                                                                                                                                                                                                                       |
| 8   | Full success (D-12)             | Dialog closes, form resets, `toast.success` with "Open position" action → navigate `/positions/$id`. Invalidate: `['dossier-position-links', dossierId]` (prefix), `dossierOverviewKeys.detail(dossierId)`, `['positions','list']`, `['position-dossier-links', position.id]`                                                                                                                                                                                                                                              |
| 9   | Partial failure (D-11)          | Position created, link write threw: dialog still closes (re-submit would duplicate the position), `toast.warning` (duration 10 000 ms) with "Retry link" action that re-calls `createPositionDossierLink(position.id, { dossier_id, link_type: 'applies_to' })`; on retry success show the full-success toast + dossier-scoped invalidations; on retry failure re-show the same warning. Invalidate only `['positions','list']` at this point. Warning ≠ success — never render clean success (Phase 62 honesty precedent) |
| 10  | Total failure                   | Dialog stays open with typed input intact; `toast.error` with generic localized copy (the shared api-client strips the edge's bilingual body — RESEARCH Pitfall 3; do not change `api-client`)                                                                                                                                                                                                                                                                                                                             |
| 11  | Hover/focus/active              | Inherited from primitives: `.btn-primary` hover darkens via `color-mix`, ghost hover fills `var(--line-soft)`, focus-visible 2px `var(--accent)` outline at 2px offset. Do not add custom hover styles                                                                                                                                                                                                                                                                                                                     |

## Bilingual & RTL Contract

- Dialog `dir` follows the app (`i18n.language === 'ar'` → RTL); Tajawal cascades automatically.
- **Per-field direction is content-driven, not locale-driven:** `title_en`/`content_en` inputs always `dir="ltr"`; `title_ar`/`content_ar` inputs always `dir="rtl"` with Tajawal — in BOTH UI locales.
- Logical properties only: `ms-*`, `me-*`, `ps-*`, `pe-*`, `text-start`, `justify-between` flex order. Never `ml/mr/pl/pr/text-left/text-right`.
- `Loader2` uses `me-2` inside the submit button (sibling precedent).
- Lookup labels (`name_en`/`name_ar`) and toast copy picked by locale; validation messages localize via Zod-message-as-i18n-key.
- Field order does not change between locales (RTL mirroring is handled by `dir`, never by reordering JSX).
- Verify live in BOTH locales at 1280px and 1024px (desktop-primary; below 768 is read-only mobile and out of scope for this creation flow).
