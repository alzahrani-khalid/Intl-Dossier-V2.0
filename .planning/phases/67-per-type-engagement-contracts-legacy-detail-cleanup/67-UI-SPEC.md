# Phase 67 — UI Design Contract

> Visual and interaction contract for PERENG-01 (org hosted-engagements section), PERENG-02 (person/EO participation section), and PERENG-03 (legacy detail deletion). Generated by gsd-ui-researcher in AUTO mode — every decision derived from the IntelDossier design system, 67-RESEARCH/67-PATTERNS, and the shipped 65/66 contracts; assumptions logged below.

**Scope:** ONE net-new visual primitive — the **per-type engagement section** (uppercase-tracked heading + bordered `var(--row-h)` row list), instantiated twice inside the shipped `DossierEngagementsTab` ("Hosted engagements" on org, "Participation" on person/EO). Everything else is composition: the Phase 66 error line reused verbatim, the tab's shipped timeline/empty/badge forms untouched, and an all-deletion requirement (PERENG-03) that needs no visual design. Zero new packages, zero new colors, zero new type values, zero new button variants.

## 1. Per-Type Engagement Section (the one net-new primitive)

Defined once, instantiated twice (§2, §3). Renders INSIDE `DossierEngagementsTab`, between the tab's existing h3 header and the existing generic timeline.

### Placement (tab structure after this phase)

```
Tab root  (p-4 sm:p-6, dir={isRTL ? 'rtl' : 'ltr'} — existing)
├─ Existing tab header: History icon + h3 "Engagements"        (unchanged)
├─ NEW (conditional): per-type section                          ← org: Hosted engagements
│   ├─ section heading                                            person/EO: Participation
│   └─ bordered row list  OR  66 error line
├─ Existing generic timeline (related dossiers + past events)   (markup unchanged;
│   └─ gains a "History" sub-heading ONLY when a per-type         gated isError branch added)
│      section rendered above it — see §4
```

### Anatomy

| Element           | Contract                                                                                                                                                                                                                                                                                                                                                                                                     |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Section heading   | `h3` exact sibling style (65 §3 / workspace `EventSection`): `text-xs font-semibold text-muted-foreground uppercase tracking-wider`, `space-y-2` to content. **No count badge** (assumption A-4)                                                                                                                                                                                                             |
| List container    | `rounded-md border` (resolves 1px `var(--line)`) + `divide-y` row dividers. Flat surface — no shadows, no gradients                                                                                                                                                                                                                                                                                          |
| Row               | `flex items-center justify-between gap-3 px-3 min-h-[var(--row-h)]` — density-aware row height (non-negotiable list contract)                                                                                                                                                                                                                                                                                |
| Row start         | Engagement name as `<Link to={getDossierDetailPath(id, 'engagement')}>` — `text-sm font-medium truncate min-w-0 text-start hover:underline`. Locale-picked: AR → `name_ar ?? name_en`; EN → `name_en`. IDs are canonical `engagement_dossiers` ids (= `dossiers.id`, RESEARCH), and the shipped `/dossiers/engagements/$id` redirect lands on `/engagements/$id/overview` — links are always live, never 404 |
| Row end           | Date `text-xs text-muted-foreground` via `formatDayFirst(date, i18n.language)` (day-first `Tue 28 Apr`; AR digits handled by the lib) + ONE badge pill in the tab's shipped form: `bg-muted text-muted-foreground text-xs px-2 py-0.5 rounded-full`. Badge copy per instance (§2/§3), always colon-form i18n with `defaultValue` slug fallback                                                               |
| Date source       | `engagement_dossiers.start_date`; fallback `dossiers.created_at` if null                                                                                                                                                                                                                                                                                                                                     |
| Sort              | `start_date` descending (newest-first, matching the tab); never `.reverse()`                                                                                                                                                                                                                                                                                                                                 |
| Names lookup      | Two-step `.in()` merge on `dossiers` (RESEARCH Pattern 2) — never an embed on the extension table (nameless rows)                                                                                                                                                                                                                                                                                            |
| **Render rule**   | Query success + 0 rows → **section NOT rendered at all** — no heading, no empty line, no placeholder (WR-04 lesson, 64-REVIEW: never a dead-end surface with no content). Query error → section **MUST render**: heading + error line (failure must never impersonate absence — Phase 66 OVRERR-01)                                                                                                          |
| Error line        | Phase 66 §1 anatomy VERBATIM: `<p role="alert" className="text-sm text-[var(--danger)] text-center py-8">` with `t('dossier:overview.sectionError')` (colon-form; key shipped EN+AR in Phase 66 — reuse, do not duplicate). No icon, no border tint, no retry button (66 A-3)                                                                                                                                |
| Loading           | Covered by the tab's existing `TabSkeleton type="list"` until branch queries settle — no per-section spinners or skeletons                                                                                                                                                                                                                                                                                   |
| Stale-while-error | Cached rows + background refetch failure → render the rows, NO error line (66 rule, mandatory)                                                                                                                                                                                                                                                                                                               |
| Hover/affordance  | Name link `hover:underline` only. Rows are otherwise non-interactive — no row click target, no hover fill, no chevron (65 state #10)                                                                                                                                                                                                                                                                         |

## Bilingual & RTL Contract

- The sections inherit the tab's existing `dir={isRTL ? 'rtl' : 'ltr'}` root — they add NO `dir` of their own.
- Logical properties only: `text-start`, `min-w-0` truncation, `justify-between` rows (start/end flip free under `dir`), `divide-y` is direction-neutral. Never `ml/mr/pl/pr/text-left/text-right`.
- JSX order inside the row-end meta cluster: date first, badge second — natural `gap-2` flow flips correctly under RTL; no `.reverse()`, no manual ordering hacks.
- Engagement names locale-picked (`name_ar ?? name_en` in AR); role/type badges via bilingual keys; dates via `formatDayFirst` (Arabic month/day names + correct digits).
- No directional icons in scope (no chevrons on rows).
- All three new keys land EN+AR in one commit with a parity assertion; colon-form lookups (`dossier:`, `engagements:`) — dot-form leaks raw keys (project memory).
- Verify live in BOTH locales at 1280px and 1024px with seeded rows (orchestrator protocol) AND a forced branch failure (CDP `Network.setBlockedURLs`) — Arabic error copy, `dir=rtl`, no overflow, badges on the visual left of row content.
