## Rollback

- **If push notifications don't deliver:** check VAPID keys exist in backend env, confirm HTTPS (HTTP will silently fail), verify service worker registered. Chrome DevTools → Application → Push → check subscription.
- **If preference toggle appears to not respect:** check `notification_category_preferences` row exists for the user. Default may be missing — INSERT a default row via SQL.
- **If deliberately-failing run wipes out `main`:** Don't commit the failure to `main`. Use a throwaway branch. If accidentally pushed to `main`, revert the commit.
- **If branch protection locks you out:** disable the rule temporarily via Settings → Branches; root cause the missing required check name.
