# Phase 66 — UI Design Contract

> Visual and interaction contract for OVRERR-01 (overview section error states) and OVRERR-02 (timeline View-details suppression). Generated by gsd-ui-researcher in AUTO mode — every decision derived from the IntelDossier design system, 66-RESEARCH/66-PATTERNS, and the shipped phase-65 error-state precedent; assumptions logged below.

**Scope:** ONE net-new visual primitive — the **section error line** — applied uniformly across the ~19 overview cards and the DossierDrawer, plus the ABSENCE of an affordance (timeline View-details suppression). Card chrome, grids, headings, skeletons, and all genuine empty states are shipped contracts — **do not redesign any card**. Zero new components, zero new colors, zero new type values.

## 1. Section Error Line (the one net-new primitive — OVRERR-01)

Applied identically to every card in `frontend/src/pages/dossiers/overview-cards/` (19 files) regardless of hook family (`useDossierOverview`, `useDossierActivityTimeline`, `useDossierPositionLinks`, `MoUStatusCard`'s inline `useQuery`, `useElectedOfficial`).

### Anatomy

```tsx
<p role="alert" className="text-sm text-[var(--danger)] text-center py-8">
  {t('overview.sectionError', {
    defaultValue: 'Failed to load this section. Check your connection and try again.',
  })}
</p>
```

| Element         | Contract                                                                                                                                                                                                                        |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Element/role    | Single `<p role="alert">` — announces to screen readers; no live-region wrapper needed (renders on state change)                                                                                                                |
| Placement       | INSIDE the card body, occupying the EXACT slot the card's empty-state line uses today (`text-sm text-center py-8` — verified shipped slot). Card chrome — container, `1px var(--line)` border, radius, h3 title — **unchanged** |
| Color           | `text-[var(--danger)]` ONLY. This is the phase-65/66 canonical danger form; `text-destructive` is the legacy `@theme` alias — do not introduce it in new lines. Never accent, never muted, never raw hex                        |
| Type            | 14px / 400 / 1.5 (`text-sm`) — same as the empty line it replaces. No icon, no border, no background fill, no button                                                                                                            |
| Copy            | `dossier:overview.sectionError` (colon-form namespace; see Copywriting Contract). One key for all 19 cards — per-card error copy is noise, the card title already names the section                                             |
| Differentiation | Error vs empty differ in copy AND color AND `role="alert"` — never color alone (a11y)                                                                                                                                           |

### State precedence (uniform across all cards — error BEFORE empty)

```
1. isLoading && no data        → existing skeleton (unchanged)
2. isError && data unavailable → section error line          ← NEW branch, checked BEFORE empty
3. data empty                  → existing empty line (unchanged copy: noData / noRecentActivity / *.empty)
4. data                        → existing content (unchanged)
```

"Data unavailable" per hook: `data === null` for `useDossierOverview` consumers (the hook returns `query.data || null` — verified), `data === undefined` for raw `useQuery`/domain hooks. The plan names the exact comparison per hook family.

### Stale-while-error rule (mandatory, uniform)

If cached data exists (`data` defined/non-null) and a background refetch fails, **render the data and NO error line**. Why: TanStack Query v5 deliberately retains last-good data on refetch failure; blanking trustworthy cached content over a transient network blip is data loss, and the phase's goal is "failure must never impersonate emptiness" — not "failure must always interrupt". A stale-but-real view is honest; a fake-zero view is not.

### Blast radius (expected, not a bug)

Cards sharing an `includeSections` variant share one query — a `related_dossiers` failure errors every card whose variant includes it (e.g. SummaryStats + Bilateral + EngagementsByStage on country). This is correct: those cards genuinely depend on the failed data. The verification matrix pre-records this so it isn't misread as collateral damage.

### In-passing raw-message fixes (security, Pitfall 9)

`DossierOverview.tsx` (full-page error, ~L234) and `DossierActivityTimeline.tsx` (~L176) currently render raw `error?.message` (PostgREST column/table names, English-only). Both swap to generic localized copy — `dossier-overview:error.description` (existing key) and `dossier:overview.sectionError` respectively; raw detail goes to `console.error` only. No layout changes.

## 2. DossierDrawer Error Branch (replaces the permanent skeleton)

`DossierDrawer.tsx` keys its body on `overviewLoading || !overview` — after fail-the-query lands, a failed overview holds the drawer in skeleton **forever**. New branch, using the drawer's EXISTING dormant bilingual keys (`dossier-drawer:error.*` — verified present in en+ar, currently unconsumed).

| Element    | Contract                                                                                                                                                                                             |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Trigger    | Overview query `isError` AND `overview === null` → render error branch instead of `DrawerSkeleton`; `data-loading="false"`                                                                           |
| Container  | `<div role="alert" className="px-4 py-16 text-center space-y-2">` inside `.drawer-body` (phase-65 `TasksTab` full-surface shape)                                                                     |
| Heading    | `t('error.load_failed_heading')` — "Could not load this dossier" / "تعذّر تحميل هذا الملف" — `text-sm font-semibold text-[var(--danger)]`                                                            |
| Body       | `t('error.load_failed_body')` — "Try refreshing the page or open the full dossier." / "حاول تحديث الصفحة أو افتح الملف الكامل." — `text-sm text-muted-foreground`                                    |
| Retry      | `t('error.retry')` — `Button variant="outline" size="sm" className="min-h-11"` → refetch the overview query. The only interactive element in the branch                                              |
| Stale rule | Cached overview present + refetch failure → render sections normally (same stale-while-error rule as cards)                                                                                          |
| DrawerHead | Must never strand in its own skeleton: on dossier-core query error the head keeps its chrome (type chip, functional close button) with title fallback `—`; the body error branch carries the message |

No shadows/gradients added; the drawer's existing `var(--shadow-drawer)` chrome is untouched.

## 3. Timeline "View details" Suppression (OVRERR-02)

The affordance is **ABSENT**, not disabled. SUPPRESS disposition recorded for calendar rows (orchestrator product call: `/calendar` has no event-focus param — a bare list link is a dishonest affordance).

| Surface                                             | Contract                                                                                                                                                                                                                                                                                                     |
| --------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `TimelineEventCard`, `EnhancedVerticalTimelineCard` | "View details" button AND its click handler render only when `resolveTimelineNavUrl(navigation_url) !== null`. When null: **no disabled button, no dead chevron, no `cursor-pointer`, no `role`/`tabIndex`** — the action region is simply not in the DOM. Row content (icon, title, date, badges) unchanged |
| Calendar timeline rows                              | Edge emits `navigation_url: null` → rows render with no affordance at all (SUPPRESS). Live-reproducible on staging (6 calendar_entries)                                                                                                                                                                      |
| MoU rows                                            | Edge emits `/mous` (mounted list) → affordance stays, navigates to the list                                                                                                                                                                                                                                  |
| Interaction/intelligence rows                       | Edge emits bare mounted dossier path (dead `?tab=` params dropped) → affordance stays                                                                                                                                                                                                                        |
| `ActivityList` (`/activity`)                        | Existing R-05 non-interactive-row shape extended with the mounted-route allowlist: guard fails → row has no `role`, no `tabIndex`, no `onClick`, no hover affordance (existing pattern, no visual change)                                                                                                    |
| i18n                                                | **Zero new keys.** `timeline.view_details` etc. unchanged — suppression removes the element, it never relabels it                                                                                                                                                                                            |
| Layout after removal                                | No placeholder, no empty actions container — sibling flex content reflows via existing `gap` (phase-65 CTA-removal rule)                                                                                                                                                                                     |

Voice: sentence case, no emoji, no marketing voice, no exclamation marks. Problem + next step. Every new key lands in BOTH `i18n/en/dossier.json` and `i18n/ar/dossier.json` in the same commit (static bundle — missing AR keys silently anglicize). Colon-form cross-namespace lookups only.

## Bilingual & RTL Contract

- The error line inherits the card's existing `dir={isRTL ? 'rtl' : 'ltr'}` container — it adds NO `dir` of its own. `text-center` is flip-safe; any start-aligned variants use `text-start` only.
- Logical properties only (`ms-*`, `ps-*`, `text-start`); never `ml/mr/pl/pr/text-left/text-right`. The error line and drawer branch contain no directional layout (centered text, no icons), so nothing flips.
- All copy via i18n keys — `overview.sectionError` added to BOTH `en/dossier.json` and `ar/dossier.json` in the same edit; key-parity assertion in tests (phase-65 precedent).
- Drawer keys already bilingual (verified); Tajawal cascades automatically in AR.
- No `.reverse()`, no directional icons in scope.
- Verify live in BOTH locales at 1280px and 1024px with a forced section failure (CDP `Network.setBlockedURLs`) — error copy in Arabic, `dir=rtl`, no overflow.
