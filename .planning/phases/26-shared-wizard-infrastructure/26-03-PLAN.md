Create `CreateWizardShell.tsx` as a thin compositional wrapper around `FormWizard` per D-05. It adds FormProvider + Form wrapper and passes wizard state as FormWizard props. It does NOT re-implement step indicators, animations, or navigation.

**Props interface:**

```typescript
import type { FieldValues } from 'react-hook-form'
import type { CreateWizardReturn } from './config/types'

interface CreateWizardShellProps<T extends FieldValues> {
  wizard: CreateWizardReturn<T>
  children: React.ReactNode
  className?: string
}
```

**Component implementation:**

```typescript
import type { ReactElement, ReactNode } from 'react'
import { useTranslation } from 'react-i18next'
import type { FieldValues } from 'react-hook-form'
import { FormProvider } from 'react-hook-form'
import { Save } from 'lucide-react'

import { FormWizard } from '@/components/ui/form-wizard'
import { Form } from '@/components/ui/form'
import { cn } from '@/lib/utils'
import type { CreateWizardReturn } from './config/types'

export function CreateWizardShell<T extends FieldValues>({
  wizard,
  children,
  className,
}: CreateWizardShellProps<T>): ReactElement {
  const { t } = useTranslation(['dossier', 'form-wizard'])

  return (
    <div className={cn('w-full max-w-2xl mx-auto px-4 sm:px-6 lg:px-8', className)}>
      {/* Draft saved indicator -- top-end corner */}
      {wizard.isDraftSaving && (
        <div className="flex items-center justify-end gap-2 mb-2 text-sm text-muted-foreground animate-in fade-in duration-250">
          <Save className="h-3.5 w-3.5" />
          <span>{t('form-wizard:draftSaved')}</span>
        </div>
      )}

      <FormProvider {...wizard.form}>
        <Form {...wizard.form}>
          <form onSubmit={(e: React.FormEvent): void => e.preventDefault()}>
            <FormWizard
              steps={wizard.steps}
              currentStep={wizard.currentStep}
              onStepChange={wizard.setCurrentStep}
              onComplete={wizard.handleComplete}
              onCancel={wizard.handleCancel}
              onSaveDraft={wizard.handleSaveDraft}
              isLoading={wizard.isSubmitting}
              isDraftSaving={wizard.isDraftSaving}
              hasDraft={wizard.hasDraft}
              canComplete={wizard.canComplete}
              completeButtonText={t('dossier:form.create')}
              allowStepNavigation={true}
              namespace="form-wizard"
              actionBarMode="auto"
            >
              {children}
            </FormWizard>
          </form>
        </Form>
      </FormProvider>
    </div>
  )
}
```

Key points per UI-SPEC:

- Container: `max-w-2xl mx-auto px-4 sm:px-6 lg:px-8` (centered, responsive padding)
- Draft indicator: `Save` icon + text at top-end, fades in/out with 250ms transition
- Action bar: `actionBarMode: 'auto'` (sticky on mobile, inline on desktop)
- Step navigation: `allowStepNavigation: true` (click completed steps)
- No new animations beyond FormWizard's existing Framer Motion transitions
- Bilingual: direction from FormWizard's built-in RTL support (uses `useDirection()` internally)
- `FormProvider` wraps everything so nested step components can use `useFormContext()`
- `Form` component from shadcn provides form context for `FormField` components
- Contextual help hook point: D-06 defers panel to Phase 31. No help panel rendered.
- Explicit return type `: ReactElement` on the component function

The Shell does NOT:

- Re-implement step indicator UI (FormWizard handles this)
- Re-implement step animations (FormWizard handles this)
- Re-implement action bar buttons (FormWizard handles this)
- Manage form state (hook handles this)
