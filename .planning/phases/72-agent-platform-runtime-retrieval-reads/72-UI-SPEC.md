# Phase 72 — UI Design Contract

> Visual and interaction contract for the bilingual, reads-only, app-wide **copilot** surface. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Source of visual truth:** the IntelDossier prototype (`frontend/design-system/inteldossier_handoff_design/`). Bureau direction (default, light). Every value below is a design-system token; the contract rule is **reference the token, never the literal**.
>
> **Scope guard:** READS-ONLY. Writes, HITL `renderAndWaitForResponse` confirmation cards, and rich generative-UI dossier cards are **Phase 73** — they do NOT appear in this contract. `generate_digest` is **preview-only** here.

**CopilotKit theming contract (D-09 — the conversational shell MUST encode this):** the CopilotKit chat surface is rendered token-only — every `--copilot-kit-*` variable is remapped to an IntelDossier token (background → `--surface`, foreground → `--ink`, primary/accent → `--accent`, border → `--line`, separators → `--line-soft`, muted text → `--ink-mute`), `dir="rtl"` + Tajawal in Arabic, **no raw hex, no card shadows on message/citation surfaces**, radii from `--radius-sm/--radius/--radius-lg`. The Option-C spike must prove **(1) RTL + token fidelity** to the CLAUDE.md bar AND **(2) full air-gap** (self-hosted runtime, zero egress, no Copilot Cloud key) before CopilotKit locks; documented fallback = headless `assistant-ui` / custom `@ag-ui/client` shell (runtime/hooks/HITL stay either way).

## Spacing Scale

Declared values (8-point scale; the only values allowed in the main scale):

| Token | Value | Usage                                                                       |
| ----- | ----- | --------------------------------------------------------------------------- |
| xs    | 4px   | Icon-to-label gaps, inline chip padding (`--space-1`)                       |
| sm    | 8px   | Compact element spacing, chip-row gap, message-meta gaps (`--space-2`)      |
| md    | 16px  | Default element spacing, message-stream vertical gap, `--gap` (`--space-4`) |
| lg    | 24px  | Section padding inside the drawer body (`--space-6`)                        |
| xl    | 32px  | Layout gaps, composer-to-thread separation (`--space-8`)                    |
| 2xl   | 48px  | Major section breaks, empty-state block (`--space-12`)                      |
| 3xl   | 64px  | Page-level / full-height empty-state vertical centering                     |

**Exceptions** (first-class density-aware design-system tokens from `colors_and_type.css`, NOT ad-hoc values — each justified):

- **12px = `--space-3`** — compact chip / inline gaps in message metadata (timestamp · role · source-count), and citation-chip internal padding. Canonical IntelDossier inline-metadata step.
- **20px = `--pad`** — density-aware card/drawer body padding. The drawer body, composer padding, and citation-card padding all bind to `--pad` (comfortable density = 20px) so they re-flow with the density tweak panel.
- **40px = `--space-10`** — empty-state vertical rhythm (gap between the hero prompt, body copy, and suggested-prompt chips in the fresh-thread empty state).
- **52px = `--row-h`** — density-aware row height for ThreadList rows (resume-past-conversation list). Tables and lists obey `--row-h` per CLAUDE.md; never hard-coded.

## Typography

Exactly **4 sizes**, exactly **2 weights** {400 regular, 600 semibold}. Sizes map 1:1 to existing IntelDossier semantic tokens.

| Role    | Token            | Size | Weight | Line Height            | Usage                                                                                       |
| ------- | ---------------- | ---- | ------ | ---------------------- | ------------------------------------------------------------------------------------------- |
| Display | `--t-page-title` | 28px | 600    | 1.1 (`--line-h-tight`) | Copilot empty-state hero prompt ("Ask about this dossier" / dossier-less "Ask the copilot") |
| Heading | `--t-card-title` | 16px | 600    | 1.35 (`--line-h-snug`) | Drawer header title, message-group titles, citation-card titles                             |
| Body    | `--t-body`       | 13px | 400    | 1.5 (`--line-h-body`)  | Message text, composer textarea, button labels, suggested-prompt chips                      |
| Meta    | `--t-meta`       | 12px | 400    | 1.5                    | Timestamps, citation source lines, role labels (assistant/you), thread-list subtitles       |

**Inherited design-system chrome (NOT new type roles introduced by this phase):** the prototype defines 8 semantic sizes; this phase introduces only the 4 above. The following are pre-existing chrome the copilot reuses verbatim, not new roles:

- `--t-mono-small` 11px / `--t-mono-tiny` 10px (JetBrains Mono) — citation IDs, `T-3`/`T+2` SLA tokens, `kbd` (the Cmd+K hint badge). Mono labels are tabular chrome, not prose.
- `--t-label` 10.5px UPPERCASE (`--tracking-label`) — classification ribbons / chip labels (DOSSIER, CONFIDENTIAL).
- `--t-kpi-value` 36px — KPI numerals; not used in the copilot reads surface, listed only to account for the full system scale.

> Rationale for the 13/12 body/meta pair: this is the **canonical IntelDossier pairing** straight from `colors_and_type.css` (`--t-body: 13px` / `--t-meta: 12px`), intentional and density-tuned for the analyst workstation — not arbitrary "too-close" sizes.

## Color

60/30/10 split. Each row gives the **token** and its resolved Bureau value for reference only — the enforced rule is "reference the token, never the literal." No raw hex, no Tailwind color literals (`text-blue-500`). No gradient backgrounds; flat surfaces. Borders `1px solid var(--line)`; **no drop-shadows on cards** — shadow is reserved for the **drawer** (`--shadow-lg`) and hovered list rows only (correct here: the copilot IS a drawer).

| Role            | Token                                                     | Resolved (Bureau)                                  | Usage                                                                                                                                                             |
| --------------- | --------------------------------------------------------- | -------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Dominant (60%)  | `--bg` / `--surface`                                      | `#f7f6f4` / `#ffffff`                              | Canvas behind the message stream; `--surface` for the drawer body and assistant message rows (neutral, never accent)                                              |
| Secondary (30%) | `--surface-raised`, `--sidebar-bg`, citation-card surface | `#ffffff` (drawers/popovers), `#ffffff` (nav rail) | The drawer/sheet chrome (`--surface-raised`), the ThreadList panel, and CitationCard surfaces — each a flat `--surface` panel with a `1px solid var(--line)` edge |
| Accent (10%)    | `--accent` (`--accent-fg` for text-on-accent)             | `oklch(58% 0.14 32)` terracotta                    | See reserved-for list below — NOTHING else                                                                                                                        |
| Destructive     | `--danger`                                                | `oklch(52% 0.18 25)`                               | "Delete conversation" confirmation only                                                                                                                           |

**Accent reserved for** (tight list — never "all interactive elements"):

1. The primary **"Ask the copilot"** drawer-entry button and the composer **Send** button (`.btn-primary` resolves its fill from `--accent`).
2. The **active / selected citation chip** (selected-source highlight).
3. The **focus ring** on the composer and on focused message/citation/thread rows.
4. The **streaming caret / "Thinking…" indicator** accent (token-bound streaming state).

Everything else stays neutral: assistant and user message rows are `--surface` with `--ink` text; secondary actions use `.btn-ghost`; dividers use `--line-soft`. The thread-list unread/active dot reuses the existing `bg-[var(--accent)] text-[var(--accent-fg)]` badge idiom from `RelationshipSidebar`.

## Copywriting Contract

Sentence case for all titles and buttons. UPPERCASE only for classification ribbons + mono labels. **No emoji. No marketing voice** (banned: Discover, Easily, Unleash, exclamation marks, "you're in!", first-person "we"). Every label below is specific (verb+noun) and bilingual.

| Element                                                               | Copy (EN)                                                                                                                                     | Copy (AR)                                                                                                                        |
| --------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| Primary CTA / drawer entry                                            | Ask the copilot                                                                                                                               | اسأل المساعد                                                                                                                     |
| Cmd+K entry row (on a dossier)                                        | Ask the copilot about this dossier                                                                                                            | اسأل المساعد عن هذا الملف                                                                                                        |
| Composer send action (icon button)                                    | aria-label: **Send message** (visible: paper-plane glyph; the aria-label is the accessible name)                                              | إرسال الرسالة                                                                                                                    |
| Stop streaming control                                                | Stop                                                                                                                                          | إيقاف                                                                                                                            |
| Empty state — heading (dossier context)                               | Ask about this dossier                                                                                                                        | اسأل عن هذا الملف                                                                                                                |
| Empty state — heading (no dossier)                                    | Ask the copilot                                                                                                                               | اسأل المساعد                                                                                                                     |
| Empty state — body                                                    | Start with a question about signals, digests, the network graph, or linked records. The copilot answers only from data you're cleared to see. | ابدأ بسؤال عن الإشارات أو الموجزات أو شبكة العلاقات أو السجلات المرتبطة. يجيب المساعد فقط من البيانات المصرّح لك بالاطلاع عليها. |
| No-answer / no-citation result (neutral, **indistinguishable-empty**) | The copilot couldn't find anything to answer that. Try rephrasing, or ask about a specific dossier.                                           | لم يجد المساعد ما يجيب عن ذلك. حاول إعادة الصياغة، أو اسأل عن ملف محدد.                                                          |
| Error state — heading                                                 | The copilot is unavailable                                                                                                                    | المساعد غير متاح                                                                                                                 |
| Error state — body                                                    | The on-prem assistant didn't respond. Retry your question; if it keeps failing, the agent-runtime service may be down.                        | لم يستجب المساعد المحلي. أعد إرسال سؤالك؛ وإذا استمر الفشل فقد تكون خدمة وحدة التشغيل متوقفة.                                    |
| Error state — action                                                  | Retry                                                                                                                                         | إعادة المحاولة                                                                                                                   |
| Streaming / loading indicator                                         | Thinking…                                                                                                                                     | جارٍ التفكير…                                                                                                                    |
| Destructive — "Delete conversation"                                   | Delete conversation                                                                                                                           | حذف المحادثة                                                                                                                     |
| Destructive confirmation dialog                                       | Delete conversation — this removes the thread and its history. This can't be undone.                                                          | حذف المحادثة — يؤدي ذلك إلى إزالة المحادثة وسجلّها. لا يمكن التراجع عن هذا الإجراء.                                              |

**INDISTINGUISHABLE-EMPTY (hard constraint, carried lock from P68/P71 memory):** the no-answer result and any reduced-result state MUST NEVER reveal clearance filtering. The substring `clearance` / `filtered` / `restricted` MUST NOT appear ANYWHERE in any copilot-rendered payload — including visible copy, aria-live announcements, AND JSON keys/values (e.g. no `stats.clearance_level`). A lower-clearance caller must not be able to learn that above-clearance content exists. The neutral copy above ("couldn't find anything to answer that") is the **only** empty messaging — it is NOT the banned generic "No results".

**Destructive confirmation approach:** "Delete conversation" opens a token-bound confirmation dialog (HeroUI/Radix Dialog re-skinned to tokens) with a `.btn-danger` confirm (`--danger`) and a `.btn-ghost` cancel; commit only on explicit confirm. **If the planner ships thread management as archive-only (no hard delete), declare destructive = none and drop this dialog** — the contract requires one or the other, not an unconfirmed delete.

## Visual Hierarchy & Interaction (Dimension 2)

- **Focal point:** the conversation thread (message stream) — the largest, highest-contrast region, occupying the drawer body.
- **Composer:** pinned at the bottom edge (logical block-end), always reachable; the only accent-fill control in the resting state (Send).
- **Secondary:** citation cards directly under assistant replies — flat `--surface` + `--line`, clearly subordinate to the message text.
- **Tertiary:** the drawer header (title + close) — minimal chrome, mirrors the dossier drawer head.

**Icon-only controls — every one declares an aria-label fallback (Dimension 2 requirement):**

| Control                       | aria-label (EN)     | aria-label (AR) |
| ----------------------------- | ------------------- | --------------- |
| Send message                  | Send message        | إرسال الرسالة   |
| Close drawer                  | Close               | إغلاق           |
| Cmd+K / FAB trigger           | Ask the copilot     | اسأل المساعد    |
| Stop stream                   | Stop                | إيقاف           |
| Copy message                  | Copy message        | نسخ الرسالة     |
| Delete conversation (per row) | Delete conversation | حذف المحادثة    |

**RTL reading order (JSX = Arabic reading order; logical properties only — `ms-*`/`me-*`/`ps-*`/`pe-*`/`text-start`/`text-end`, never `ml/mr/pl/pr/left/right/text-left/text-right`):**

- **Header row** (`flex items-center justify-between`, mirrors `DrawerHead`): JSX child 1 = title / chip-row → renders at the **inline-start / right in RTL** (reading start); JSX child 2 = close button → renders at the **inline-end / left in RTL**. In Arabic the analyst reads the title first (rightmost), close sits on the far left.
- **Message row:** role label + timestamp lead at the inline-start (right in RTL); the per-message copy action sits at the inline-end (left in RTL). Assistant vs user differentiation is by alignment to the start/end edge using logical properties, not physical left/right.
- **Composer row:** textarea fills the inline span; Send sits at the inline-end (left in RTL, right in LTR); Stop (while streaming) adjacent to Send.
- `dir="rtl"` set on the drawer container in Arabic; Tajawal applies via the `html[dir="rtl"]` rule. Directional lucide glyphs (e.g. a send arrow) flip with `rotate-180` under RTL. Verify at 1024px and 1400px (analyst-workstation widths); mobile ≤767 is the read-only sheet (fits this reads-only phase).
