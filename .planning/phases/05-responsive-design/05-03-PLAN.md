Per D-06 priority 1 and RESP-01/RESP-02: Dashboard must work at 320px, 640px, 768px, 1024px, 1280px+.

1. Audit all dashboard route and component files. For EACH file:

2. Apply responsive grid patterns per UI-SPEC:
   - Metric/stat cards: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4`
   - Chart containers: `w-full` with responsive padding `p-3 sm:p-4 lg:p-6`
   - Page container: `container mx-auto px-4 sm:px-6 lg:px-8`
   - Page title: `text-2xl lg:text-3xl font-semibold text-start mb-4 lg:mb-6`

3. Touch targets (RESP-02):
   - All buttons: `min-h-11 min-w-11` (44px)
   - All links/anchors that are standalone (not inline text): `min-h-11 inline-flex items-center`
   - Form inputs: `min-h-11`
   - Checkboxes/radios: expand hit area with padding to reach 44x44px
   - Gap between adjacent interactive elements: `gap-2` minimum (8px)

4. Typography per UI-SPEC:
   - Section headers: `text-sm lg:text-base font-semibold` (step down on mobile)
   - Page title: `text-lg sm:text-xl lg:text-2xl`

5. Hide non-essential information on mobile (progressive disclosure):
   - Secondary stats, detailed charts, or less-used widgets can be hidden on mobile: `hidden sm:block` or `hidden md:block`
   - Keep primary KPIs and action buttons always visible

6. Any tables on dashboard: ensure they use `enableViewToggle` from Plan 02

7. All CSS must use logical properties: ms-_, me-_, ps-_, pe-_, text-start, text-end
8. Use `useDirection()` for RTL — never re-derive isRTL from i18n

1. Audit all kanban route and component files. Understand the column structure (likely columns with cards).

2. Apply responsive layout for kanban columns:
   - Mobile (< 640px): Columns stack vertically, each column is full-width. Only show one column at a time with a tab/pill selector at the top to switch between columns (e.g., "Todo | In Progress | Review | Done"). OR show all columns stacked vertically with collapsible sections.
   - Tablet (640-1023px): Horizontal scroll container for columns with `overflow-x-auto` and `-webkit-overflow-scrolling: touch`. Each column min-width 280px.
   - Desktop (1024px+): All columns visible side-by-side in flex row.

3. Kanban cards:
   - Mobile: Full-width cards with `p-3 min-h-12` (48px touch target for cards)

   - "Add card" button: `min-h-11 min-w-11`
