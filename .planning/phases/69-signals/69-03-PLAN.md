  <action>
    Create `frontend/src/hooks/useSignalKeyboardTriage.ts`. This is the only net-new file with no
    codebase analog — build from scratch using standard DOM addEventListener.

    Define and export:

    interface UseSignalKeyboardTriageOptions {
      signals: Signal[]
      onAcknowledge: (id: string) => void
      onDismiss: (id: string) => void
      onEscalate: (signal: Signal | null) => void
      containerRef: React.RefObject<HTMLUListElement>
    }

    interface UseSignalKeyboardTriageResult {
      focusedIndex: number
      setFocusedIndex: React.Dispatch<React.SetStateAction<number>>
    }

    Export function `useSignalKeyboardTriage(options: UseSignalKeyboardTriageOptions): UseSignalKeyboardTriageResult`.

    Implementation:
    - const [focusedIndex, setFocusedIndex] = useState(0)
    - useEffect with the following keydown handler attached to containerRef.current:
      const handler = (e: KeyboardEvent): void => {
        // Guard: ignore when user is typing in a form element
        const tag = (e.target as HTMLElement).tagName
        if (tag === 'INPUT' || tag === 'TEXTAREA' || tag === 'SELECT' || (e.target as HTMLElement).isContentEditable) return
        if (e.key === 'j') { e.preventDefault(); setFocusedIndex((i) => Math.min(i + 1, options.signals.length - 1)) }
        else if (e.key === 'k') { e.preventDefault(); setFocusedIndex((i) => Math.max(i - 1, 0)) }
        else if (e.key === 'a') options.onAcknowledge(options.signals[focusedIndex]?.id ?? '')
        else if (e.key === 'd') options.onDismiss(options.signals[focusedIndex]?.id ?? '')
        else if (e.key === 'e') options.onEscalate(options.signals[focusedIndex] ?? null)
      }
      Attach to containerRef.current, not window (prevents cross-page key leakage).
      Cleanup: return () => el.removeEventListener('keydown', handler)

    - Dependency array: [focusedIndex, options.signals, options.onAcknowledge, options.onDismiss, options.onEscalate, options.containerRef]
    - Reset focusedIndex to 0 in a separate useEffect when options.signals.length changes
      (prevents out-of-bounds after filter changes reduce the list)

    RTL note: j/k are logical up/down (not physical direction). No RTL-specific handling needed here.
    The keys are the same in both LTR and RTL modes.

    File target: ~90 lines including type definitions.

  </action>
