All values from `colors_and_type.css` `--space-*` tokens. Multiples of 4 only.

Sensitivity level (integer 1–4) is shown as a mono badge, never as raw hex color.
Use `var(--ink-mute)` text on `var(--line-soft)` background for all levels.
**Do not use color-coding for sensitivity** — indistinguishable-empty posture
(Phase 68 D-09) forbids any visual differentiation that leaks clearance context.

## Interaction Contract

### Keyboard Triage Inbox (D-02)

The `SignalTriageList` container (`<ul>`) handles `keydown` events:

| Key      | Action                                               | RTL note                                                                                                                                           |
| -------- | ---------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| `j`      | Move focus to next signal (logical down)             | Same key in both directions — no physical mapping                                                                                                  |
| `k`      | Move focus to previous signal (logical up)           | Same key in both directions                                                                                                                        |
| `a`      | Acknowledge focused signal (`status → acknowledged`) | Immediate, no dialog                                                                                                                               |
| `d`      | Dismiss focused signal (`status → dismissed`)        | Immediate, reversible. No confirmation dialog — dismissal is a low-stakes triage action. Toast with "Dismissed. Restore?" undo link for 5 seconds. |
| `e`      | Open EscalateSignalDialog for focused signal         | Opens dialog, focus moves to dialog                                                                                                                |
| `Escape` | Close any open dialog / clear focused row            | Standard focus trap exit                                                                                                                           |

- `j`/`k` navigation: `aria-activedescendant` on the container; focused row gets `aria-selected="true"` role + accent ring
- **Do NOT use physical left/right keys for navigation** — RTL-safe means logical only
- Focus ring: `outline: 2px solid var(--accent); outline-offset: -2px` on the `<li>` row
- Keyboard event handler attaches to the list container, NOT the window (prevents cross-page leakage)

### Dismiss + Restore (D-04)

- Dismiss sets `status = 'dismissed'` and moves signal to the 'Dismissed' filter view
- Toast: "Signal dismissed." with a "Restore" button for 5 seconds. Toast uses `role="status"` (not `role="alert"` — non-urgent)
- "Restore" sets `status` back to `acknowledged` (not `new` — it was already seen)
- The 'Dismissed' filter is always accessible from the toolbar filter pills

### Escalate → Work Item (D-10, D-11, D-12)

1. User presses `e` or clicks "Escalate" — `EscalateSignalDialog` opens
2. Dialog pre-fills: task title from signal title, description from body, priority from severity
3. User confirms → creates `task` (`source: 'task'`, `workflow_stage: 'todo'`) + copies dossier links to `work_item_dossiers`
4. Signal `status → escalated`; signal moves to 'Escalated' filter
5. In the 'Escalated' filter row, a "View work item →" link renders with the `task` ID (deeplink to Kanban)

### Status Lifecycle Transitions (D-03)

```
new ──→ acknowledged  (key: a, or click "Acknowledge")
new ──→ dismissed     (key: d, or click "Dismiss")
new ──→ escalated     (key: e, then confirm dialog)
acknowledged ──→ dismissed  (key: d)
acknowledged ──→ escalated  (key: e)
dismissed ──→ acknowledged  (restore action)
```

No transition from `escalated` back to any status (immutable once escalated).

## RTL Contract

This is the most critical section. The executor must verify every point before
marking any task complete.

### Free-text signal content

Signal `title` and `body` (content) are stored as-entered (D-09). They may be
Arabic, English, or mixed. **Wrap every rendered instance in `<bdi>`:**

```tsx
// CORRECT — isolates directionality of the free-text value
<bdi className="text-body">{signal.title}</bdi>

// WRONG — assumes a fixed direction
<span style={{ textAlign: 'right' }}>{signal.title}</span>
```

`<bdi>` is the standard HTML element for bidirectional isolation. No React-specific
workaround needed.

### Layout

- All margins/padding: logical properties (`ms-*`, `me-*`, `ps-*`, `pe-*`)
- All text alignment: `text-start` (never `text-left`/`text-right`)
- Directional icons in rows (chevrons): use existing `useDirection()` hook + `rotate-180` class per existing IntakeQueue pattern
- `dir` on the SignalTriageList container: read from `useDirection()` and pass as `dir={isRTL ? 'rtl' : 'ltr'}`
- Filter toolbar: logical flex layout — pills wrap naturally in both directions

### The RTL self-check the executor runs before each commit

1. Signal row in `dir="rtl"`: title block appears on the right, badges on the left?
2. No `textAlign: "right"` anywhere in new code?
3. No `ml-*`/`mr-*`/`pl-*`/`pr-*` Tailwind classes?
4. Every `<bdi>` present on free-text title/body?
5. Keyboard hints in AR language when `isRTL` is true?
6. `EscalateSignalDialog` has `dir={isRTL ? 'rtl' : 'ltr'}` on `DialogContent`?

### Indistinguishable-empty rule (Phase 68 D-09, inherited)

The empty state copy above is **identical** whether there are:
(a) genuinely zero signals, or
(b) signals that exist but are above the user's clearance level.

The executor must NOT add any copy like "No visible signals" or "Signals may be
filtered by clearance" or "N signals are above your clearance level". The blank
queue and the clearance-denied queue are visually and textually indistinguishable.

### Destructive actions

There are NO hard-delete or irreversible destructive actions in this phase:

- Dismiss is reversible (D-04) — no confirmation dialog required, just a 5-second toast undo
- Escalate is irreversible but constructive (creates a task) — compact dialog with "Escalate" / "Cancel"
- No signal deletion in P69

## i18n Namespace

A **new namespace** `intelligence-signals` must be created and registered in
`frontend/src/i18n/index.ts` before any component that consumes it is rendered.

Registration pattern (mirrors Phase 68 precedent from quick 260605-u2z):

```typescript
// frontend/src/i18n/index.ts — add to the resources object
import intelligenceSignalsEn from './en/intelligence-signals.json'
import intelligenceSignalsAr from './ar/intelligence-signals.json'

// In resources:
en: { 'intelligence-signals': intelligenceSignalsEn, ... }
ar: { 'intelligence-signals': intelligenceSignalsAr, ... }
```

Files to create:

- `frontend/src/i18n/en/intelligence-signals.json`
- `frontend/src/i18n/ar/intelligence-signals.json`

Use the namespace in components:

```typescript
const { t } = useTranslation('intelligence-signals')
t('queue.emptyState.heading') // → "No signals."
```

**Do not** use dot-form (`useTranslation('intelligence.signals')`) — that
resolves against the `defaultNS` (`translation`/common bundle) and silently
misses all keys per the known dot-vs-colon trap (MEMORY note).

## Signal Row Anatomy (reference for executor)

Below is the precise field layout for a single `SignalTriageRow` in the list.
All widths are logical (start/end, not left/right).

```
┌─────────────────────────────────────────────────────────────────────────┐
│  [L1 badge]  [TITLE bdi]                [SEV badge] [CAT chip] [STATUS] │
│              [Tue 28 Apr · human_entered · Saudi Arabia dossier]         │
│              [AI confidence badge — only if ai_generated]               │
└─────────────────────────────────────────────────────────────────────────┘
```

In RTL (`dir="rtl"`), the flex row reads right-to-left:

- Signal title block appears on the **right** (reading start)
- Badges cluster on the **left**

`L1 badge` = sensitivity level badge (e.g. "L1", "L3") — mono small, `bg-line-soft`

**Important:** The sensitivity level badge is a visual affordance, NOT a color

## AI Confidence Badge (D-08)

- **When shown:** Only on signals where `source_type = 'ai_generated'` AND `ai_confidence IS NOT NULL`
- **When hidden:** All `source_type = 'human_entered'` signals — no badge, no placeholder
- **Format:** `AI · {N}%` where N = `Math.round(ai_confidence * 100)` — mono small font
- **Colors:** `text-accent` foreground + `bg-accent-soft` background (the same accent tint used for AI suggestion in IntakeQueue.tsx)
- **Placement:** Third line of the signal row meta area, or inline after the category chip — planner's call on exact placement
- **ARIA label:** `aria-label="AI generated, {N}% confidence"` / Arabic: `aria-label="ذكاء اصطناعي، ثقة {N}%"`
