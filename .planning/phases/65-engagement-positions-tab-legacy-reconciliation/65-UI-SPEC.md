### Section anatomy

| Element             | Contract                                                                                                                                                                                                                                                                                                                                               |

| Section heading     | `h3`, exact sibling style: `text-xs font-semibold text-muted-foreground uppercase tracking-wider`, `space-y-2` to content. | Empty state         | Section always renders. Zero entries ΓåÆ single muted line inside the container area: `text-sm text-muted-foreground`, copy per table below. | Error state         | Query error ΓåÆ same muted-line treatment with `text-[var(--danger)]`-class token color via existing destructive text convention, copy per table below                                                                                                                                                                                                   |

### Tab-level empty-state interplay

The existing whole-tab empty state (`empty.calendar.*`) now triggers only when derived events AND scheduled entries are BOTH empty (and while the entries query is unresolved, keep current derived-only behavior ΓÇö no flash). Its Add event button is live per CTA #6. When entries exist but derived dates don't, render the normal layout (header bar + Scheduled events section).

## Bilingual & RTL Contract

- Tab nav, tab pages, and the entries section take `dir` from `useDirection()`; Tajawal cascades automatically in AR.
- Logical properties only (`ms-*`, `me-*`, `ps-*`, `text-start`, `justify-between` flex order). Never `ml/mr/pl/pr/text-left/text-right`. Row layout = `justify-between` so start/end flip free under `dir`.
- Entry titles locale-picked (`title_ar` in AR with `title_en` fallback); badge labels via bilingual `calendar:types.*`; dates via `toFormatLocale` for AR (Arabic month/day names, correct digits) and `en-GB` for EN (day-first `Tue 28 Apr`).
- No `.reverse()`; no directional icons in scope (`Plus`, calendar glyphs are symmetric).
- `tabs.positions` lands in BOTH workspace.json files in the same edit (Pitfall 7 ΓÇö silent EN fallback in AR).
- Verify live in BOTH locales at 1280px and 1024px; tab nav horizontal scroll-snap already handles narrow widths.
