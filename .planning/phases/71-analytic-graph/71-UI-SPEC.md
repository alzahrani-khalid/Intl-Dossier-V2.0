### Inherited prose chrome (NOT new scale entries ŌĆö explicit checker exception)

The following reuse existing design-system classes and tokens. They are **inherited chrome**, not
additional typographic scale tokens, and must not be counted as new font sizes:

- **Mono labels / tabular metrics** ŌĆö `--t-mono-small` (11px) via `.t-mono` for path-hop counts,
  `query_time_ms`, and the complexity badge numerals. Mono is a _family_ (`--font-mono`), reused from
  the existing stats strip, not a new scale size.
- **Uppercase labels** ŌĆö `--t-label` (10.5px, `.t-label`) for any `T-label` classification/section
  micro-label. Reused verbatim from the prototype.
- **KPI / stat numerals** ŌĆö the existing Network-panel stat block (`node_count`, `edge_count`,
  `max_degree`, `query_time_ms`) keeps its current numerals. The Analyze result-count chip reuses the
  same treatment; no new KPI scale is introduced.

**Net new typographic scale introduced by this phase: 0.** Everything maps to the 4 roles above plus
inherited chrome.

## Color

Bureau (light) is the locked direction. 60 / 30 / 10 split, all via `var(--*)` tokens or `@theme`-mapped
Tailwind utilities (`bg-bg`, `bg-surface`, `text-ink`, `border-line`, `bg-accent`, `text-accent-ink`).
**No raw hex. No Tailwind color literals (`text-blue-500`).**

| Role            | Value                                                                             | Usage                                                                                                                                          |
| --------------- | --------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| Dominant (60%)  | `--bg` `#f7f6f4` (`bg-bg`)                                                        | Network-panel canvas behind the query picker and result region                                                                                 |
| Secondary (30%) | `--surface` `#ffffff` (`bg-surface`)                                              | The query-picker `.card`, the result-card, Cmd+K result rows, the React Flow canvas surface                                                    |
| Accent (10%)    | `--accent` `oklch(58% 0.14 32)` (terracotta) + `--accent-ink` for text on neutral | See reserved-for list below                                                                                                                    |
| Destructive     | `--danger` `oklch(52% 0.18 25)` (`text-danger` / `--danger-soft` chip)            | **Error state banner only** (query failed / network error). There are NO destructive _actions_ in this phase ŌĆö analytic queries are read-only. |

**Accent reserved for (explicit ŌĆö never "all interactive elements"):**

1. The single **primary "Run analysis" CTA** (`.btn-primary`, accent fill) per query template.
2. The **active query-template selection** state in the picker (accent left/inline-start hairline or
   `--accent-soft` chip background ŌĆö mirror the Phase-70 `.tab.active` accent-underline pattern, NOT a
   filled block).
3. The **active Graph/List tab underline** (already accent in the existing `Tabs` ŌĆö reused as-is).
4. The **highlighted subgraph** in the React Flow "Graph view": result nodes/edges are emphasized with
   `--accent` / `--accent-soft`; non-result context nodes stay neutral `--ink-mute` / `--line`.
5. **Selected/focused result row** affordance (the existing `RelationshipNavigator` row uses
   `hover:bg-accent`; the Analyze result reuses row hover ŌåÆ `--line-soft` per the design-system hover
   rule, with accent reserved for the _selected_ row).

Semantic non-accent tokens permitted where they already carry meaning in the panel: complexity badge
(`--ok` / `--warn` / `--danger` soft variants, exactly as the existing stats strip uses them) and the
`performance_warning` banner (`--warn`). These are inherited semantic chrome, not decorative color.

> **Note on the existing panel's current classes:** `RelationshipGraphPage.tsx` today uses shadcn-default
> utilities (`bg-muted`, `text-muted-foreground`, `text-2xl font-bold`). Those `@theme`-mapped utilities
> _do_ resolve to `--ink-mute` etc., but the **new Analyze surfaces must be authored with the raw token
> path** (`bg-surface`, `text-ink`, `text-ink-mute`, `var(--t-body)`) to stay on-brand and pass the
> Definition-of-Done checklist. Do not retroactively re-skin the whole panel (out of scope) ŌĆö only the
> new query-picker, result-card, and Cmd+K-result markup.

### Clearance-reduction copy contract (LOCKED ŌĆö non-negotiable, P68 D-09 / GRAPH-03)

- The empty state and the reduced-result state use **identical copy** to a genuine empty/partial result.
- **There must be NO "filtered by clearance", "restricted", "you do not have access", or any
  classification-revealing string anywhere in this phase's copy** (search the diff: zero occurrences of
  `clearance`, `restricted`, `filtered`, `hidden`, `permission` in user-visible strings).
- A lower-clearance caller silently sees fewer rows/nodes/hops. The empty state must read as "no results
  exist", never hint that above-clearance data was withheld. This is indistinguishable-empty: the copy is
  a property of the contract, not a state variant.

### Destructive actions

**None.** Analytic graph queries are read-only `SECURITY INVOKER` RPC calls. No create/update/delete, no
confirmation dialogs. (The `query_graph` agent tool is GRAPH-04 and is verified by direct RPC invocation,
not a UI write.)

## Layout & Responsive

Desktop-primary analyst workstation. Build for 1280px, verify at 1024px and 1400px.

- Max page width 1400px (inherited from the panel). The Analyze picker + result region sit inside the
  existing panel layout.
- Picker form: single-column on `< 768px`; 2-up field grid `sm:` and above (mirrors the panel's existing
  `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4` controls).
- The structured result is the mobile-default (graph view is desktop-primary; React Flow stays available
  but is secondary on small widths).
- **Logical properties only** (`ms-*`, `me-*`, `ps-*`, `pe-*`, `text-start`, `text-end`, `start-*`,
  `end-*`). No `ml-*`/`pl-*`/`text-left`. Directional chevrons flip via `rotate-180` on RTL (panel
  already does this).
- Borders `1px solid var(--line)`. **No drop-shadows on cards** ŌĆö shadow only on the drawer tier
  (`--shadow-lg`) and hovered list rows. **No gradient backgrounds** ŌĆö surfaces flat.
- Corner radii from `--radius-sm` / `--radius` / `--radius-lg` (Bureau 8/12/16). No hardcoded px radii.
