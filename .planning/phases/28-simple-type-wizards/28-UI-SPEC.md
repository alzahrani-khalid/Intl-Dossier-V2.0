## Typography

Uses project's Compact UI scale from `index.css` tokens. Phase 28 wizards use these 4 roles with 2 weights:

| Role            | Size             | Weight         | Line Height | CSS Token                     | Usage                                                            |
| --------------- | ---------------- | -------------- | ----------- | ----------------------------- | ---------------------------------------------------------------- |
| Body            | 13px (0.8125rem) | 400 (normal)   | 1.5         | `--text-base`                 | Form field values, review field values                           |
| Label           | 12px (0.75rem)   | 400 (normal)   | 1.375       | `--text-sm`                   | FormLabel, review field labels (`text-xs text-muted-foreground`) |
| Section heading | 12px (0.75rem)   | 600 (semibold) | 1.25        | `--text-sm` + `font-semibold` | ReviewSection title (`text-sm font-semibold`)                    |
| Page title      | 14px (0.875rem)  | 600 (semibold) | 1.25        | `--text-lg` + `font-semibold` | Wizard page heading (`text-lg font-semibold`)                    |

Weight budget: 2 weights -- 400 (normal) for body and labels, 600 (semibold) for headings. Labels are visually differentiated from body text by smaller size (12px vs 13px) and `text-muted-foreground` color treatment, so 400 weight is sufficient.

RTL rule: All Arabic text fields use `writingDirection: "rtl"` via `dir={direction}` from `useDirection()` hook. Never use `textAlign: "right"`.

## Interaction Contracts

### Organization Wizard (3 steps)

**Step 1 -- SharedBasicInfoStep:** Reused as-is. Bilingual names, abbreviation, description, tags, classification.

**Step 2 -- OrgDetailsStep:** Layout matches CountryDetailsStep pattern.

- `org_type`: Select dropdown with 5 options (government, ngo, private, international, academic). Full-width. `min-h-11`.
- `org_code`: Input, full-width. `min-h-11`. Placeholder: abbreviated org identifier.
- `website`: Input with `type="url"`. Full-width. `min-h-11`.
- `headquarters_en` + `headquarters_ar`: 2-column grid (`grid grid-cols-1 sm:grid-cols-2 gap-4`). Arabic field gets `dir={direction}`. Both `min-h-11`.
- No founding_date field in schema (CONTEXT D-01 mentions it but schema does not include it -- follow schema as source of truth).

**Step 3 -- Review:** OrganizationReviewStep with 2 ReviewSections.

### Topic Wizard (2 steps)

**Step 1 -- SharedBasicInfoStep with inline theme_category:** The `theme_category` Select is rendered inline within the SharedBasicInfoStep or immediately after it within the same FormWizardStep. Single-select dropdown: policy, technical, strategic, operational. Full-width. `min-h-11`.

**Step 2 -- Review:** TopicReviewStep with 1 ReviewSection (Basic Info + theme_category row).

### Person Wizard (3 steps)

**Step 1 -- SharedBasicInfoStep:** Reused as-is.

**Step 2 -- PersonDetailsStep:**

- `title_en` + `title_ar`: 2-column grid (`grid grid-cols-1 sm:grid-cols-2 gap-4`). Free-text Input (not dropdown per D-02). Arabic field gets `dir={direction}`. Both `min-h-11`.
- `photo_url`: File picker button + thumbnail preview (128x128px rounded, `object-cover`). If no photo, show placeholder avatar icon. Simple `<input type="file" accept="image/*">` per D-03 -- no cropping.
- `biography_en` + `biography_ar`: Stacked full-width Textarea fields. Each with `min-h-[88px]` (2 visible lines minimum). Arabic field gets `dir={direction}`.

**Step 3 -- Review:** PersonReviewStep with 2 ReviewSections. Photo shown as 48x48px thumbnail in review. Biography truncated to 120 chars with ellipsis (matching country description pattern).

## Layout Patterns

### Wizard Page Layout (all three types, identical to country)

```
div.space-y-4
  Link (back to list) -- ChevronLeft icon + "Back to {Type}" label
  h1.text-lg.font-semibold -- "{Type} wizard title"
  CreateWizardShell
    Step 1: SharedBasicInfoStep
    Step 2: {TypeDetailsStep} (org/person only)
    Step N: {TypeReviewStep}
```

### Form Field Layout

- Single field: Full-width within `FormWizardStep.space-y-6`
- Bilingual pair: `grid grid-cols-1 sm:grid-cols-2 gap-4`
- All inputs: `min-h-11` (44px touch target)
- All selects: `min-h-11` on SelectTrigger

### Review Layout

- Sections: `rounded-lg border border-border bg-card p-4 space-y-3`
- Section header: flex row with title (start) + Edit button (end)
- Fields: `grid grid-cols-1 sm:grid-cols-2 gap-3 text-sm`
- Edit button: `variant="ghost" size="sm"` with Pencil icon + "Edit" label. `min-h-11 min-w-11`.

## State & Interaction Details

### Draft Persistence

Each wizard persists to localStorage using existing `useFormDraft<T>` hook:

- Organization: `dossier-create-organization`
- Topic: `dossier-create-topic`
- Person: `dossier-create-person`

### Post-Creation Navigation

All three use `getDossierDetailPath()` to navigate to the new dossier's detail page after successful creation (per D-11).

### Validation Timing

- Field-level: `onBlur` validation via react-hook-form + zodResolver
- Step-level: Validated before advancing to next step (existing FormWizard behavior)
- Submit: Full schema validation on review step submit

### Photo Upload (Person only)

- File picker: Native `<input type="file" accept="image/*">` hidden, triggered by styled button
- Preview: 128x128px thumbnail with `rounded-lg object-cover`
- State: Store as base64 data URL in form state; convert to Storage upload on submit
- Remove: Small "Remove Photo" button appears next to thumbnail when photo is set

## RTL Considerations

All patterns inherit from the established RTL-safe conventions:

- Logical properties only: `ms-*`, `me-*`, `ps-*`, `pe-*`, `text-start`, `text-end`
- Arabic text fields: `dir={direction}` from `useDirection()` hook
- ChevronLeft icon for "back" navigation (correct for RTL -- points left = "return")
- Grid layouts: `grid-cols-1 sm:grid-cols-2` -- no directional assumptions
- Touch targets: `min-h-11 min-w-11` on all interactive elements
