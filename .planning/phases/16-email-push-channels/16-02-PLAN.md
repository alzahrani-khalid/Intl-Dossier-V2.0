  <action>
    Create `backend/src/queues/digest-scheduler.ts` following the exact pattern from `deadline-scheduler.ts` (per D-03):

    1. `registerDigestScheduler(): Promise<void>` -- registers two repeatable jobs on `notificationQueue`:
       - `'process-daily-digests'` with `repeat: { every: 60 * 60 * 1000 }` (hourly) and `jobId: 'daily-digest-processor'`
       - `'process-weekly-digests'` with `repeat: { every: 24 * 60 * 60 * 1000 }` (daily) and `jobId: 'weekly-digest-processor'`
       - Log registration with `logInfo('Digest schedulers registered')`

    2. `processDigestJob(jobName: string): Promise<void>` -- called when job fires:
       - For `'process-daily-digests'`:
         a. Get current UTC hour
         b. Query `email_notification_preferences` for users where `digest_frequency = 'daily'`
         c. For each user, convert their `daily_digest_time` + `quiet_hours_timezone` to UTC and check if it matches current hour
         d. For matching users, call `supabase.rpc('get_user_digest_content', { p_user_id: userId })`
         e. Get user language from `user_preferences`
         f. Call `renderDailyDigestTemplate(language, today, content)`
         g. Insert into `email_queue` with `template_type: 'digest_daily'`, priority 5 (low)

       - For `'process-weekly-digests'`:
         a. Get current UTC day of week (0-6)
         b. Query users where `digest_frequency = 'weekly'` AND `weekly_digest_day` matches current day
         c. Same flow as daily: get content, render, insert into email_queue with `template_type: 'digest_weekly'`

    3. Wrap each user's processing in individual try/catch -- one user's failure should not block others. Log errors with `logError()`.

    4. Import from existing modules: `notificationQueue` from `./notification.queue`, `renderDailyDigestTemplate`/`renderWeeklyDigestTemplate` from `../services/digest-template.service`, `supabase` from the existing supabase client.

    5. Wire `processDigestJob` into the notification worker's job handler in `notification.queue.ts` by adding cases for the new job names (or document for the executor to add the switch case).
