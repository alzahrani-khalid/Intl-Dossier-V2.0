### Component: PushOptInBanner

**Trigger:** Appears after user receives first actionable notification (task assignment or overdue deadline). Never on cold page load (D-10).

**Layout:**

```
+----------------------------------------------------------+
|  [Bell icon]  Stay updated even when you're away         |
|               Get notified about urgent deadlines and    |
|               new assignments directly in your browser.  |
|                                                          |
|  [Enable notifications]              [Not now]           |
+----------------------------------------------------------+
```

**States:**

1. **Hidden** -- default; user has no actionable notifications yet, or has dismissed, or already subscribed
2. **Visible** -- user has received first actionable notification AND has not dismissed AND push not yet enabled
3. **Loading** -- after clicking "Enable notifications", browser permission dialog is open
4. **Success** -- permission granted, banner auto-dismisses with success toast
5. **Denied** -- permission denied by browser, show error toast, banner dismisses

**Styling contract:**

- Background: `bg-muted/50` with `border border-border rounded-lg`
- Padding: `p-4` (16px)
- Icon: `Bell` from lucide-react, `h-5 w-5 text-primary-600`
- Headline: `text-sm font-semibold text-foreground`
- Body: `text-xs text-muted-foreground`
- CTA button: `h-9 px-4 bg-primary text-primary-foreground rounded-md text-sm font-medium` (min-w-11 min-h-11 touch target via padding)
- Dismiss: `text-xs text-muted-foreground hover:text-foreground` as ghost button, `min-h-11 min-w-11`
- RTL: Uses `ms-*`, `me-*`, `ps-*`, `pe-*` logical properties. `dir` set via `isRTL` from `useTranslation`
- Mobile: Full-width stack on base, inline row on sm+
- Layout: `flex flex-col sm:flex-row items-start sm:items-center gap-3`

**Dismissal persistence:** Calls API to set `push_prompt_dismissed_at` in `user_preferences` table (D-11).

## Interaction Contracts

### Soft-Ask Flow

1. User logs in and navigates normally (no banner shown)
2. User receives first actionable notification (assignment or overdue deadline)
3. `useNotificationCenter` hook detects first actionable notification AND checks `push_prompt_dismissed_at` is null AND `Notification.permission !== 'granted'`
4. PushOptInBanner renders inside NotificationPanel or as a toast-like banner below the header
5. User clicks "Enable notifications" --> browser permission dialog opens
6. If granted: subscribe via Push API, send subscription to backend, show success toast, banner auto-dismisses
7. If denied: show error toast with guidance, banner dismisses, set `push_prompt_dismissed_at`
8. User clicks "Not now": set `push_prompt_dismissed_at` in `user_preferences`, banner dismisses

### Push Notification Click

1. Service worker receives push event, displays native notification
2. User clicks notification --> `notificationclick` event fires
3. Service worker opens or focuses app window at the notification's `url` (deep link to relevant dossier/work item)

### Email Unsubscribe

1. User clicks "Unsubscribe" link in email footer
2. Link navigates to `/settings?section=notifications` in the app
3. User adjusts toggles in NotificationsSettingsSection (existing UI, no new work)

## RTL Contract

All new components MUST follow these rules (from CLAUDE.md):

| Rule                    | Implementation                                                                                              |
| ----------------------- | ----------------------------------------------------------------------------------------------------------- |
| Logical properties only | `ms-*`, `me-*`, `ps-*`, `pe-*`, `text-start`, `text-end` -- never `ml-*`, `mr-*`, `text-left`, `text-right` |
| Dir attribute           | `dir={isRTL ? 'rtl' : 'ltr'}` on PushOptInBanner container                                                  |
| Icon direction          | Bell icon is non-directional -- no flip needed                                                              |
| Email RTL               | `dir="rtl"` inline attribute on `<html>`, `<body>`, `<td>` for Arabic emails                                |
| Push notification RTL   | `dir: 'rtl'` and `lang: 'ar'` in push payload for Arabic users                                              |
