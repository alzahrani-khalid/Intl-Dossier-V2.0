## Spacing Scale

Declared values (all multiples of 4, mapped to Phase 33 density tokens):

| Token | Value | Usage                                                                            |
| ----- | ----- | -------------------------------------------------------------------------------- |
| xs    | 4px   | Icon gap inside direction-button tagline row                                     |
| sm    | 8px   | Gap between section title and its controls; gap between direction buttons        |
| md    | 12px  | Padding inside direction buttons, swatch border radius baseline                  |
| lg    | 16px  | Gap between drawer sections (vertical rhythm), drawer header/body padding-inline |
| xl    | 20px  | Drawer body `padding-block`, header `padding-block`                              |
| 2xl   | 24px  | Total vertical gap between the 6 sections (`gap-6`)                              |

Exceptions:

- Drawer panel width: **360px** fixed on sm+ (≥640px), **100vw** on base (<640px). Not a scale value — a container dimension from D-04.
- Gear trigger touch target: **44×44px minimum** (CLAUDE.md mandate) — button wraps a 20–24px `Settings2` icon with auto-centered padding.
- Swatch row swatches: **height 24px, flex-1, gap 8px** — handoff-literal dimensions for the 5-swatch hue row (D-06).
- Direction-button inner padding: **10px** — handoff-literal to preserve visual rhythm (D-17).

## Typography

All sizes consume `--font-body` (system-ui until Phase 35). Line heights tuned for dense control panel readability. Exactly **4 distinct sizes** (11/13/14/16) and exactly **2 weights** (400, 600).

| Role                     | Size | Weight | Line Height | Usage                                                                                                                          |
| ------------------------ | ---- | ------ | ----------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Drawer title             | 16px | 600    | 1.3         | "Tweaks" / "تعديلات" in `DrawerHeader`                                                                                         |
| Section heading          | 13px | 600    | 1.3         | "Direction" / "Mode" / "Hue" / etc. — uppercase optional via utility                                                           |
| Direction button primary | 13px | 600    | 1.3         | "Chancery" / "Situation Room" label (D-17 locked)                                                                              |
| Degree readout           | 13px | 600    | 1.3         | Live hue value "158°" — rendered in `--font-mono` with inline `dir="ltr"` (D-08: Western-Arabic numerals + ° symbol preserved) |
| Body / control label     | 14px | 400    | 1.5         | Light/Dark toggle labels, Comfortable/Compact labels, EN/ع labels                                                              |
| Direction button tagline | 11px | 400    | 1.4         | "Formal, warm amber" caption under primary label (D-17 locked — semantic weight: labels each direction's palette identity)     |

**Weight policy:** exactly 2 weights used — **400 (regular)** and **600 (semibold)**. No 500/700 mixing.

**Size count:** exactly 4 distinct sizes — **11 / 13 / 14 / 16**. The degree readout shares the 13px tier with section headings and direction-primary labels; its `--font-mono` family and `dir="ltr"` wrapper provide the visual differentiation without adding a new size tier.

## Color

All color references resolve to Phase 33 runtime tokens via `var(--*)`. No hard-coded hex in the drawer UI. The drawer inherits whatever direction × mode × hue the user has set — it MUST redraw itself correctly as the user drags the hue slider or switches direction.

| Role            | Token                                                        | Usage                                                                                                                                                                          |
| --------------- | ------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Dominant (60%)  | `var(--surface)`                                             | Drawer panel background                                                                                                                                                        |
| Secondary (30%) | `var(--bg)`                                                  | Backdrop (tinted + blurred — see below); inactive direction-button background uses `var(--surface)` too, with `var(--line)` border providing separation                        |
| Accent (10%)    | `var(--accent)` / `var(--accent-soft)` / `var(--accent-ink)` | Active direction button (`bg-accent-soft`, `border-accent`, `text-accent-ink`); active swatch 2px ring via `var(--ink)`; slider track fill; focus ring via `var(--focus-ring)` |
| Destructive     | not used                                                     | No destructive action in this drawer                                                                                                                                           |

**Accent reserved for (exhaustive list):**

1. Active direction button chrome (border + soft-fill + accent-ink text per D-17)
2. Slider filled track (before the thumb)
3. Active mode toggle (light/dark pill — whichever is current)
4. Active density toggle (comfortable/compact pill)
5. Active locale toggle (EN or ع pill)
6. "Classification" switch `on` state (HeroUI Switch `color="primary"` — primary bridged to `--accent` per Phase 33 D-06)
7. Focus ring on any focused interactive element (`outline: 2px solid var(--focus-ring)`)

**NOT accent-colored:** drawer header, section headings, body text, inactive buttons, inactive swatches, close button chrome.

**Backdrop specification (D-04 verbatim):**

- Background: `color-mix(in srgb, var(--bg) 82%, transparent)`
- Filter: `backdrop-filter: blur(3px)`
- Click-to-close: yes (HeroUI Drawer default; confirm via component docs)

**Swatch row colors (D-06 verbatim, not tokens — this is the only place the drawer prints non-token color):**

- Swatch 1: `oklch(58% 0.14 22)`
- Swatch 2: `oklch(58% 0.14 158)`
- Swatch 3: `oklch(58% 0.14 190)`
- Swatch 4: `oklch(58% 0.14 258)`
- Swatch 5: `oklch(58% 0.14 330)`
- Active state: `box-shadow: 0 0 0 2px var(--ink)` when `|currentHue − swatch| < 4`

## Interaction Contract

### Drawer open/close

- **Trigger:** gear button in `frontend/src/components/layout/SiteHeader.tsx` (D-05). 44×44px button, `Settings2` icon 20–24px, `aria-label={t('tweaks.open')}`. Researcher flags `Header.tsx` as potential secondary trigger site — planner decides after auditing render maps; default assumption is single trigger is enough for Phase 34.
- **Open state:** `isTweaksOpen` boolean — owned at App-level (small Zustand store OR React Context). Gear button gets `aria-expanded={isTweaksOpen}` and an active visual state (`bg-accent-soft` when open).
- **Edge:** HeroUI `placement="right"` in LTR. In RTL (`<html dir="rtl">`), HeroUI React Aria layer SHOULD auto-flip to physical-left. **Planner verification required** via `mcp__heroui-react__get_component_docs`: if it does not auto-flip, pass placement dynamically: `placement={i18n.dir() === 'rtl' ? 'left' : 'right'}`.
- **Close paths:** (a) click close button (`X` icon, top inline-end corner of DrawerHeader), (b) press `Escape`, (c) click backdrop, (d) click gear trigger again (toggle).
- **Focus management:** HeroUI Drawer uses React Aria → focus traps inside drawer on open, returns to gear trigger on close. Zero custom wiring.
- **Animation:** HeroUI default slide-in from inline-end. Respect `prefers-reduced-motion: reduce` — HeroUI React Aria honors this automatically; planner verifies.

### Responsive behavior

| Breakpoint    | Panel width                   | Backdrop                                                                                                  |
| ------------- | ----------------------------- | --------------------------------------------------------------------------------------------------------- |
| base (<640px) | `100vw` (full-screen overlay) | Still renders (backdrop + blur) but functionally invisible behind full-width panel — keep for consistency |
| sm (≥640px)   | `360px` fixed                 | Visible tinted + blurred backdrop                                                                         |
| md+ (≥768px)  | `360px` fixed                 | Same                                                                                                      |
| lg/xl/2xl     | `360px` fixed                 | Same                                                                                                      |

### Section layout inside drawer body

Vertical stack with `gap-6` (24px) between sections. Each section:

```
[Section heading — 13px/600]
[Control row(s) — per section below]
```

1. **Direction** — 2×2 grid of direction buttons (4 total). Each button is 2-line stacked (primary label 13/600 + tagline 11/400), logical-start text alignment, padding 10px, `rounded-[var(--radius-sm)]`. Active: `bg-accent-soft border-accent text-accent-ink`. Inactive: `bg-surface border-line text-ink`.
2. **Mode** — segmented pill toggle (Light | Dark). Single row, `flex gap-2`, each pill flex-1, 44px min touch height. Active pill uses accent chrome; inactive uses `bg-surface border-line`.
3. **Hue** — two sub-rows, `gap-sm`:
   - Row A: Range slider (0–360, step 1) flex-1 + degree readout `<span dir="ltr">{hue}°</span>` inline-end, **13px/600**, `--font-mono`. Slider track filled portion uses `var(--accent)`.
   - Row B: 5 swatch buttons, `flex gap-2`, each `flex-1 h-6 rounded-[var(--radius-sm)]`. Active swatch: `2px solid var(--ink)` ring.
4. **Density** — segmented pill toggle (Comfortable | Compact). Same chrome as Mode.
5. **Classification** — label `Classification` + HeroUI `Switch` (boolean on/off). Switch uses `color="primary"` which bridges to `--accent` via Phase 33 D-06.
6. **Locale** — segmented pill toggle (EN | ع). Same chrome as Mode. Click updates `id.locale`, calls `i18n.changeLanguage(lang)`, sets `<html dir/lang>` attributes (D-13).

### Keyboard flow

- Tab order: close button → Direction buttons (4 tabstops) → Mode pills (2) → Hue slider → Hue swatches (5) → Density pills (2) → Classification switch → Locale pills (2) → loop.
- Arrow keys within segmented groups (Mode/Density/Locale) — HeroUI default behavior from React Aria; no custom wiring.
- Slider: Left/Right arrows ±1°, Home/End = 0/360, PageUp/Down ±10°. HeroUI Slider defaults.
- Escape dismisses drawer.

### RTL behavior checklist

- Drawer slides from inline-end: physical right in LTR, physical left in RTL. **Planner to verify HeroUI `placement` semantics** — if props are physical-only, pass dynamically.
- All section headings, labels, taglines use `text-start` (logical) — never `text-left`/`text-right`.
- Swatch row naturally mirrors (flexbox row follows `dir`). Swatch order `[22, 158, 190, 258, 330]` in both DOM and visual — handoff-literal.
- Degree readout `<span dir="ltr">{hue}°</span>` — degree symbol + number stay Western-numeral even in RTL (D-08).
- Direction buttons: logical text alignment (start-aligned); no rotation.
- Close `X` icon: symmetric — no `rotate-180` needed.
- Gear icon: symmetric — no `rotate-180` needed.

### Persistence contract

Every control writes immediately on change:

| Control        | localStorage key | Value shape                                                    |
| -------------- | ---------------- | -------------------------------------------------------------- |
| Direction      | `id.dir`         | `'chancery'` \| `'situation'` \| `'ministerial'` \| `'bureau'` |
| Mode           | `id.theme`       | `'light'` \| `'dark'`                                          |
| Hue            | `id.hue`         | stringified number `'0'` through `'360'`                       |
| Density        | `id.density`     | `'comfortable'` \| `'compact'`                                 |
| Classification | `id.classif`     | `'true'` \| `'false'`                                          |
| Locale         | `id.locale`      | `'en'` \| `'ar'`                                               |

Direction change also writes the direction's default `id.theme` + `id.hue` per D-16 map (Chancery=light/22, Situation=dark/190, Ministerial=light/158, Bureau=light/32). No toast. Live UI flip is the confirmation.
