## Spacing Scale

Declared values (all multiples of 4, mapped to Phase 33 density tokens):

| Token | Value | Usage                                                                            |
| ----- | ----- | -------------------------------------------------------------------------------- |
| xs    | 4px   | Icon gap inside direction-button tagline row                                     |
| sm    | 8px   | Gap between section title and its controls; gap between direction buttons        |
| md    | 12px  | Padding inside direction buttons, swatch border radius baseline                  |

- Drawer panel width: **360px** fixed on sm+ (ÔëÑ640px), **100vw** on base (<640px). Not a scale value ÔÇö a container dimension from D-04.

- Swatch row swatches: **height 24px, flex-1, gap 8px** ÔÇö handoff-literal dimensions for the 5-swatch hue row (D-06).
- Direction-button inner padding: **10px** ÔÇö handoff-literal to preserve visual rhythm (D-17).

## Color

All color references resolve to Phase 33 runtime tokens via `var(--*)`. No hard-coded hex in the drawer UI. The drawer inherits whatever direction ├ù mode ├ù hue the user has set ÔÇö it MUST redraw itself correctly as the user drags the hue slider or switches direction.

| Role            | Token                                                        | Usage                                                                                                                                                                          |
| --------------- | ------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Dominant (60%)  | `var(--surface)`                                             | Drawer panel background                                                                                                                                                        |
| Secondary (30%) | `var(--bg)`                                                  | Backdrop (tinted + blurred ÔÇö see below); inactive direction-button background uses `var(--surface)` too, with `var(--line)` border providing separation                        |
| Accent (10%)    | `var(--accent)` / `var(--accent-soft)` / `var(--accent-ink)` | Active direction button (`bg-accent-soft`, `border-accent`, `text-accent-ink`); active swatch 2px ring via `var(--ink)`; slider track fill; focus ring via `var(--focus-ring)` |
| Destructive     | not used                                                     | No destructive action in this drawer                                                                                                                                           |

**Accent reserved for (exhaustive list):**

1. Active direction button chrome (border + soft-fill + accent-ink text per D-17)
2. Slider filled track (before the thumb)
3. Active mode toggle (light/dark pill ÔÇö whichever is current)
4. Active density toggle (comfortable/compact pill)
5. Active locale toggle (EN or Ï╣ pill)
6. "Classification" switch `on` state (HeroUI Switch `color="primary"` ÔÇö primary bridged to `--accent` per Phase 33 D-06)
7. Focus ring on any focused interactive element (`outline: 2px solid var(--focus-ring)`)

**NOT accent-colored:** drawer header, section headings, body text, inactive buttons, inactive swatches, close button chrome.

**Backdrop specification (D-04 verbatim):**

- Background: `color-mix(in srgb, var(--bg) 82%, transparent)`
- Filter: `backdrop-filter: blur(3px)`
- Click-to-close: yes (HeroUI Drawer default; confirm via component docs)

**Swatch row colors (D-06 verbatim, not tokens ÔÇö this is the only place the drawer prints non-token color):**

- Swatch 1: `oklch(58% 0.14 22)`
- Swatch 2: `oklch(58% 0.14 158)`
- Swatch 3: `oklch(58% 0.14 190)`
- Swatch 4: `oklch(58% 0.14 258)`
- Swatch 5: `oklch(58% 0.14 330)`
- Active state: `box-shadow: 0 0 0 2px var(--ink)` when `|currentHue ÔêÆ swatch| < 4`

## Interaction Contract

### Drawer open/close

- **Trigger:** gear button in `frontend/src/components/layout/SiteHeader.tsx` (D-05). 44├ù44px button, `Settings2` icon 20ÔÇô24px, `aria-label={t('tweaks.open')}`. Researcher flags `Header.tsx` as potential secondary trigger site ÔÇö planner decides after auditing render maps; default assumption is single trigger is enough for Phase 34.
- **Open state:** `isTweaksOpen` boolean ÔÇö owned at App-level (small Zustand store OR React Context). Gear button gets `aria-expanded={isTweaksOpen}` and an active visual state (`bg-accent-soft` when open).
- **Edge:** HeroUI `placement="right"` in LTR. In RTL (`<html dir="rtl">`), HeroUI React Aria layer SHOULD auto-flip to physical-left. **Planner verification required** via `mcp__heroui-react__get_component_docs`: if it does not auto-flip, pass placement dynamically: `placement={i18n.dir() === 'rtl' ? 'left' : 'right'}`.
- **Close paths:** (a) click close button (`X` icon, top inline-end corner of DrawerHeader), (b) press `Escape`, (c) click backdrop, (d) click gear trigger again (toggle).
- **Focus management:** HeroUI Drawer uses React Aria ÔåÆ focus traps inside drawer on open, returns to gear trigger on close. Zero custom wiring.
- **Animation:** HeroUI default slide-in from inline-end. Respect `prefers-reduced-motion: reduce` ÔÇö HeroUI React Aria honors this automatically; planner verifies.

### Responsive behavior

| ------------- | ----------------------------- | --------------------------------------------------------------------------------------------------------- |
| base (<640px) | `100vw` (full-screen overlay) | Still renders (backdrop + blur) but functionally invisible behind full-width panel ÔÇö keep for consistency |
| sm (ÔëÑ640px)   | `360px` fixed                 | Visible tinted + blurred backdrop                                                                         |
| md+ (ÔëÑ768px)  | `360px` fixed                 | Same                                                                                                      |
| lg/xl/2xl     | `360px` fixed                 | Same                                                                                                      |

### Section layout inside drawer body

Vertical stack with `gap-6` (24px) between sections. Each section:

```
[Section heading ÔÇö 13px/600]
[Control row(s) ÔÇö per section below]
```

1. **Direction** ÔÇö 2├ù2 grid of direction buttons (4 total). Each button is 2-line stacked (primary label + tagline), logical-start text alignment, padding 10px, `rounded-[var(--radius-sm)]`. Active: `bg-accent-soft border-accent text-accent-ink`. Inactive: `bg-surface border-line text-ink`.
2. **Mode** ÔÇö segmented pill toggle (Light | Dark). Single row, `flex gap-2`, each pill flex-1, 44px min touch height. Active pill uses accent chrome; inactive uses `bg-surface border-line`.
3. **Hue** ÔÇö two sub-rows, `gap-sm`:
   - Row A: Range slider (0ÔÇô360, step 1) flex-1 + degree readout `<span dir="ltr">{hue}┬░</span>` inline-end, `--font-mono`. Slider track filled portion uses `var(--accent)`.
   - Row B: 5 swatch buttons, `flex gap-2`, each `flex-1 h-6 rounded-[var(--radius-sm)]`. Active swatch: `2px solid var(--ink)` ring.
4. **Density** ÔÇö segmented pill toggle (Comfortable | Compact). Same chrome as Mode.
5. **Classification** ÔÇö label `Classification` + HeroUI `Switch` (boolean on/off). Switch uses `color="primary"` which bridges to `--accent` via Phase 33 D-06.
6. **Locale** ÔÇö segmented pill toggle (EN | Ï╣). Same chrome as Mode. Click updates `id.locale`, calls `i18n.changeLanguage(lang)`, sets `<html dir/lang>` attributes (D-13).

### Keyboard flow

- Tab order: close button ÔåÆ Direction buttons (4 tabstops) ÔåÆ Mode pills (2) ÔåÆ Hue slider ÔåÆ Hue swatches (5) ÔåÆ Density pills (2) ÔåÆ Classification switch ÔåÆ Locale pills (2) ÔåÆ loop.
- Arrow keys within segmented groups (Mode/Density/Locale) ÔÇö HeroUI default behavior from React Aria; no custom wiring.
- Slider: Left/Right arrows ┬▒1┬░, Home/End = 0/360, PageUp/Down ┬▒10┬░. HeroUI Slider defaults.
- Escape dismisses drawer.

### RTL behavior checklist

- Drawer slides from inline-end: physical right in LTR, physical left in RTL. **Planner to verify HeroUI `placement` semantics** ÔÇö if props are physical-only, pass dynamically.
- All section headings, labels, taglines use `text-start` (logical) ÔÇö never `text-left`/`text-right`.
- Swatch row naturally mirrors (flexbox row follows `dir`). Swatch order `[22, 158, 190, 258, 330]` in both DOM and visual ÔÇö handoff-literal.
- Degree readout `<span dir="ltr">{hue}┬░</span>` ÔÇö degree symbol + number stay Western-numeral even in RTL (D-08).
- Direction buttons: logical text alignment (start-aligned); no rotation.
- Close `X` icon: symmetric ÔÇö no `rotate-180` needed.
- Gear icon: symmetric ÔÇö no `rotate-180` needed.

### Persistence contract

Every control writes immediately on change:

| Control        | localStorage key | Value shape                                                    |
| -------------- | ---------------- | -------------------------------------------------------------- |
| Direction      | `id.dir`         | `'chancery'` \| `'situation'` \| `'ministerial'` \| `'bureau'` |
| Mode           | `id.theme`       | `'light'` \| `'dark'`                                          |
| Hue            | `id.hue`         | stringified number `'0'` through `'360'`                       |
| Density        | `id.density`     | `'comfortable'` \| `'compact'`                                 |
| Classification | `id.classif`     | `'true'` \| `'false'`                                          |
| Locale         | `id.locale`      | `'en'` \| `'ar'`                                               |

Direction change also writes the direction's default `id.theme` + `id.hue` per D-16 map (Chancery=light/22, Situation=dark/190, Ministerial=light/158, Bureau=light/32). No toast. Live UI flip is the confirmation.
