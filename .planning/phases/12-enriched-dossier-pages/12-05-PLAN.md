**PersonMetadataCard.tsx:**

- Props: `{ dossierId: string }`
- Shows: organization affiliation, role/title, last engagement date, contact info
- Compact key-value layout, bilingual names
- Per D-10: metadata card pattern (not elaborate enrichment)

**EngagementHistoryCard.tsx:**

- Props: `{ dossierId: string }`
- Chronological timeline of engagements this person participated in
- Vertical line: `border-s-2 border-border` with dot markers (per UI-SPEC)
- Each entry: engagement name (link), date, stage at participation
- Max 5 recent, "View all engagements" link to Engagements tab

**Forum Cards (per D-09, D-10 compact pattern):**

**ForumMetadataCard.tsx:**

- Props: `{ dossierId: string }`
- Shows: forum type, frequency, host organization, participant count
- Compact metadata layout
- Per D-10: metadata card + activity feed, no filler

**ForumSessionsCard.tsx:**

- Props: `{ dossierId: string }`
- Shows recent/upcoming forum sessions (from Phase 9 lifecycle)
- Each session: name, date, lifecycle stage badge
- Compact list, max 3-5 sessions

- Shows: office title (bilingual), party (bilingual), constituency (bilingual), term dates, term status

- Layout: key-value pairs in card container

**ElectedOfficialCommitteesCard.tsx:**

- Props: `{ dossierId: string }`
