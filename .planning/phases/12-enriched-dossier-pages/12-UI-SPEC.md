## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value                 | Usage                                                                      |
| ----- | --------------------- | -------------------------------------------------------------------------- |
| xs    | 4px (`gap-1`)         | Icon gaps, inline badge padding                                            |
| sm    | 8px (`gap-2`)         | Compact element spacing, icon-to-text gap in tabs                          |
| md    | 16px (`gap-4`, `p-4`) | Default element spacing, card internal padding, tab nav horizontal padding |
| lg    | 24px (`gap-6`, `p-6`) | Section padding on sm+ breakpoints, card grid gap                          |
| xl    | 32px (`gap-8`, `p-8`) | Layout gaps on lg+ breakpoints                                             |
| 2xl   | 48px (`gap-12`)       | Major section breaks (not used in this phase)                              |
| 3xl   | 64px (`gap-16`)       | Page-level spacing (not used in this phase)                                |

Exceptions:

- **Touch targets:** 44px minimum (`min-h-11 min-w-11`) for all interactive elements (tab links, sidebar toggle, quick-add button, tier collapse toggles, mobile relationships trigger).
- **Sidebar width:** 280px expanded, 48px collapsed (icon strip). These are not spacing tokens but fixed layout dimensions.
- **Sticky header height:** ~57px (matches WorkspaceShell: `py-3` + content). Tab nav offset: `top-[57px]`.

**Source:** CLAUDE.md mobile-first rules, WorkspaceShell.tsx pattern (`py-3`, `px-4 sm:px-6 lg:px-8`).

## Typography

This phase uses the project's existing compact type scale. Phase 12 declares **4 size roles**:

| Role                 | Size Token            | Pixel   | Weight Token    | Weight | Line Height            |
| -------------------- | --------------------- | ------- | --------------- | ------ | ---------------------- |
| Body                 | `text-base`           | 13px    | `font-normal`   | 400    | `leading-normal` (1.5) |
| Label / Tab          | `text-sm`             | 12px    | `font-medium`   | 500    | `leading-snug` (1.375) |

**Source:** `frontend/src/index.css` `--text-*` tokens, `frontend/tailwind.config.ts` fontSize map, DossierDetailLayout.tsx existing heading classes.

## Color

This phase uses the project's semantic token system. Colors adapt to all 4 themes (Canvas, Azure, Lavender, BlueSky) and dark mode automatically via CSS custom properties.

## Interaction Contracts

### DossierShell Header

| Interaction                           | Behavior                                                                                                                                                                                            |
| ------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Scroll down                           | Header remains sticky at `top-0`, tab nav sticky at `top-[57px]`. Both use `backdrop-blur` transparency.                                                                                            |
| Click dossier title breadcrumb        | Navigate to dossier list page for that type                                                                                                                                                         |
| Click sidebar toggle (desktop)        | Animate sidebar width from 280px to 48px (or reverse). Use `transition-all duration-[250ms]` (matches `--transition-base`). Persist open/closed state in `localStorage` key `dossier-sidebar-open`. |
| Click "Relationships" button (mobile) | Open bottom sheet with RelationshipSidebar content. Sheet initially at 50vh, draggable to 100vh.                                                                                                    |

### DossierTabNav

| Interaction                  | Behavior                                                                                                             |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| Click tab                    | Navigate to nested route (e.g., `/dossiers/countries/$id/tasks`). URL updates, browser back/forward works.           |
| Active tab off-screen        | `scrollIntoView({ behavior: 'smooth', inline: 'center' })` on mount and tab change. Matches WorkspaceTabNav pattern. |
| Horizontal overflow (mobile) | `overflow-x-auto snap-x snap-mandatory scrollbar-none`. Each tab is `snap-center`.                                   |

**Tab Order (all types):**

1. Overview
2. Engagements
3. Docs
4. Tasks
5. Timeline
6. Audit
7. [Type-specific extra tabs: Positions / MoUs / Committees]

### RelationshipSidebar

| Interaction            | Behavior                                                                                                         |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------- |
| Click tier header      | Toggle tier section collapse with `rotate-180` chevron animation. Default: all tiers expanded.                   |
| Click linked dossier   | Navigate to that dossier's detail page (Overview tab).                                                           |
| Click [+ Link Dossier] | Open popover containing DossierSelector search input. On select: link relationship with tier/type picker inline. |
| Remove relationship    | Show `AlertDialog` confirmation: "Remove link to {dossierName}?" with destructive action button.                 |
| Collapsed icon strip   | Show dossier type icons vertically. Hover/focus: tooltip with linked dossier count. Click icon: expand sidebar.  |

### Loading States

| Component               | Skeleton Pattern                                                                |
| ----------------------- | ------------------------------------------------------------------------------- |
| DossierShell header     | `Skeleton` bar: 200px wide x 24px height for title, 100px x 16px for subtitle   |
| DossierTabNav           | 6 skeleton pills: `Skeleton` 60px x 32px each, horizontal row                   |
| Overview card grid      | 2x2 grid of `Skeleton` cards: full-width x 120px height each with `rounded-lg`  |
| RelationshipSidebar     | 3 tier groups: `Skeleton` 24px header + 3x `Skeleton` 40px list items per group |
| Elected Officials table | 5 skeleton rows: `Skeleton` full-width x 44px height each                       |

### Empty States

| Context                        | Visual                                                                                                                |
| ------------------------------ | --------------------------------------------------------------------------------------------------------------------- |
| Overview cards (no data)       | Card shell renders with `text-muted-foreground text-sm text-center py-8` message inside. No illustration.             |
| RelationshipSidebar (no links) | `text-sm text-muted-foreground text-center py-6` message + prominent [+ Link Dossier] button below.                   |
| Tab content (no items)         | Centered icon (muted, 48px) + heading + body text + optional CTA button. Consistent across all tabs.                  |
| Elected Officials list (empty) | Full-page empty state: icon + heading + "No elected officials have been added yet" + [+ Add Elected Official] button. |

## Responsive Breakpoints

| Breakpoint     | Layout Behavior                                                                                                                                                                         |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Base (0-639px) | Single column cards. Full-width main content. No sidebar visible. "Relationships" button in header triggers bottom sheet. Tab nav horizontal scroll. Hamburger menu for nav.            |
| `sm` (640px+)  | Cards remain single column. Slightly wider padding (`px-6`). Header actions show text labels alongside icons.                                                                           |
| `md` (768px+)  | Card grid becomes 2 columns (`grid-cols-2`). Tab nav may still scroll. No sidebar yet.                                                                                                  |
| `lg` (1024px+) | RelationshipSidebar appears (`lg:flex`). Main content area shrinks to accommodate sidebar. "Relationships" header button hides. Full tab labels visible without scroll on most screens. |
| `xl` (1280px+) | No additional layout changes. More breathing room.                                                                                                                                      |

**Source:** CLAUDE.md mobile-first mandate, D-05 (below lg = bottom sheet), D-04 (desktop = icon toggle).

## RTL Contract

All components in this phase MUST follow these RTL rules:

| Rule                   | Implementation                                                                                                                                    |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| DossierShell root      | `dir={direction}` via `useDirection()` hook                                                                                                       |
| Sidebar position       | DOM order: main content FIRST, sidebar LAST. In RTL flex-row, sidebar naturally moves to LEFT side. Do NOT use physical `right/left` positioning. |
| Tab nav text           | `text-start` alignment. No `text-right` or `text-left`.                                                                                           |
| Breadcrumb chevrons    | `ChevronRight` with `className={isRTL ? 'rotate-180' : ''}` (matches existing DossierDetailLayout pattern)                                        |
| Spacing                | Only logical properties: `ms-*`, `me-*`, `ps-*`, `pe-*`, `border-s`, `border-e`, `rounded-s-*`, `rounded-e-*`                                     |
| Timeline vertical line | `border-s-2` (renders on RIGHT in RTL, LEFT in LTR)                                                                                               |
| Sidebar collapse icon  | PanelRightClose/PanelRightOpen swap based on `isRTL`                                                                                              |

## Accessibility Contract

| Requirement             | Implementation                                                                                                               |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| Tab navigation          | Tab links use `<Link>` with `aria-current="page"` on active tab. Focus ring: `focus-visible:ring-2 focus-visible:ring-ring`. |
| Sidebar toggle          | `aria-expanded={sidebarOpen}`, `aria-label="Toggle relationship sidebar"`.                                                   |
| Tier collapse           | `aria-expanded` on tier header button. Content region has `role="region"` with `aria-labelledby` pointing to tier heading.   |
| Bottom sheet            | Uses existing sheet component with proper focus trap, `aria-modal="true"`, `role="dialog"`.                                  |
| Elected Officials table | `role="table"` with `aria-label`. Sortable columns use `aria-sort`.                                                          |
| Color contrast          | All text meets WCAG AA 4.5:1 against its background. Ensured by existing `--base-400/500` adjustments in CSS tokens.         |
| Keyboard                | All interactive elements reachable via Tab. Escape closes bottom sheet and popovers. Arrow keys navigate within tab bar.     |
