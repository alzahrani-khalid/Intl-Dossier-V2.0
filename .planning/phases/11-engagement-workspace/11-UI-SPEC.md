## Interaction Contracts

### I-01: Tab Navigation

- **Trigger:** Click or tap any tab link in `WorkspaceTabNav`
- **Behavior:** TanStack Router navigates to nested route (`/engagements/$id/{tab}`)
- **Active state:** 2px bottom border in `primary` color, `text-primary`, `bg-background`
- **Inactive state:** `text-muted-foreground`, on hover `text-foreground`
- **Mobile:** Horizontal scroll container with `overflow-x-auto`, `snap-x snap-mandatory`, active tab auto-scrolls into view via `scrollIntoView({ behavior: 'smooth', inline: 'center' })`
- **Desktop:** All 6 tabs visible in a single row, no scroll needed
- **Sticky behavior:** Tab nav sticks below the LifecycleBar during scroll
- **Touch target:** Each tab link is `min-h-11 min-w-11` with `px-3 py-2`
- **RTL:** Uses `useDirection()` hook, `dir` attribute set on container, logical properties for padding (`ps-*`, `pe-*`)

### I-02: Lifecycle Stage Popover (D-05)

- **Trigger:** Click a completed stage chip in LifecycleBar
- **Content:** Who transitioned, when (formatted date), transition note (if any), time spent in stage
- **Behavior:** Radix `Popover` opens below the stage chip, `w-64 p-3`, dismisses on outside click or Escape
- **Typography:** Stage name in `text-sm font-medium`, metadata in `text-xs text-muted-foreground`, note in `text-xs italic`
- **Current stage click:** No popover, stage is non-interactive (cursor-default)
- **Upcoming stage click:** Opens transition confirmation dialog (I-03)

### I-03: Lifecycle Stage Transition (D-06)

- **Trigger:** Click any upcoming or completed (backward) stage chip
- **Adjacent transition (1 step):** Immediate transition, no dialog
- **Non-adjacent transition (2+ steps):** Inline note prompt appears below stepper with textarea + Cancel/Confirm buttons
- **Confirm button:** `variant="default"`, calls `useLifecycleTransition` mutation
- **Cancel:** `variant="ghost"`, closes prompt, Escape key also cancels
- **Toast:** Success toast via sonner on transition complete
- **Loading:** Confirm button shows spinner during mutation

### I-04: Kanban Drag-and-Drop (Tasks Tab)

- **Trigger:** Drag a work item card between columns
- **Behavior:** Uses `@dnd-kit/core` via existing `useEngagementKanban.handleDragEnd`
- **Visual feedback:** Dragging card has elevated shadow (`shadow-lg`), drop zone highlights with `border-primary/50 border-dashed`
- **Columns:** Grouped by lifecycle stage (Decision D-06 in REQUIREMENTS), or by workflow status (todo, in_progress, review, done)
- **Mobile:** Columns stack vertically in single-column view, drag disabled on touch (tap to move via dropdown)

### I-05: Generate Briefing (Docs Tab)

- **Trigger:** Click "Generate Briefing" button in Docs tab header
- **Behavior:** Calls `useGenerateEngagementBrief` mutation
- **Loading state:** Button shows spinner, disabled during generation
- **Success:** New brief appears in list, success toast
- **Error:** Error toast with retry suggestion

### I-06: After-Action Route

- **URL:** `/engagements/$id/after-action`
- **Behavior:** Renders within WorkspaceShell but **hides tab navigation** (full-page form). LifecycleBar remains visible for context.
- **Detection:** WorkspaceShell checks if current child route is `after-action` and conditionally renders `WorkspaceTabNav`

## Layout Contracts

### L-01: WorkspaceShell Structure

```
+--------------------------------------------------+
| Header: Engagement Name + Type Badge + Actions    |  sticky top-0 z-20
+--------------------------------------------------+
| LifecycleBar (LtrIsolate wrapper)                 |  sticky below header z-15
+--------------------------------------------------+
```

## RTL Considerations

| Element            | RTL Behavior                                                                                                                       |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------- |
| LifecycleBar       | Wrapped in `LtrIsolate` -- always renders left-to-right (progress flows LTR in both languages)                                     |
| Tab Navigation     | Follows document direction via `dir` attribute from `useDirection()`. First tab (Overview) appears at reading start (right in RTL) |
| Kanban Columns     | Follow document direction. First column (todo) appears at reading start                                                            |
| Tab content text   | All text uses `writingDirection` from document direction. No explicit `text-right`                                                 |
| Logical properties | All margin/padding use `ms-*`, `me-*`, `ps-*`, `pe-*`. No physical `ml-*`, `mr-*`                                                  |
| Directional icons  | Back chevron: `chevron-right` in RTL (points right = "return"). Forward chevron: `chevron-left` in RTL (points left = "go deeper") |

## Accessibility

| Requirement         | Implementation                                                                                                     |
| ------------------- | ------------------------------------------------------------------------------------------------------------------ |
| Tab navigation      | `role="tablist"` on tab container, each tab link has `aria-selected` based on active route match                   |
| Lifecycle stepper   | Existing `role="group"` with `aria-label`, `aria-current="step"` on current stage (already in LifecycleStepperBar) |
| Keyboard navigation | Arrow keys navigate between tabs, Enter activates, Escape closes popovers                                          |
| Screen reader       | Stage popovers use `aria-describedby` linking trigger to content                                                   |
| Focus management    | Auto-focus on first interactive element when tab content loads                                                     |
| Reduced motion      | All animations wrapped in `motion-safe:` prefix (existing pattern)                                                 |
