    Verify the row wrapper is a SINGLE interactive element. Two acceptable shapes:

    **Option A — `<button>` row (preferred when navigation is purely programmatic):**
    ```tsx
    &lt;button
      type="button"
      onClick={(): void =&gt; onEngagementClick?.(row)}
      onKeyDown={(e): void =&gt; { if (e.key === 'Enter' || e.key === ' ') { e.preventDefault(); onEngagementClick?.(row); } }}
      className="flex w-full items-center gap-3 rounded-lg p-3 text-start min-h-11 hover:bg-muted/50 focus-visible:ring-2 focus-visible:ring-ring"
      data-testid="engagement-row"
    &gt;
      {/* row content */}
    &lt;/button&gt;
    ```

    **Option B — `<Link>` row (if TanStack Router `Link` is already imported):**
    ```tsx
    &lt;Link
      to={routePath}  // e.g. "/dossiers/engagements/$engagementId/overview"
      params={{ engagementId: row.id }}
      className="flex w-full items-center gap-3 rounded-lg p-3 text-start min-h-11 ..."
      data-testid="engagement-row"
    &gt;
      {/* row content */}
    &lt;/Link&gt;
    ```

    Required attributes regardless of choice:
    - `min-h-11` (44px min touch target — project mandate)
    - `data-testid="engagement-row"` so the spec can target rows reliably
    - `text-start` (NOT `text-left` — RTL-safe per CLAUDE.md project rules)
    - One of: `onClick` + `onKeyDown(Enter|Space)` for `<button>`, OR navigation `to` for `<Link>`
    - No nested interactive children inside the row (e.g., the chevron must be `aria-hidden="true"` and NOT a button)

    If the current implementation has a CHILD `<button>` inside a `<div>` row, refactor so the OUTER row is the interactive element and the child becomes a presentational `<span>` with `aria-hidden`. Otherwise the child captures clicks and the outer row's handler never fires.
