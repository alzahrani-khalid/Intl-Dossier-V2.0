## Token mapping rules

Apply the token mapping table at `58-01-PLAN.md:110-127` verbatim (Pattern 1). Apply Wave 2 additions at `58-02-PLAN.md:80-120` verbatim. Apply the Wave 1 opacity-step pattern at `58-01-PLAN.md:128-142` verbatim for NotificationsWidget.tsx (identical 8-category vocabulary to NotificationPreviewTimeline).

Wave-5-specific decisions (per RESEARCH section 6):

- **SLA tone palette (3 stops: ok / risk / bad)** — files: `SLACountdown`, `SLAAtRiskList`, `SLAEscalationsList`, `SLAPolicyForm`, `SLAComplianceTable`. Map: ok → `text-sla-ok` / `bg-sla-ok/10`; risk → `text-sla-risk` / `bg-sla-risk/10`; bad → `text-sla-bad` / `bg-sla-bad/10`. (Prefer the `sla-*` family over generic `success/warning/danger` for SLA-context files — the tokens exist at `index.css:76-78` for this exact purpose.)
- **KPI delta (3 stops: positive / negative / flat)** — file: `KpiWidget`. Map: up → `text-success`; down → `text-danger`; flat → `text-ink-mute`.
- **Connection state (3 stops)** — file: `RealtimeStatus`. Map: connected → `text-success`; reconnecting → `text-warning`; offline → `text-ink-mute`.
- **Scenario outcome (4 stops: success / likely / uncertain / unlikely)** — file: `OutcomeList`. Map: success → `text-success` / `bg-success/10`; likely → `text-info` / `bg-info/10`; uncertain → `text-warning` / `bg-warning/10`; unlikely → `text-danger` / `bg-danger/10`. 4 tokens for 4 stops — no opacity step.
- **Notification category (8 stops)** — file: `NotificationsWidget`. **Reuse Wave 1 NotificationPreviewTimeline opacity-step palette verbatim** (mentions/deadlines/calendar/workflow/assignments/intake/signals/system). See `58-01-PLAN.md:128-142` palette table.
- **Event type (5 stops, EventsWidget)** — Map: meeting → `text-info`; deadline → `text-warning`; milestone → `text-accent`; conflict → `text-danger`; other → `text-ink-mute`. 5 tokens — no opacity step.
- **Task priority/status (TaskListWidget)** — Apply Wave 2 RiskList 4-tier severity precedent if 4 priority stops exist (low / medium / high / critical → `success / warning / danger/80 / danger`); apply Wave 2 confidence-tier mapping if status stops exist (`pending / in_progress / review / completed / cancelled` → `info / warning / accent / success / ink-mute`).
- **Blue-purple collisions (4 files: 1, 3, 5, 6)** — Each `text-blue-*` site decides by semantic context: link/CTA → `text-accent`; informational icon → `text-info`; selected → `text-primary`. For `purple` sites, follow Wave 2 precedent: `purple → accent`.
