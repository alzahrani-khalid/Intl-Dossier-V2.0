## Token mapping rules

Apply Pattern 1 (`58-01-PLAN.md:110-127`) and Wave 2 additions (`58-02-PLAN.md:80-120`) verbatim. Apply Pattern 5 semantic-context discrimination for status enums.

Wave-6D-specific decisions:

- **Hook-returned color metadata** — when a hook returns color information consumed by a JSX component (e.g., `useWidgetDashboard().widgets[i].color`), prefer the token-mapped Tailwind class (`text-info`) if the consumer's JSX accepts a className. Use the raw `var(--info)` CSS custom property only if the consumer expects a raw CSS color string (e.g., for an inline `style={{ color: ... }}` or a charting library prop). **Verify consumer expectations before choosing** — inspect 1-2 call sites of each hook.
- **useDossierPresence (presence-state palette)** — online → `text-success` / `var(--ok)`; away → `text-warning` / `var(--warn)`; offline → `text-ink-mute` / `var(--ink-mute)`; busy → `text-danger` / `var(--danger)`.
- **useOCR (confidence-tier palette)** — high (≥0.8) → `text-success`; medium (0.6–0.8) → `text-warning`; low (<0.6) → `text-danger` (D-58-02-01 confidence-tier precedent). Per Wave 2, low-confidence is "skeptical-of-AI" — danger color reinforces "do not trust blindly."
- **useWidgetDashboard (widget-type palette)** — 22 raw hex values. Each widget type's color must map to a token in `index.css` @theme block. If a widget type has no clean semantic mapping, choose the closest by category (KPI → accent; notifications → info; tasks → warning; alerts → danger; ambient → ink-mute). Document non-obvious mappings inline as `// D-58-06D-NN: <widget-type> → <token>`.
- **useSavedSearchTemplates (40 annotations)** — saved-search-template palette likely has multiple status enums (favorite, shared, archived, system-default). Apply Pattern 5: favorite → `text-warning` (yellow-star semantic) OR `text-accent` (decided by file's actual usage); shared → `text-info`; archived → `text-ink-mute`; system-default → `text-ink-mute bg-muted`.
- **Blue-purple collision (1 file: useSavedSearchTemplates)** — per-file decision: link/CTA → `text-accent`; informational → `text-info`. Purple → `accent`.
