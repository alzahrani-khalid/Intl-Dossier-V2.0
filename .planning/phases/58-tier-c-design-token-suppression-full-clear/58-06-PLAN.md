## Token mapping rules

Apply Pattern 1 (`58-01-PLAN.md:110-127`) and Wave 2 additions (`58-02-PLAN.md:80-120`) verbatim. For raw hex literals in the 5 raw-hex files, prefer the token-mapped utility (`text-info`, `bg-accent/10`, etc.) wherever a chart library or React-Flow node prop accepts a Tailwind class; use the raw `var(--info)` / `var(--accent)` / etc. CSS custom property reference only when the consumer expects a raw CSS color string.

**Pattern 5 — semantic-context discrimination for status enums** (the key tool for Wave 6A):

| Status semantic                                                    | Token mapping                                  |
| ------------------------------------------------------------------ | ---------------------------------------------- |
| Terminal, undesirable (e.g., repealed, rejected, breach, expired)  | `bg-danger/10 text-danger border-danger/30`    |
| Terminal, neutral (e.g., archived, superseded, closed)             | `bg-muted text-ink-mute border-line`           |
| Terminal, positive (e.g., enacted, approved, completed, compliant) | `bg-success/10 text-success border-success/30` |
| In-progress, attention-needed (e.g., at-risk, pending-review)      | `bg-warning/10 text-warning border-warning/30` |
| In-progress, normal (e.g., draft, in-progress)                     | `bg-info/10 text-info border-info/30`          |
| Informational, no action needed (e.g., scheduled, queued)          | `bg-info/10 text-info border-info/30`          |
| Highlighted/featured (e.g., recommended, primary)                  | `bg-accent/10 text-accent border-accent/30`    |
| Neutral default (no semantic meaning beyond "exists")              | `bg-muted text-ink-mute border-line`           |

For status enums with >5 stops where 2+ map to the same semantic family (e.g., legislation `repealed` + `expired` both terminal-undesirable), use the Wave 1 opacity-step pattern: keep one at solid token, push the other to `/80` for distinction (e.g., `repealed → bg-danger`, `expired → bg-danger/80`).

**Specific guidance for legislation.types.ts (63 annotations — heaviest):**

- Verify the file uses 21 LegislationStatus enum values × 3 fields (bg/text/border). The executor must enumerate each status value and apply Pattern 5 per status. If the executor encounters a status name not covered by the table above (e.g., a domain-specific status), STOP and document the decision inline as `// D-58-06A-NN: <status> → <token> because <semantic-context-reason>`.
