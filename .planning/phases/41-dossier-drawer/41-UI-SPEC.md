## Spacing Scale

Declared values (multiples of 4, sourced from `colors_and_type.css#L109-119` `--space-*`):

| Token       | Value | Usage in this phase                                                                                                                         |
| ----------- | ----- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| `--space-1` | 4px   | Inline icon gaps inside CTA buttons; chip internal padding                                                                                  |
| `--space-2` | 8px   | Chip-to-chip gap in head; mini-KPI value-to-label gap                                                                                       |
| `--space-3` | 12px  | Sticky-head internal vertical rhythm; act-row internal gap                                                                                  |
| `--space-4` | 16px  | **Default element spacing** ‚Äî body section gap, kpi-mini-strip gap, mobile drawer `padding-inline` (handoff `app.css#L856`); CTA button gap |
| `--space-5` | 20px  | Bureau-density `--pad` (handoff `colors_and_type.css#L99`) ‚Äî drawer head + body padding-inline at desktop                                   |
| `--space-6` | 24px  | Section-to-section gap inside drawer body                                                                                                   |
| `--space-8` | 32px  | Drawer head bottom padding before body scroll begins                                                                                        |

**Density:** Drawer respects `--row-h` (52 / 40 / 32) and `--pad` (20 / 14 / 10) tokens for any row-height interactive element (commitment rows, upcoming rows, activity rows). Density is a global tweak (Phase 33 / 34); the drawer must not hard-code its row heights.

**Touch target exceptions (D-11):**

- Close ‚úï button: `min-block-size: 44px; min-inline-size: 44px` (overrides density)
- All CTA buttons (`Log engagement`, `Brief`, `Follow`, `Open full dossier`): `min-block-size: 44px`
- Commitment rows + upcoming rows: row hit-area `min-block-size: 44px` at all density modes (above the 32px Dense `--row-h`)
- Sticky-head chips (`DOSSIER`, `CONFIDENTIAL`): not interactive ‚Äî no touch-target floor

**Drawer chrome dimensions (verbatim from `app.css#L425-432, L855-858`):**

- Desktop width: `width: min(720px, 92vw)`
- Mobile (`max-md:` ‚â§640px): `width: 100vw; border-inline-start: 0; box-shadow: none`
- Drawer shadow (desktop only, drawer is the **only** allowed `--shadow-lg` consumer per CLAUDE.md): `box-shadow: -24px 0 60px rgba(0, 0, 0, 0.25)` exactly (handoff line 427) ‚Äî preserved in both LTR and RTL since the offset reads correctly relative to the drawer edge in both
- Backdrop: drawer-overlay class ‚Äî `rgba(0, 0, 0, 0.2)` no-blur per prototype `README.md` "Transparency & blur" rule
- Z-index stack: drawer at `z-41`, work-item detail dialog opened from commitment row at `z-50+` (D-08; drawer stays in DOM behind the dialog)

**RTL typography rules (CLAUDE.md mandate):**
