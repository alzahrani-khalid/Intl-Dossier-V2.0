# Phase 62 — UI Design Contract

> Visual and interaction contract for the export-pack phase. Generated by gsd-ui-researcher (auto mode), verified by gsd-ui-checker. Sources: 62-CONTEXT.md D-01..D-08 (locked), 62-RESEARCH.md, IntelDossier Bureau tokens (`frontend/design-system/inteldossier_handoff_design/colors_and_type.css`), existing `ExportDossierDialog.tsx`.

Three UI surfaces are in scope:

1. **ExportDossierDialog rework** — app surface, IntelDossier tokens binding (D-03, D-04, D-08)
2. **Exported HTML briefing pack** — self-contained document with its own inline stylesheet; this spec defines only the failure block and print rules, not a redesign (D-02, D-08)
3. **New-tab open flow** — placeholder state + popup-blocked fallback (D-07, discretion exercised below)

## Spacing Scale

Declared values (Bureau `--space-*` tokens):

| Token       | Value | Usage in this phase                                                                           |
| ----------- | ----- | --------------------------------------------------------------------------------------------- |
| `--space-1` | 4px   | Icon-to-text gaps inside banners                                                              |
| `--space-2` | 8px   | Gap between stacked banner lines; checkbox-to-label gap                                       |
| `--space-3` | 12px  | Compact padding (`p-3`) on the dossier info card, mobile banner padding — see exception below |
| `--space-4` | 16px  | Default padding (`p-4`) on banners and advanced-options panel; section gap inside dialog body |
| `--space-6` | 24px  | Vertical rhythm between dialog config groups (`space-y-6`, existing)                          |

Exceptions:

- **12px (`--space-3`):** outside the standard {4, 8, 16, 24, 32, 48, 64} set. It is a Bureau design-system density token (project-level addition to the standard scale), used exclusively for compact card padding (`p-3`) to match the IntelDossier density rhythm. Declared exception — do not substitute 8px or 16px where the prototype uses `--space-3`.
- The exported pack document uses print units (pt/cm) inside its own stylesheet — `@page` margin 1.5cm, body 12pt. These are document-print conventions, not app spacing tokens. Allowed.
- Failure block (`.section-error`) padding inside the pack: `16px 20px` (matches the pack's existing `.content-card` rhythm).

## Typography

App dialog (this phase's new/changed elements only — pre-existing dialog text is untouched):

| Role               | Size                       | Weight | Line Height |
| ------------------ | -------------------------- | ------ | ----------- |
| Body / banner text | 14px (`text-sm`)           | 400    | 1.5         |
| Meta / secondary   | 12px (`text-xs`)           | 400    | 1.5         |
| Group label        | 14px (`text-sm`)           | 600    | 1.5         |

## Surface 1 — ExportDossierDialog Rework Contract

File: `frontend/src/components/dossier/ExportDossierDialog.tsx`

### Structure (top to bottom)

1. **Header** — unchanged: FileText icon (`h-5 w-5 text-[var(--accent)]`) + title + description.
2. **Dossier info card** — unchanged: `rounded-[var(--radius-sm)] border border-[var(--line)] bg-[var(--line-soft)] p-3 sm:p-4`.
3. **Info line (NEW, replaces the entire format picker group):** a single flat note row. Markup contract: `Info` icon (lucide, `h-4 w-4 text-[var(--ink-mute)] shrink-0 mt-0.5`) + `format.html_info` text at `text-sm text-[var(--ink-mute)]`, laid out `flex items-start gap-2`, `text-start`. No border, no background, no accent — it is a statement of fact, not a callout. The PDF/Word radio cards, the `format` state, and the `format.label` group are deleted.
4. **Language group** — keep the existing radio-card pattern and selected/unselected styling exactly as-is, trimmed to two options: `English` / `العربية`. The `both` option and its map entry are deleted. Default: `i18n.language === 'ar' ? 'ar' : 'en'` (current UI language), set via initial state — not hardcoded `'both'`.
5. **Advanced options** — unchanged: toggle button, cover-page checkbox, TOC checkbox, section checkboxes (all sections stay per D-05).
6. **Footer** — Close (outline, label changed from `Cancel` per the copywriting contract) + Export (primary, accent). Export button content unchanged (FileDown icon / spinner + label).

**Visual anchor:** the Export button (accent primary) in the dialog footer is the primary focal point of the surface; the FileText icon in the dialog header is the secondary anchor. No other element competes for accent emphasis.

### RTL contract

- Dialog inherits `dir` from the document root; all new layout uses logical utilities (`gap`, `items-start`, `text-start`, `ms-*`/`ps-*` if needed). No `ml-/mr-/pl-/pr-/text-left/text-right`.
- The info line and warning banner icons sit at the reading start (start side) in both directions via `flex` + logical flow — no manual flipping.
- AR strings verified in the copy table; Tajawal applies automatically under `dir="rtl"`.

### Accessibility

- Warning and error banners: `role="alert"`. Success banner: `role="status"` with `aria-live="polite"`.
- Language radio group keeps existing Label/`htmlFor` wiring; deleting the third option must not break the `sr-only` RadioGroupItem pattern.
- Info line is plain text — no interactive role.

## Surface 2 — Exported HTML Pack: Failure Block + Print Rules

File: `supabase/functions/dossier-export-pack/index.ts` (inline stylesheet + section renderers). This contract does NOT redesign the pack — existing cover/TOC/section/table/timeline/contact styles are retained.

### Failure block (D-08) — `.section-error`

Rendered inside the section's normal `<div class="section">` wrapper, after the `<h2 class="section-title">`, whenever that section's data query failed. Visual contract (add to the inline stylesheet):

```css
.section-error {
  border: 1px solid #d1d5db;
  border-inline-start: 4px solid #b45309;
  background-color: #fffbeb;
  padding: 16px 20px;
  page-break-inside: avoid;
  break-inside: avoid;
  -webkit-print-color-adjust: exact;
  print-color-adjust: exact;
}
.section-error p {
  margin: 0;
  color: #92400e;
  font-size: 11pt;
  font-style: normal;
  text-align: start;
}
```

- `border-inline-start` resolves correctly under the document's `dir` attribute — no `${isRTL ? 'right' : 'left'}` interpolation needed for this block.
- Copy: `This section could not be generated.` / `تعذّر إنشاء هذا القسم.` — rendered in the pack's language (single string, matching the existing inline-bilingual renderer pattern).
- **Distinctness rule (D-08):** failure ≠ empty. `.no-data` stays italic, centered, gray `#9ca3af`, borderless. `.section-error` is start-aligned, normal style, amber-barred, boxed. A reader must be able to tell them apart at a glance and in grayscale print (the 4px bar + box carry the distinction even if backgrounds drop).
- The failed section still appears in the TOC and still occupies its page — the pack never silently omits an advertised section.

### Print rules (D-02 + discretion)

Add an `@media print` block (the existing `@page { size: A4; margin: 1.5cm }` and `.page-break` are retained):

```css
@media print {
  .section {
    break-before: page;
    page-break-before: always;
  }
  .cover-page {
    break-after: page;
    page-break-after: always;
  }
  .toc {
    break-after: page;
    page-break-after: always;
  }
  .data-table tr {
    break-inside: avoid;
    page-break-inside: avoid;
  }
  .data-table thead {
    display: table-header-group;
  }
  .content-card,
  .contact-card,
  .section-error {
    break-inside: avoid;
    page-break-inside: avoid;
  }
  .document-footer {
    position: fixed;
  }
}
```

Discretion additions beyond the D-02 list (recorded as decisions): repeat table headers across page breaks (`table-header-group`); keep cards and the failure block unsplit. Both modern `break-*` and legacy `page-break-*` properties are declared for engine coverage.

**Constraint:** when editing the stylesheet, preserve the round-fkn XSS hardening — `escapeHtml` on every user-data insertion stays; no new DB-derived interpolation enters the `<style>` block.

### Document title / save filename

`<title>Briefing Pack - {name}</title>` is retained (it seeds the browser's print/save filename). Cover page, TOC, section order, and `No X found` empty strings are all unchanged.

## Surface 3 — New-Tab Open Flow (D-07)

Files: `frontend/src/hooks/useDossierExport.ts`, `frontend/src/services/dossier-export.service.ts`, dialog click handler.

### Flow contract

1. **Synchronous open:** `window.open('', '_blank')` is called synchronously in the Export click handler, before any `await` (popup-blocker safe).
2. **Placeholder state (discretion — decided):** immediately after opening, write a minimal placeholder document into the tab so the user never stares at a blank page during generation:
   - `<title>` = `newTab.generating` string in the requesting UI language
   - Body: single centered line of the same string; background `#f7f6f4`, text `#6b6459`, `system-ui` font at 14px; `dir` set to match the UI language. No spinner, no logo, no animation — one quiet sentence.
   - This placeholder is frontend-authored (i18n `newTab.generating`), not part of the edge response.
3. **On success:** `newTab.document.open(); newTab.document.write(html); newTab.document.close()` replaces the placeholder with the pack. Dialog shows the success state (with or without the failed-sections warning per Surface 1).
4. **On popup blocked** (`window.open` returned `null`): fall back to a Blob download — `new Blob([html], { type: 'text/html; charset=utf-8' })`, anchor `download` click, `URL.revokeObjectURL`. Filename pattern (discretion — decided): `briefing-pack-{dossier-slug}-{YYYYMMDD}.html` where the slug is the lowercased, hyphenated English dossier name (non-alphanumerics collapsed to `-`). Dialog shows the `popupBlocked` notice.
5. **On error:** `newTab?.close()` — never leave an orphaned placeholder tab — then show the dialog error state.

### Failed-sections transport

Read from the `X-Failed-Sections` response header (comma-separated section type keys, e.g. `positions,mous`). Frontend maps each key through the existing `sections.*` i18n keys for the warning banner. (The edge must set `Access-Control-Expose-Headers: X-Failed-Sections` — see 62-RESEARCH.md Open Question 3.)
