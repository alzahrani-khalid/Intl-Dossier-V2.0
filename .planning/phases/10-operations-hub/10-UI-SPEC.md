## Spacing Scale

Declared values (multiples of 4, using existing Tailwind utilities):

| Token | Value | Tailwind | Usage in this phase                                                |
| ----- | ----- | -------- | ------------------------------------------------------------------ |
| xs    | 4px   | `gap-1`  | Icon-to-text gaps inside attention items, stat card icon padding   |
| sm    | 8px   | `gap-2`  | Compact spacing between badge and text, inline elements            |
| md    | 16px  | `gap-4`  | Default spacing between zone cards, padding inside zone containers |
| lg    | 24px  | `gap-6`  | Spacing between zones in the grid layout                           |
| xl    | 32px  | `gap-8`  | Page-level horizontal padding on desktop                           |
| 2xl   | 48px  | `gap-12` | Not used in this phase                                             |
| 3xl   | 64px  | `gap-16` | Not used in this phase                                             |

Exceptions:

- Touch targets: 44px minimum (`min-h-11 min-w-11`) on all interactive elements (attention items, stat cards, collapsible zone headers, action buttons)
- Mobile page padding: 16px (`px-4`) base, scaling to 24px (`px-6`) at sm, 32px (`px-8`) at lg

## Layout Contract

### Desktop (md: 768px+)

```
+----------------------------------------------------------+
| ActionBar: [Greeting + Date]     [+New Eng] [+New Req] [Cmd+K] [Role ▼] |
+----------------------------------------------------------+
| AttentionZone (full width, spans 2 cols)                 |
+----------------------------------------------------------+
| TimelineZone (1 col)       | EngagementsZone (1 col)    |
| ─────────────────────────  | ─────────────────────────  |
| Today                      | ▾ Intake (3)               |
| Tomorrow                   | ▾ Preparation (5)          |
| This Week                  | ▾ Execution (8)            |
| Next Week                  | ▸ Follow Up (2)            |
|                            | ▸ Closed (12)              |
+----------------------------------------------------------+
| QuickStatsBar (4 cards in row, full width)               |
+----------------------------------------------------------+
| ActivityFeed (full width)                                |
+----------------------------------------------------------+
```

**Grid:** `grid grid-cols-1 md:grid-cols-2 gap-6`

- ActionBar: `col-span-full` (sticky on desktop: `sticky top-0 z-10 bg-background/95 backdrop-blur`)
- AttentionZone: `col-span-full`
- TimelineZone: `col-span-1`
- EngagementsZone: `col-span-1`
- QuickStatsBar: `col-span-full`, inner grid `grid grid-cols-2 sm:grid-cols-4 gap-4`
- ActivityFeed: `col-span-full`

### Mobile (< 768px)

All zones stack vertically in priority order. Each zone wrapped in `ZoneCollapsible`:

| Zone            | Default State | Priority (Officer view) |
| --------------- | ------------- | ----------------------- |
| AttentionZone   | Expanded      | 1 (always first)        |
| TimelineZone    | Collapsed     | 2                       |
| EngagementsZone | Collapsed     | 3                       |
| QuickStatsBar   | Collapsed     | 4                       |
| ActivityFeed    | Collapsed     | 5                       |

ActionBar: Scrolls with page (not sticky). Buttons stack: greeting on one line, actions on next.

**Breakpoint decision (Claude's Discretion):** Use `md` (768px) for 2-column to 1-column transition. At `sm` (640px), two zones side by side would each be ~280px which is too narrow for the engagement stage groups and timeline event cards.

### Role-Adaptive Zone Ordering

Zone order changes per role. The component array is reordered, not hidden:

| Role       | Zone Order                                             |
| ---------- | ------------------------------------------------------ |
| Leadership | Engagements, QuickStats, Attention, Timeline, Activity |
| Officer    | Attention, Timeline, QuickStats, Engagements, Activity |
| Analyst    | Timeline, Activity, Attention, Engagements, QuickStats |

On mobile, the first zone in the role order is `defaultExpanded: true`, others `defaultExpanded: false`.

**Source:** CONTEXT.md D-09

## Interaction Contract

### Click-Through Navigation (OPS-06)

Every data item in the dashboard is clickable and navigates to its entity:

| Element                        | Click Destination                                                  |
| ------------------------------ | ------------------------------------------------------------------ |
| Attention item (work item)     | `/tasks/{id}` or `/intake/{id}` based on `entity_type`             |
| Attention item (engagement)    | `/engagements/{id}` (future: workspace in Phase 11)                |
| Timeline event card            | `/engagements/{engagement_id}`                                     |
| Engagement in stage group      | `/engagements/{id}`                                                |
| Quick Stat: Active Engagements | `/engagements` (list, filtered to active)                          |
| Quick Stat: Open Tasks         | `/tasks` (list, filtered to open)                                  |
| Quick Stat: SLA At Risk        | `/intake` (list, filtered to SLA at risk)                          |
| Quick Stat: Upcoming Week      | `/calendar` (week view)                                            |
| Activity feed item             | Entity route based on `entity_type` + `entity_id`                  |
| "Show all" (Timeline)          | Inline expand: removes max-5 limit, shows all items in section     |
| "Show all" (Engagements)       | Inline expand: removes top-5 limit, shows all engagements in stage |

### Collapsible Zones (Mobile)

- Tap zone header to toggle expand/collapse
- `motion/react` `AnimatePresence` with `m.div` for height animation (matches existing `timeline.tsx` pattern)
- Duration: `--transition-base` (250ms), ease-out
- Badge count visible on collapsed header (e.g., "Attention Needed (3)")

### Action Bar Interactions

| Element            | Action                                                                                          |
| ------------------ | ----------------------------------------------------------------------------------------------- |
| [+ New Engagement] | Calls `useWorkCreation().openPalette()` with type preset                                        |
| [+ New Request]    | Calls `useWorkCreation().openPalette()` with intake preset                                      |
| [Cmd+K]            | Opens existing Cmd+K palette (Phase 8)                                                          |
| Role dropdown      | `DropdownMenu` with 3 options: Leadership, Officer, Analyst. Selection persists to localStorage |

### Loading States

Each zone loads independently (progressive loading):

| Zone            | Loading State                                                      |
| --------------- | ------------------------------------------------------------------ |
| AttentionZone   | 3 skeleton cards (`Skeleton` with severity-colored border shimmer) |
| TimelineZone    | 2 skeleton day groups with 2 event placeholders each               |
| EngagementsZone | 3 skeleton stage group headers                                     |
| QuickStatsBar   | 4 skeleton stat cards with pulsing number placeholder              |
| ActivityFeed    | 5 skeleton activity rows                                           |

### Empty States

| Zone            | Empty State                                                         |
| --------------- | ------------------------------------------------------------------- |
| AttentionZone   | Green success banner: "All clear -- nothing needs attention" (D-03) |
| TimelineZone    | Muted text: "No upcoming events this week"                          |
| EngagementsZone | Muted text: "No active engagements" with [+ New Engagement] link    |
| QuickStatsBar   | Shows "0" for all metrics (never empty)                             |
| ActivityFeed    | Muted text: "No recent activity"                                    |

### Error States

| Zone     | Error State                                                                                                                                                                      |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Any zone | Inline error card: "Could not load {zone name}. Check your connection and try again." with [Retry] button that calls `queryClient.invalidateQueries()` for that zone's query key |

### Stalled Engagement Display (Claude's Discretion)

Use text format, not progress bar: **"18 days in Preparation"**. Rationale: A progress bar implies a known total duration, but diplomatic engagement stages have no fixed length. Text is more honest and scannable.

### Greeting Format (Claude's Discretion)

Time-of-day aware greeting using local time:

- 5:00 - 11:59 -> "Good morning"
- 12:00 - 16:59 -> "Good afternoon"
- 17:00 - 4:59 -> "Good evening"

Displayed as: `"{greeting}, {user_display_name}"` followed by date in locale format on a second line.

### Quick Stats Card Display (Claude's Discretion)

Each stat card shows: **icon + number + label**. No trend arrows (no historical data source exists yet).

| Stat               | Icon (lucide)   | Alert condition      |
| ------------------ | --------------- | -------------------- |
| Active Engagements | `Briefcase`     | none                 |
| Open Tasks         | `CheckSquare`   | none                 |
| SLA At Risk        | `AlertTriangle` | value > 0: red badge |
| Upcoming This Week | `CalendarDays`  | none                 |

## RTL Contract

All layout uses logical properties per CLAUDE.md mandates:

| Pattern           | LTR Class                 | RTL Equivalent                          | Notes                                   |
| ----------------- | ------------------------- | --------------------------------------- | --------------------------------------- |
| Page padding      | `ps-8`                    | auto-flips                              | Logical property, no override needed    |
| ActionBar layout  | `flex` + `gap`            | auto-flips                              | `justify-between` works bidirectionally |
| Text alignment    | `text-start`              | auto-flips                              | Never use `text-left` / `text-right`    |
| Margin direction  | `ms-*` / `me-*`           | auto-flips                              | Never use `ml-*` / `mr-*`               |
| Metric numbers    | Wrapped in `<LtrIsolate>` | forces LTR                              | Numbers always read left-to-right       |
| Directional icons | As-is                     | `className={isRTL ? 'rotate-180' : ''}` | Chevrons in collapsible headers         |
| Severity badge    | `ms-2`                    | auto-flips                              | Badge after text in reading order       |

Container `dir` attribute: Set `dir={isRTL ? 'rtl' : 'ltr'}` on the `OperationsHub` root container.

## Accessibility Contract

| Requirement             | Implementation                                                               |
| ----------------------- | ---------------------------------------------------------------------------- |
| Touch targets           | All clickable elements: `min-h-11 min-w-11` (44x44px)                        |
| Keyboard navigation     | All interactive items focusable via Tab, activatable via Enter/Space         |
| ARIA landmarks          | `OperationsHub`: `role="main"`. Each zone: `role="region"` with `aria-label` |
| ARIA live regions       | AttentionZone: `aria-live="polite"` for realtime updates                     |
| Color contrast          | All text meets WCAG AA 4.5:1 against its background                          |
| Severity not color-only | Each severity level has text label ("Overdue", "Due Soon") alongside color   |
| Collapsible state       | `aria-expanded` on zone headers, `aria-controls` linking to content          |
| Screen reader           | Stat cards: `aria-label="{value} {label}"` (e.g., "12 Active Engagements")   |
| Reduced motion          | Collapse animations respect `prefers-reduced-motion: reduce`                 |
