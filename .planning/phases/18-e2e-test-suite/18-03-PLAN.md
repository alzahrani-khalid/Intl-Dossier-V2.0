**Critical: @dnd-kit drag implementation** in `WorkItemKanbanPage.dragCardToColumn`:
```ts
async dragCardToColumn(cardTitle: string, targetColumn: string): Promise<void> {
  const card = this.page.getByRole('article', { name: cardTitle })
  const target = this.page.getByRole('region', { name: targetColumn })
  const cardBox = await card.boundingBox()
  const targetBox = await target.boundingBox()
  if (!cardBox || !targetBox) throw new Error('drag bounding box missing')
  // @dnd-kit needs activation distance (8px default) — small move first then large move
  await this.page.mouse.move(cardBox.x + cardBox.width / 2, cardBox.y + cardBox.height / 2)
  await this.page.mouse.down()
  await this.page.mouse.move(cardBox.x + cardBox.width / 2 + 10, cardBox.y + cardBox.height / 2 + 10, { steps: 5 })
  await this.page.mouse.move(targetBox.x + targetBox.width / 2, targetBox.y + targetBox.height / 2, { steps: 20 })
  await this.page.mouse.up()
  await this.page.waitForTimeout(200) // dnd-kit drop animation
}
```
DECISION: `locator.dragTo()` does NOT work with @dnd-kit because @dnd-kit listens for `pointermove` with a minimum activation distance, and Playwright's `dragTo` uses HTML5 drag events. Manual pointer sequence is the documented workaround. [RESEARCH.md]
