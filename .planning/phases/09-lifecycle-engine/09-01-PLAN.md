**Migration 1: `20260329000001_add_lifecycle_stage.sql`** (LIFE-01, LIFE-02, LIFE-03, per D-04, D-13)

1. Add `lifecycle_stage` column to `engagement_dossiers`:
   ```sql
   ALTER TABLE engagement_dossiers
   ADD COLUMN lifecycle_stage TEXT NOT NULL DEFAULT 'intake'
   CHECK (lifecycle_stage IN ('intake','preparation','briefing','execution','follow_up','closed'));
   ```
2. Create `lifecycle_transitions` audit table per D-13:
   ```sql
   CREATE TABLE lifecycle_transitions (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     engagement_id UUID NOT NULL REFERENCES engagement_dossiers(id) ON DELETE CASCADE,
     from_stage TEXT,
     to_stage TEXT NOT NULL CHECK (to_stage IN ('intake','preparation','briefing','execution','follow_up','closed')),
     user_id UUID NOT NULL REFERENCES auth.users(id),
     note TEXT,
     transitioned_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
     duration_in_stage_seconds INTEGER,
     created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   );
   ```
3. Create a trigger function `compute_lifecycle_duration()` that calculates `duration_in_stage_seconds` from the previous transition's `transitioned_at` for the same `engagement_id`. Attach as BEFORE INSERT trigger on `lifecycle_transitions`.
4. Create indexes: `idx_lifecycle_transitions_engagement_id` on `(engagement_id)`, `idx_lifecycle_transitions_to_stage` on `(to_stage)`, `idx_lifecycle_transitions_user_id` on `(user_id)`.
5. Enable RLS on `lifecycle_transitions`. Add policies following the `engagement_participants` pattern — join through `engagement_dossiers` to `dossiers` for org-based access check. Policy for SELECT (authenticated users in same org), INSERT (authenticated users), UPDATE/DELETE (owner or admin).
6. Add index on `engagement_dossiers(lifecycle_stage)` for filtering queries.
