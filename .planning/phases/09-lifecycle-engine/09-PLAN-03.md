  <name>Task 1: LifecycleStepperBar component</name>
  <files>frontend/src/components/engagements/LifecycleStepperBar.tsx</files>
  <action>
Create the CRM pipeline-style stepper bar per D-01 through D-05 and the UI-SPEC layout contract.

**Component structure:**

```tsx
// Props as defined in UI-SPEC
interface LifecycleStepperBarProps {
  currentStage: LifecycleStage
  transitions: LifecycleTransition[]
  onTransition: (toStage: LifecycleStage, note?: string) => void
  disabled?: boolean
}
```

**Implementation requirements:**

1. **Stage rendering:** Map over `LIFECYCLE_STAGES` array. Each stage is a `<button>` with:
   - `min-h-11 min-w-11` touch target (44px per CLAUDE.md)
   - `flex-1 min-w-[72px]` sizing
   - Stage label from `LIFECYCLE_STAGE_LABELS[stage][language]` using `useTranslation('lifecycle')` — or use `t('stages.${stage}')` from locale file
   - `aria-current="step"` on current stage per accessibility spec

2. **Stage states** (per UI-SPEC Color section):
   - **Completed** (index < currentIndex): `bg-primary/15 border border-primary-300 text-primary-700`
   - **Current** (index === currentIndex): `bg-primary/15 border-2 border-primary text-primary-600`
   - **Upcoming** (index > currentIndex): `bg-base-100 border border-base-200 text-base-400`
     Use `cva` (class-variance-authority) for variant management, following existing HeroUI wrapper patterns.

3. **Connectors** between stages: horizontal 2px lines. Completed: `bg-primary-300`. Upcoming: `bg-base-200`. Use `flex` with `gap-1` or explicit connector divs.

4. **LtrIsolate wrapper:** Entire stepper wrapped in `<LtrIsolate className="w-full">` so progress reads left-to-right in both LTR and RTL per STATE.md decision.

5. **Click handler:** On any stage button click:
   - Calculate `Math.abs(targetIndex - currentIndex)`.
   - If adjacent (diff === 1): call `onTransition(targetStage)` immediately.
   - If non-adjacent (diff > 1): open inline note prompt (see below).
   - If clicking current stage: no-op.

6. **Inline note prompt** (per D-03, UI-SPEC note prompt section):
   - Use `useState` to track `pendingStage: LifecycleStage | null`.
   - When `pendingStage` is set, render a row below the stepper with:
     - `<textarea>` with placeholder from `t('transition.notePlaceholder')`, `min-h-[60px]`, `text-sm`
     - Cancel button (ghost variant, `min-h-11`): clears pendingStage
     - Confirm button (primary variant, `min-h-11`): calls `onTransition(pendingStage, noteValue)`, clears state
   - Auto-focus textarea on open. Escape key cancels (keyboard listener).
   - Animate with `transition-all duration-200` (slide down). Respect `prefers-reduced-motion`.

7. **Tooltips on completed stages** (per D-05):
   - Wrap each completed stage button in `<Tooltip>` / `<TooltipTrigger>` / `<TooltipContent>` from `@/components/ui/tooltip`.
   - Find the transition entry where `to_stage === stage` to get the `transitioned_at` date.
   - Display: `t('stepper.entered', { date: formatDate(transition.transitioned_at) })`.
   - Use existing `formatDate` utility from engagement pages (check imports in EngagementDetailPage).

8. **"Next" suggestion chip** (per UI-SPEC interaction pattern step 6):
   - If `currentStage !== 'closed'`, show a subtle chip next to the current stage: `t('stepper.nextSuggestion', { stage: nextStageName })`.
   - Clicking the chip is equivalent to clicking the next stage button.
   - Style: `text-xs bg-primary/10 text-primary-600 rounded-full px-2 py-0.5`.

9. **Responsive** (per UI-SPEC):
   - Mobile (< sm): `overflow-x-auto` on LtrIsolate, inner row `min-w-[480px]`, `snap-x snap-mandatory` for scroll snapping. `pb-2` for scroll indicator clearance.
   - sm+: No scroll, stages flex to fill with `flex-1`.

10. **Accessibility:**
    - Container: `role="group"` with `aria-label={t('stepper.ariaLabel')}` (= "Lifecycle stages").
    - Tab through stages, Enter/Space to select.
    - `focus-visible:ring-2 focus-visible:ring-ring`.

11. **Keyboard navigation:** Standard button keyboard handling. Add `onKeyDown` for Escape in note prompt.

All text uses `useTranslation('lifecycle')` namespace. No hardcoded strings.

<task type="auto">
  <name>Task 2: LifecycleTimeline component</name>
  <files>frontend/src/components/engagements/LifecycleTimeline.tsx</files>
  <action>
Create the audit trail timeline per D-13, D-14 and the UI-SPEC LifecycleTimeline layout contract. This is a custom simple timeline — NOT the Aceternity animated Timeline from ui/timeline.tsx (per UI-SPEC note).

**Component structure:**

```tsx
interface LifecycleTimelineProps {
  transitions: LifecycleTransition[]
  isLoading?: boolean
}
```

**Implementation requirements:**

1. **Collapsible container:**
   - Use `Collapsible` / `CollapsibleTrigger` / `CollapsibleContent` from `@/components/ui/collapsible`.
   - Heading: `t('timeline.heading')` (= "Lifecycle History") with `ChevronDown`/`ChevronUp` icon (direction-neutral per UI-SPEC RTL table).
   - Default state: collapsed on mobile (`< md`), expanded on `md+`. Use a media query or `useMediaQuery` hook. Set initial `open` state based on screen width.

2. **Timeline entries** (reverse chronological — most recent at top):
   - Sort `transitions` by `transitioned_at` descending.
   - Each entry layout:
     ```
     [dot] [Stage: to_stage label]  [user_name]  [formatted date]
     |     [Note: transition.note]  (if non-null, italic text-base-500)
     |     [Duration badge: "3d 4h in {from_stage}"]  (if duration_in_stage_seconds)
     ```

3. **Timeline dot styling:**
   - 8px circle (`w-2 h-2 rounded-full`).
   - Most recent (index 0): `bg-primary-500`.
   - Others: `bg-base-300`.
   - Connected by 1px vertical line: `border-s border-base-200` (logical property for RTL).

4. **Duration badge:**
   - Create a helper function `formatDuration(seconds: number): string` that returns human-readable duration:
     - `< 60`: `"<1m"`
     - `< 3600`: `"{m}m"`
     - `< 86400`: `"{h}h {m}m"`
     - `>= 86400`: `"{d}d {h}h"`
   - Badge style: `text-xs bg-base-100 rounded-full px-2 py-0.5`.
   - Text: `t('timeline.duration', { duration: formatDuration(secs), stage: stageLabel })`.
   - Only show when `duration_in_stage_seconds` is non-null and `from_stage` is non-null.

5. **Note display:**
   - Only rendered when `transition.note` is non-null and non-empty.
   - Style: `text-sm text-base-500 italic`.

6. **Initial stage entry:**
   - The first transition (last in sorted order) typically has `from_stage: null`.
   - Display note as `t('timeline.initial')` (= "Initial stage").

7. **Empty state:**
   - If `transitions` is empty and not loading: display `t('timeline.empty')` in `text-sm text-base-400`.

8. **Loading state:**
   - If `isLoading`: show 3 skeleton pulse lines using `Skeleton` component or plain `animate-pulse` divs.

9. **RTL:**
   - Timeline dot and line on `start` side using `ms-*` (margin-start).
   - Text flows naturally with `text-start`.
   - No `ml-*`, `mr-*`, `pl-*`, `pr-*` — all logical properties.

10. **Typography** per UI-SPEC:
    - Entry heading: `text-sm font-semibold` (14px)
    - Entry body/note: `text-xs` (12px)
    - Duration badge: `text-xs` (11px)

All text uses `useTranslation('lifecycle')` namespace.
