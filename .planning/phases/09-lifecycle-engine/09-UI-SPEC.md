Uses project Canvas theme (Eucalyptus, hue 155) CSS custom properties. No new colors introduced.

| Role            | Value                                            | Usage in this phase                                                 |
| --------------- | ------------------------------------------------ | ------------------------------------------------------------------- |
| Dominant (60%)  | `--base-50` / `hsl(0 0% 98%)`                    | Page background, dialog background                                  |
| Secondary (30%) | `--base-100` / `hsl(0 0% 96.1%)`                 | Stepper inactive stages, timeline panel background                  |
| Accent (10%)    | `--primary-500` / `hsl(154 51.6% 47.1%)`         | Current stage highlight, active stepper border, primary CTA buttons |

### Stage Indicator Colors (semantic progression)

Single accent progression using primary color with opacity tiers (not per-stage hues). This keeps the palette minimal and theme-consistent.

| Stage State | Background                           | Border                    | Text          |
| ----------- | ------------------------------------ | ------------------------- | ------------- |
| Completed   | `primary-500/15` (green 15% opacity) | `primary-300`             | `primary-700` |
| Current     | `primary-500/15`                     | `primary-500` (2px solid) | `primary-600` |
| Upcoming    | `base-100`                           | `base-200`                | `base-400`    |

Accent reserved for: current lifecycle stage indicator border, "Advance Stage" suggestion chip, primary CTA buttons in promotion dialog, active timeline dot.

## Component Inventory

### 1. LifecycleStepperBar

**Requirement:** LIFE-01, LIFE-02, LIFE-03 (D-01 through D-05)
**File:** `frontend/src/components/engagements/LifecycleStepperBar.tsx`
**Reuses:** `LtrIsolate`, `Tooltip`/`TooltipTrigger`/`TooltipContent` from `@/components/ui/tooltip`

#### Layout

```
Desktop (sm+):
+--[LtrIsolate dir="ltr" w-full]-------------------------------------------+
|  [intake] --[connector]-- [preparation] --[connector]-- [briefing] ...   |
|  (each stage is a button, flex-1, min-w-[72px], min-h-11)               |
+--------------------------------------------------------------------------+

Mobile (< sm):
+--[LtrIsolate dir="ltr" w-full overflow-x-auto]--+
|  [scrollable row, min-w-[480px], snap-x]         |
|  [intake] -- [preparation] -- ... -- [closed]    |
+--------------------------------------------------+
```

#### Props

```typescript
interface LifecycleStepperBarProps {
  currentStage: LifecycleStage
  transitions: LifecycleTransition[]
  onTransition: (toStage: LifecycleStage, note?: string) => void
  disabled?: boolean
}
```

#### Behavior

- **Click any stage:** Triggers `onTransition(targetStage)`. If target is non-adjacent (Math.abs(index diff) > 1), show inline note textarea below the stepper (collapse after submit or cancel). Note is optional per D-03.
- **Hover/tap completed stage:** Tooltip shows "Entered: {formatted date}" per D-05. Uses existing `formatDate` utility from engagement pages.
- **Current stage:** Highlighted with `border-2 border-primary` and `bg-primary/15`.
- **Completed stages:** `bg-primary/15 border border-primary-300 text-primary-700`.
- **Upcoming stages:** `bg-base-100 border border-base-200 text-base-400`.
- **Connectors:** Horizontal line segments between stages. Completed connectors use `bg-primary-300`. Upcoming use `bg-base-200`. Height: 2px. Width: 8px (`gap-1` handles this naturally with flex).
- **Keyboard:** Tab through stages, Enter/Space to select. `focus-visible:ring-2 focus-visible:ring-ring`.
- **ARIA:** `role="group"` on container with `aria-label="Lifecycle stages"`. Each button has `aria-current="step"` on current stage.

#### Responsive

- **Mobile (base):** Horizontal scroll with `overflow-x-auto`, `min-w-[480px]` inner row, `snap-x snap-mandatory` for scroll snapping. `pb-2` for scroll indicator clearance. Labels remain full text at `--text-xs` (11px).
- **sm+:** No scroll needed, stages flex to fill width with `flex-1`.
- **RTL:** Entire bar wrapped in `LtrIsolate` -- progress reads left-to-right in all languages per STATE.md decision. Stage labels use bilingual `LIFECYCLE_STAGE_LABELS[stage]` based on current language.

#### Inline Note Prompt (non-adjacent transitions)

When user clicks a non-adjacent stage, a collapsible row appears below the stepper:

```
+--[note prompt row, animate-in slide-down]----------------------------+
|  [Textarea: "Optional note for skipping stages..." min-h-[60px]]    |
|  [Cancel (ghost)]  [Confirm Transition (primary)]                    |
+----------------------------------------------------------------------+
```

- Textarea: `text-sm`, placeholder bilingual, not required.
- Buttons: `min-h-11`, gap-2. Cancel = ghost variant. Confirm = primary variant.
- Auto-focuses textarea on open. Escape key cancels.

### 2. IntakePromotionDialog

**Requirement:** LIFE-04 (D-06, D-07, D-08)
**File:** `frontend/src/components/engagements/IntakePromotionDialog.tsx`
**Reuses:** `AlertDialog` from `@/components/ui/alert-dialog`, form inputs from existing form patterns

#### Layout (Modal -- not slide-over)

Decision: Modal dialog is preferred over slide-over. The promotion action is a focused, single-purpose operation with a clear confirm/cancel flow. Modal prevents accidental interaction with the background, matching the existing archive AlertDialog pattern on the engagement detail page.

```
+--[AlertDialog overlay bg-black/80]----------------------------------+
|  +--[AlertDialog content, max-w-lg, mx-4]-------------------------+ |
|  |  [Title: "Promote to Engagement"]                              | |
|  |  [Description: "Review mapped fields before creating..."]      | |
|  |                                                                 | |
|  |  --- Field Preview Section ---                                  | |
|  |  [Title (en): ____editable____]                                | |
|  |  [Title (ar): ____editable____]                                | |
|  |  [Objectives: ____mapped from description, editable____]       | |
|  |  [Engagement Type: ____select____]                             | |
|  |  [Category: ____select____]                                    | |
|  |  [Linked Dossiers: ____chips from intake ticket____]           | |
|  |                                                                 | |
|  |  [Cancel (outline)]          [Promote to Engagement (primary)] | |
|  +----------------------------------------------------------------+ |
+----------------------------------------------------------------------+
```

#### Props

```typescript
interface IntakePromotionDialogProps {
  ticket: IntakeTicket
  open: boolean
  onOpenChange: (open: boolean) => void
  onPromote: (data: IntakePromotionRequest) => void
  isPending?: boolean
}
```

#### Behavior

- **Trigger:** "Promote to Engagement" button on intake ticket detail. Only visible for `request_type === 'engagement'` tickets with `status !== 'converted'` per D-08.
- **Field mapping:** Title copies from ticket title. Description maps to objectives. Dossier links carry over. All fields are editable before confirmation.
- **Engagement type + category:** Required selects. No default -- user must choose.
- **On confirm:** Calls `promoteIntakeToEngagement()`. Shows loading spinner on button. On success, closes dialog, navigates to new engagement.
- **On success (source ticket):** Ticket status becomes `converted`. Page shows read-only banner: "This request was promoted to an engagement" with link to the new engagement.
- **Responsive:** `max-w-lg` on sm+, full-width with `mx-4` on mobile. Scrollable content if fields overflow viewport.

#### Converted Ticket Banner

Shown on intake ticket detail when `ticket.convertedToId` is set:

```
+--[Banner: bg-primary/10 border-s-4 border-primary p-4 rounded-e-md]--+
|  [CheckCircle icon]  This request was promoted to an engagement.      |
|  [View Engagement ->]                                                  |
+------------------------------------------------------------------------+
```

- Uses logical properties (`border-s-4`, `rounded-e-md`, `ps-4`) for RTL.
- Link navigates to `/engagements/{convertedToId}`.

### 3. LifecycleTimeline

**Requirement:** LIFE-06 (D-13, D-14)
**File:** `frontend/src/components/engagements/LifecycleTimeline.tsx`
**Note:** Custom simple timeline component -- NOT the Aceternity animated `Timeline` from `ui/timeline.tsx` (that component is scroll-driven and designed for marketing pages, not data display).

#### Layout

```
+--[Collapsible panel: "Lifecycle History" heading with chevron]--------+
|  +--[Vertical timeline]--------------------------------------------+ |
|  |  o  [Stage: Execution]  [User name]  [2026-03-28 14:30]        | |
|  |  |  [Note: "Skipped briefing -- pre-existing brief available"]  | |
|  |  |  [Duration badge: "3d 4h in Preparation"]                    | |
|  |  |                                                               | |
|  |  o  [Stage: Preparation]  [User name]  [2026-03-25 10:00]      | |
|  |  |  [Duration badge: "1d 2h in Intake"]                         | |
|  |  |                                                               | |
|  |  o  [Stage: Intake]  [System]  [2026-03-24 08:00]              | |
|  |     [Initial stage]                                              | |
|  +------------------------------------------------------------------+ |
+------------------------------------------------------------------------+
```

#### Props

```typescript
interface LifecycleTimelineProps {
  transitions: LifecycleTransition[]
  isLoading?: boolean
}
```

#### Behavior

- **Collapsible:** Default collapsed on mobile, expanded on md+. Uses `Collapsible` from `@/components/ui/collapsible`.
- **Order:** Most recent transition at top (reverse chronological).
- **Timeline dot:** 8px circle. Active (most recent) uses `bg-primary-500`. Others use `bg-base-300`. Connected by 1px `bg-base-200` vertical line.
- **Duration badge:** Inline chip showing human-readable duration ("3d 4h", "45m"). Uses `--text-xs`, `bg-base-100`, `rounded-full`, `px-2 py-0.5`.
- **Note:** Displayed below the entry heading in `--text-sm text-base-500` italic. Only shown if `transition.note` is non-null.
- **Empty state:** "No transitions recorded yet." in `--text-sm text-base-400`.
- **RTL:** Timeline renders with logical properties. Dot and line on `start` side. Text flows naturally with `text-start`.

## Interaction Patterns

### Stage Transition Flow

1. User sees stepper bar at top of engagement detail page (below header, above tabs).
2. User clicks a stage button.
3. If adjacent (1 step forward or back): immediate transition via `onTransition(stage)`. Optimistic UI update on stepper. Toast confirmation.
4. If non-adjacent (skip or jump): inline note prompt appears below stepper with optional textarea. User can type a note or leave blank. "Confirm Transition" executes, "Cancel" dismisses.
5. Tooltip on completed stages shows entry date on hover/long-press.
6. "Next: {stage}" suggestion chip appears next to current stage (subtle, not blocking). Clicking it is equivalent to clicking the next stage button.

### Intake Promotion Flow

1. User views an engagement-type intake ticket.
2. "Promote to Engagement" button visible in ticket actions area.
3. Click opens modal with pre-mapped fields (all editable).
4. User selects required engagement_type and category.
5. "Create Engagement" confirms. Loading state on button.
6. On success: dialog closes, redirect to new engagement at intake stage.
7. Source ticket: status becomes `converted`, banner replaces actions.

### Forum Session Creation Flow

1. User views a forum dossier detail page.
2. "New Session" button in the sessions section header.
3. Click opens side sheet with pre-filled fields.
4. User adjusts dates, title, description.
5. "Create Session" confirms. Loading state on button.
6. On success: sheet closes, session list refreshes with new entry.

## RTL Considerations

| Concern                      | Solution                                                                          |
| ---------------------------- | --------------------------------------------------------------------------------- |
| Stepper direction            | Wrapped in `LtrIsolate` -- always LTR (intake on left, closed on right)           |
| Stepper labels               | Bilingual via `LIFECYCLE_STAGE_LABELS[stage].ar` / `.en` based on `i18n.language` |
| Timeline dot position        | Use `ms-*` (margin-start) not `ml-*`. Dot on start side.                          |
| Converted banner border      | `border-s-4` not `border-l-4`                                                     |
| Dialog/Sheet text alignment  | `text-start` not `text-left`                                                      |
| Sheet slide direction        | Sheet component handles `side="end"` which is logical                             |
| All spacing                  | `ms-*`, `me-*`, `ps-*`, `pe-*` only. Never `ml-*`, `mr-*`, `pl-*`, `pr-*`         |
| Chevron icons in collapsible | Use `ChevronDown`/`ChevronUp` (vertical = direction-neutral)                      |
| No `.reverse()` on arrays    | Natural order everywhere. `LtrIsolate` handles stepper direction.                 |
| No `textAlign: "right"`      | Use `text-start` or `writingDirection: "rtl"` only                                |
