## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value               | Usage                                                            |
| ----- | ------------------- | ---------------------------------------------------------------- |
| xs    | 4px (`--space-1`)   | Icon gaps, badge inner padding, row divider margin               |
| sm    | 8px (`--space-2`)   | Inline element spacing, chip horizontal padding                  |
| md    | 16px (`--space-4`)  | Default element spacing, form field gap, card inner padding      |
| lg    | 24px (`--space-6`)  | Section padding, tab content top margin                          |
| xl    | 32px (`--space-8`)  | Card group gaps, drawer section separation                       |
| 2xl   | 48px (`--space-12`) | Major section breaks within a page                               |
| 3xl   | 64px                | Page-level vertical breathing (top of Digests tab to first card) |

Exceptions: 12px (`--space-3`) and 20px (`--pad`, comfortable-density card padding) are existing density-system tokens reused as-is — both multiples of 4, inherited from the prototype, not newly declared. The density-aware row height `--row-h: 52px` (comfortable) governs digest-list and alert-list row heights — also inherited, not a new declaration.

| Destructive     | `var(--danger)`                        | `oklch(52% 0.18 25)` | Unsubscribe confirmation and Delete alert rule confirmation only                                            |

**Accent reserved for (explicit list — not all interactive elements):**

1. The "Subscribe to digest" primary button (on dossier page and in Digests tab empty state)
2. The "Add alert" primary button (on Alerts tab empty state)
3. The "Generate digest now" primary button (on Digests tab and dossier page)
4. The "Publish digest" primary button in the HITL preview drawer/modal
5. The active tab indicator underline on the intelligence hub tab strip (Digests / Alerts / Signals / Reports)
6. Focus rings on interactive elements (`outline: 2px solid var(--accent)`)

**Severity / semantic data colors** — used in digest rollup severity badges and alert-rule severity filter badges. These are **semantic data colors, not decorative accent**, and do not count against the 10% accent budget:

- `var(--danger)` / `var(--danger-soft)` — `high` and `urgent` severity signal badges in the digest rollup
- `var(--warn)` / `var(--warn-soft)` — `medium` severity badges
- `var(--ok)` / `var(--ok-soft)` — `low` severity badges
- `var(--info)` / `var(--info-soft)` — informational badges (e.g. `new_signal` condition type chip)

**All colors MUST resolve to `var(--*)` tokens.** No raw hex. No Tailwind color literals (`text-blue-500`, `bg-red-500`). No card drop-shadows — borders only (`1px solid var(--line)`). Shadow reserved for drawers and preview modals (`var(--shadow-lg)`).

## Phase 70 Surfaces — Interaction Contracts

### Surface 1: `/intelligence` hub — Digests tab and Alerts tab

**Context:** P69 shipped Signals + Reports tabs. P70 adds Digests + Alerts as adjacent tabs in the same hub. Tab strip order (EN, LTR): Reports | Signals | **Digests** | **Alerts**. (RTL mirrors: tab strip renders in reverse physical order; logical/reading order is preserved via `dir="rtl"`.)

**Tab strip:** Reuse the existing P69 tab strip pattern. The active tab gets `var(--accent)` underline (2px). Inactive tabs: `var(--ink-mute)` label text. Tab labels: sentence case in EN (`Digests`, `Alerts`), Tajawal Arabic in AR (`ملخصات`, `تنبيهات`).

**Focal point — Digests tab:** The rollup-count summary header at the top of each published digest card (signals count + engagements count + commitments count, rendered in `--t-kpi-value` 36px mono). This is the primary information hierarchy anchor.

**Focal point — Alerts tab:** The active/inactive toggle and severity badge on each alert-rule row. Active rules have full-opacity `var(--ink)` title text; inactive rules use `var(--ink-faint)`.

**Digests tab — layout:**

- PageHeader: "Digests" (28px / weight 600 / `--t-page-title`) with secondary action "Generate digest now" (ghost button, `var(--ink-mute)` text, no accent unless it is the primary action in context)
- List of published digest cards, sorted newest-first
- Each card: 1px border `var(--line)`, `var(--radius)` 12px, `var(--surface)` background, `var(--shadow-sm)` on hover
- Card anatomy (top to bottom):
  - Header row: dossier glyph (24px circular) + dossier name (16px / 600) + frequency chip (12px uppercase label: DAILY / WEEKLY / MONTHLY) + period string (12px, `var(--ink-mute)`, format: `Tue 28 Apr – Fri 01 May`)
  - Rollup count strip: 3 KPI cells (`--t-kpi-value` 36px) — Signals, Engagements, Commitments — separated by `1px var(--line)` vertical dividers, no gaps (flush grid). Counts are integers; zero renders as `0` not `—`.
  - Section list: up to 5 linked signal titles (13px body) with severity badges. Overflow: "+ N more" link (12px, `var(--accent-ink)` color).
  - Footer: generated timestamp (`Mon 15 Jun`, `14:30 GST`, 12px `var(--ink-faint)`) + "Unsubscribe" ghost button (rightmost / end-aligned)

**Alerts tab — layout:**

- PageHeader: "Alerts" (28px / 600) with "Add alert" primary CTA (accent button, rightmost)
- List of alert-rule rows, each row height `var(--row-h)` (52px comfortable)
- Row anatomy (start to end, logical LTR; RTL flips):
  - Dossier glyph (20px) + rule name or auto-label (16px / 600) — start-aligned
  - Condition chip: "New signal" (12px uppercase label, `var(--info-soft)` background)
  - Severity filter chip: "HIGH / URGENT" only when set (12px, `var(--danger-soft)` bg, `var(--danger)` text)
  - Channel icons: in-app bell / email / webhook — 16px stroked glyphs, `var(--ink-mute)` color
  - Active toggle: HeroUI Toggle, accent when active
  - Delete icon-button (trash glyph, 20px, `var(--ink-faint)` color) — aria-label: "Delete alert rule" (EN) / "حذف قاعدة التنبيه" (AR)
- Hover state: row background fades to `var(--line-soft)` at 120ms ease-out

### Surface 2: Digest Reader (in-app rendered digest)

**Trigger:** User clicks a digest card in the Digests tab → inline expand or navigate to digest detail view (planner's call on routing, but contract is the same).

**Focal point:** The rollup count summary header (3 KPI cells, `--t-kpi-value` 36px mono). This renders the period summary at a glance before the analyst reads line items.

**Reader anatomy:**

- Back link: "← Digests" (12px, `var(--ink-mute)`) — uses IntelDossier back-arrow icon (`arrow-forward` in RTL)
- Title: dossier name + frequency + period (28px / 600, sentence case)
- Rollup header: 3 KPI cells (Signals, Engagements, Commitments) in `--t-kpi-value` 36px / JetBrains Mono, with `1px var(--line)` vertical separators, flush grid (no gaps)
- Sections: one collapsible `<details>` section per content plane (Signals / Engagements / Commitments / Status changes)
  - Section heading: 12px uppercase label + item count badge
  - Signals section: each signal row shows title (13px, as entered — single language, no translation), severity badge (`var(--danger-soft)` high/urgent, `var(--warn-soft)` medium, `var(--ok-soft)` low), category chip (12px), and date `Tue 28 Apr`
  - Engagements section: engagement name + date
  - Commitments section: commitment summary + due date `T+3` (mono)
  - Status changes section: dossier field changed + brief label
- Free-text signal titles are rendered exactly as entered (D-04: no system translation, no `title_en`/`title_ar` pairs). The chrome (section headings, labels, dates) is bilingual via i18n.
- Footer: "Published Mon 15 Jun at 14:30 GST" (12px, `var(--ink-faint)`) + "Unsubscribe" ghost link (end-aligned)

**Clearance:** Content is already clearance-gated by the SECURITY INVOKER RPC. Empty sections (zero items after clearance filtering) render with an indistinguishable-empty state: the section heading is shown with a `0` count, no body copy, no "filtered by clearance" messaging (P68 D-09).

### Surface 3: Digest Subscription Controls

**3a. Per-dossier "Subscribe to digest" (D-12)**

Location: on any dossier detail page (any of the 8 types), in the action area near the dossier header (consistent placement: below dossier glyph and name, above tabs). Not in a tab — it is a persistent in-context affordance.

Anatomy (unsubscribed state):

- "Subscribe to digest" — `btn-primary` accent button (`var(--accent)` background, `var(--accent-fg)` text, 13px / 600, `var(--radius-sm)` 8px)
- Frequency selector appears inline on click: three radio chip options — Daily / Weekly / Monthly (12px label, `var(--line)` border when unselected, `var(--accent-soft)` fill when selected)
- Confirm: "Subscribe" button (accent, small) + "Cancel" link (ghost)

Anatomy (subscribed state):

- Frequency chip only (e.g. "WEEKLY DIGEST", 12px uppercase label, `var(--accent-soft)` background)
- "Unsubscribe" ghost link (12px, `var(--ink-mute)`) — clicking opens a confirmation dialog (see Copywriting)

**3b. Subscription management list (in `/intelligence` Digests tab)**

The Digests tab shows the user's published digests. At the top of the tab, a compact "Your subscriptions" strip (collapsed by default, expandable) lists active subscriptions: dossier glyph + name + frequency chip + "Edit frequency" link + "Unsubscribe" ghost link. Row height: `var(--row-h)`. This is NOT a full management screen — it is an inline strip.

### Surface 4: Alert Rule Management (create / edit / delete)

**Alert rule list** — described in Surface 1, Alerts tab layout above.

**Create/Edit alert rule — drawer (not a full page)**

Trigger: "Add alert" button (from Alerts tab or dossier page) opens a 480px-wide drawer from the end edge (`var(--shadow-lg)`).

Drawer anatomy:

- Drawer header: "Add alert rule" (16px / 600) + close icon-button (aria-label: "Close")
- Form fields (top to bottom, each labeled at 12px uppercase label):
  1. **Dossier** — DossierTypeSelector + DossierSelector (existing component pattern). Label: "Dossier". Required.
  2. **Condition** — read-only chip: "New signal on this dossier" (12px, `var(--info-soft)` background). No dropdown in v1 — condition is fixed.
  3. **Severity filter** — optional toggle group: "High" + "Urgent" (accent when selected, `var(--line)` border when unselected). Helper text: "Only alert for signals at these severities. Leave unselected to alert on all signals." (13px, `var(--ink-mute)`)
  4. **Channels** — checkbox group: In-app (default, always enabled, not unchecked-able) / Email / Webhook. Each checkbox label has a 13px body label + 12px secondary description below. Email: "Delivered to your account email via on-prem SMTP." Webhook: "POST to configured endpoint. Deep-link only."
  5. **Active** — HeroUI Toggle, default ON. Label: "Alert active"
- Footer: "Add alert" primary accent button + "Cancel" ghost button (end-aligned / start of footer in RTL)
- Validation: inline error below each field (13px, `var(--danger)` text). Required fields have an asterisk aria-label companion (not rendered as a visual asterisk — use aria-required).
- Edit mode: same drawer, title changes to "Edit alert rule", destructive "Delete alert rule" link appears in the footer (start-aligned / far end in RTL), `var(--danger)` text color.

**Delete confirmation — inline alert dialog (not a full modal)**

Triggered by "Delete alert rule" link in the edit drawer. Uses HeroUI AlertDialog (or Radix Dialog) pattern:

- Heading: "Delete this alert rule?" (16px / 600)
- Body: "You will no longer be notified of matching signals on this dossier." (13px)
- Actions: "Delete" (`var(--danger)` accent button) + "Keep alert" ghost button

### Surface 5: Generate Digest — HITL preview → confirm → publish

**Trigger:** "Generate digest now" button. Available in two locations:

1. On any dossier page (inline in the action area, ghost button when subscription exists, primary button when not yet subscribed)
2. In the Digests tab, per-row action (ghost button labeled "Generate now")

**HITL flow:**

Step 1 — Period picker (inline popover, not a full modal):

- Three options: Daily / Weekly / Monthly (radio chips, same pattern as subscription frequency selector)
- "Generate preview" accent button

Step 2 — Preview drawer (720px wide, `var(--shadow-lg)`):

- Drawer header: "Digest preview" (16px / 600) + close icon-button
- Content: full digest reader render (same anatomy as Surface 2) but with a prominent top banner: "Preview — not published" (12px uppercase label, `var(--warn-soft)` background)
- The preview content is clearance-gated via the `generate_digest` SECURITY INVOKER RPC; the analyst only sees within-clearance items
- Footer: "Publish digest" primary accent button + "Discard" ghost button (end-aligned)

Step 3 — Publish confirmation (inline alert in the same drawer, no navigation):

- After "Publish digest" click: a confirmation banner replaces the preview top banner: "Digest published Mon 15 Jun at 14:30 GST" (`var(--ok-soft)` background)
- The "Publish digest" button changes to "Close" (ghost)
- The drawer auto-closes after 2 seconds OR on manual close

**Failure state:** If `generate_digest` RPC returns an error, the preview drawer shows an inline error banner: "Couldn't generate digest. Check your connection and retry." with a "Retry" button (accent ghost). The "Publish digest" button is disabled while in error state.

## Visual Dimension Notes (Dimension 2)

These declarations prevent checker flags on visual anti-patterns.

**Anti-template compliance:** All new surfaces (digest cards, alert-rule rows, HITL drawer) use design tokens. No gradient backgrounds, no raw Tailwind color literals, no shadcn default chrome. Borders: `1px solid var(--line)`. Card shadows: none (hover row shadow only: `var(--shadow-sm)`). Drawer shadows: `var(--shadow-lg)`.

**Hierarchy through scale contrast:** The rollup count header uses `--t-kpi-value` (36px) against 13px body text — a 2.8× scale ratio. Alert-rule rows use 16px rule name against 12px chip labels. Each surface has exactly one hierarchy anchor.

**Depth and layering:** Drawers (create/edit alert, HITL preview) render above the page with `var(--shadow-lg)`. Popovers (frequency picker) use `var(--shadow)`. Cards have no shadow at rest, `var(--shadow-sm)` on hover.

**Hover / focus / active states:**

- List rows (digest cards, alert rows): background fades to `var(--line-soft)` at 120ms `--dur-fast` ease-out. No scale/transform.
- Buttons: border darkens to `--ink-faint` on hover. Press: `color-mix(in srgb, var(--ink) 85%, var(--accent))`. No bounce/spring.
- Toggle (active alert): `var(--accent)` fill when active; `var(--line)` fill when inactive.
- Focus rings: `outline: 2px solid var(--accent)`, `outline-offset: 2px`.

**Icon-only actions that require visible labels:**

- Delete alert rule: icon-only trash button → `aria-label="Delete alert rule"` (EN) / `"حذف قاعدة التنبيه"` (AR). Tooltip on hover renders the same label text.
- Close drawer: icon-only × button → `aria-label="Close"` (EN) / `"إغلاق"` (AR).
- Refresh / Retry: icon-only → always paired with visible text in P70 (no icon-only retry buttons).

**RTL contract:**

- All containers use `dir={isRTL ? 'rtl' : 'ltr'}` and logical properties exclusively: `ms-*`, `me-*`, `ps-*`, `pe-*`, `text-start`, `text-end`, `rounded-s-*`, `rounded-e-*`.
- Tajawal applies globally on `dir="rtl"` (inherited from `html[dir="rtl"] * { font-family: var(--font-arabic) }` in colors_and_type.css).
- No `text-right`, `text-left`, `ml-*`, `mr-*`, `pl-*`, `pr-*` anywhere in P70 components.
- Directional icons: back arrow uses IntelDossier `arrow-forward` icon (points right → visually "back" in RTL). Chevron for "drill in": `chevron-left`. Channel icons (bell, mail, webhook) are non-directional — no flip needed.
- Drawer slides from `end` edge (CSS `inset-inline-end: 0`) so it appears from the left in RTL and from the right in LTR, matching user expectation.

**Breakpoints:**

- 1400px+: full layout, digest cards in 2-column grid (2:1 ratio in hub), alert rows full-width.
- 1024–1399px: single-column digest cards, alert rows full-width.
- 768–1023px: sidebar collapses to icon rail; hub content stacks vertically.
- Below 768px: read-only mobile view. Create/edit alert drawer becomes full-screen modal. No drag-and-drop.
