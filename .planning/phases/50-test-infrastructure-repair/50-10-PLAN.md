1. **i18n-text-assertion drift** (most common): The global `vi.mock('react-i18next')` in `frontend/tests/setup.ts` has `t: (key) => translations[key] ?? key`. When a test renders `<CommitmentList/>` and the component renders `t('commitments.title')`, the rendered DOM contains the literal string `commitments.title` (the key fallback), not `'Commitments'`. Tests authored against English strings (`screen.getByText('Commitments')`) fail. Two paths to fix per D-11:
   - Extend the global setup.ts `translations` map (preferred when 1-2 keys are needed and the keys exist in `frontend/public/locales/en/*.json`) — small surgical addition, no rewrite.
   - Or update the test to assert on the i18n key directly (acceptable when the test file is already being touched for another reason).

2. **Impl markup drift** (less common): The component's rendered markup (data-testid names, ARIA labels, button labels) changed since the test was authored. Update the test to match current impl shape (per D-10: test name is contract; assertion details serve the contract).
