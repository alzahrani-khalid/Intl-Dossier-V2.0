## Spacing Scale

All shell spacing values are multiples of 4 and use Tailwind utilities. Logical properties ONLY (`ps-*/pe-*/ms-*/me-*/start-*/end-*`) — per `CLAUDE.md §"Web/Tailwind RTL Requirements"`.

| Token | Value         | Usage in shell                                                          |
| ----- | ------------- | ----------------------------------------------------------------------- |
| xs    | 4px (`-1`)    | nav-item icon gap, locale button padding, topbar badge inset            |
| sm    | 8px (`-2`)    | sidebar user card padding, topbar right-cluster gap on phone, chip ms   |
| md    | 12px (`-3`)   | topbar icon-button padding, sidebar section internal gap variants       |
| md+   | 14px (`-3.5`) | topbar desktop gap, sidebar nav vertical gap                            |
| lg    | 16px (`-4`)   | topbar search horizontal gap, phone drawer head padding                 |
| xl    | 20px (`-5`)   | topbar desktop horizontal padding, classification ribbon padding-inline |
| 2xl   | 24px (`-6`)   | page-level padding (main slot forwards to page)                         |

**Exceptions:**

- Sidebar internal padding `14px 10px 10px` (top 14px / x 10px / bottom 10px) uses `-3.5 -2.5 -2.5` — these are allowed multiples-of-2 not multiples-of-4 because they map 1:1 to the handoff CSS (`app.css:41`). Documented deviation.
- `inset-inline-start:0` on active nav bar (no spacing token — it's a position anchor).
- Touch targets: **every interactive element in shell meets `min-h-11 min-w-11` (44×44)**. Hamburger = `36×36` in handoff but MUST be boosted to 44×44 minimum for touch (`md:h-9 md:w-9 h-11 w-11`). Topbar icon buttons at `p-1.5` expand hit-area via a wrapper or `min-h-11 min-w-11`.

## Classification Chrome (SHELL-03) — Component Shape Decision

**LOCKED: single `<ClassificationBar />` component with internal switch by direction** — handoff parity (`shell.jsx:193-208`). Rationale:

1. The three visual positions share one data source (`workspace + classification level + session info + user`) and one visibility toggle (`id.classif`). A single component keeps this coherent.

// Tailwind-only expression (logical-property variant):

## Interaction Contracts

### Drawer (mobile overlay)

1. **Open:** click hamburger → `setMenuOpen(true)` → body gets `overflow: hidden`, backdrop fades in 150ms, panel slides in 220ms ease.
2. **Focus:** on open, focus moves to first focusable element inside drawer (usually first nav item). React Aria `Drawer` handles via `autoFocus`.
3. **Close triggers (all):** ESC, backdrop click, re-click hamburger, any nav-item click, route-change event (TanStack Router subscription inside `AppShell`).
4. **Scroll lock:** active while drawer open (body `overflow: hidden`).

### Direction switcher

1. Click button → `setDirection(key)` → `applyTokens()` fires ONE `setProperty` loop (Phase 33 D-09) → entire shell repaints atomically.
2. Theme + hue reset to direction defaults (`DIRECTIONS[key].defaultTheme` + `defaultAccentHue`) per handoff `shell.jsx:31-35`.
3. `localStorage` persists `id.dir` key (carried forward from Phase 33/34).

### Locale toggle

1. Click `EN` or `ع` → `setLocale(key)` → writes `id.locale` localStorage key (Phase 34 D-12).
2. `<html lang>` and `<html dir>` update via existing `LanguageProvider` cascade.
3. Font stack auto-switches via Phase 35 `@theme` + Tajawal cascade. No flicker.

### Theme toggle

1. Click Sun/Moon → `setMode(m => m === 'dark' ? 'light' : 'dark')` → single `applyTokens()` write.

### Search input

1. Focus → `.tb-search` border → `var(--accent)` (Bureau direction only, per `app.css:540`; other directions keep default border).
2. `⌘K` keyboard shortcut — Phase 36 renders the HINT ONLY. **Palette behavior deferred** (CONTEXT deferred list).
3. Placeholder = `t('shell.search.placeholder')`.

### Tweaks button

1. Click → `useTweaksOpen()` hook (Phase 34 D-05) opens Tweaks drawer. No local state in Phase 36 shell.

### Notification bell

1. Click → **no-op in Phase 36** (Phase 42 owns dropdown UI). Button renders with proper `aria-label` and badge count from `useNotifications` hook (if mounted) OR hardcoded badge count of 0 when no hook.
