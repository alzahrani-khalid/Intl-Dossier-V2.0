<objective>
Close Gap-3 from 43-HUMAN-UAT.md (keyboard sweep — 30 fails, HIGH severity).
The spec at `frontend/tests/e2e/qa-sweep-keyboard.spec.ts` reports
`visibleCount=0` but `reached.size=1` for every route, so the strict
membership assertion (`reached.size === visibleCount`) trivially passes
for empty sets but tells us nothing about real reachability.

Three suspected causes (UAT analysis):

1. **`<main>` self-focus + tabIndex={0} (added by 43-11)** — `page.locator('main').first().focus()` puts focus on `<main>` itself; the first `Tab` press moves to the next focusable, which can be OUTSIDE main (a topbar/sidebar control), permanently exiting the scope before any Tab walks within main.

2. **`waitForRouteReady` settles before route paint completes** — `<main>` becomes visible while the route is still hydrating its interactive children; the count happens at a moment when the DOM tree under `<main>` is empty/skeleton.

3. **Playwright `:visible` filter scoped under `main`** — the chained selector `'main button:visible'` may not compose as expected when `<main>` itself has `tabindex` attributes; Playwright's CSS-engine pseudo-classes can return zero matches under specific composition orderings.

This is a **spec-side bug**. Production code stays as-is — 43-11's
`tabIndex={0}` on `<main>` is correct (it closed WCAG
`scrollable-region-focusable`).

Fixes:

A. **Pre-focus first interactive INSIDE main** rather than `<main>` itself
— guarantees the Tab walk starts from a position that's already a member
of the interactive set, so the first Tab moves WITHIN main.

B. **Extend `waitForRouteReady`** to additionally wait for `networkidle`
AND for the count of `main button, main a, main input, main [role=button]`
to be > 0 (best-effort; routes legitimately empty stay zero with a
skip-test escape hatch).

C. **Replace the chained `:visible` enumeration with a `page.evaluate`
filter** that enumerates focusable elements in `<main>` via
`querySelectorAll` and filters by `offsetParent !== null` + non-zero
`getBoundingClientRect`. This bypasses any Playwright selector-composition
quirk and is also the canonical way to count visible focusables in axe-core.

Purpose: Reveal real reachability data so the keyboard sweep becomes a
useful gate. With visibleCount=0 today the assertion is silently green
for the wrong reason — that's worse than a failing assertion.

Output: 1 spec file modified (definitely). 1 helper file modified

```ts
test(`${route.name} [${locale}] — Tab reaches every visible interactive (membership)`, async ({
  page,
}) => {
  await loginForListPages(page, locale)
  await page.goto(route.path)
  await settlePage(page)
  await waitForRouteReady(page)
  await page.waitForLoadState('networkidle').catch(() => {}) // best-effort; some realtime routes never go idle

  // Enumerate visible interactives via DOM filter (bypasses Playwright's
  // chained `:visible` selector-composition quirk on `<main tabindex="0">`).
  const visibleHandles = await page
    .locator('main')
    .first()
    .evaluateAll((mainNodes: Element[]) => {
      const main = mainNodes[0]
      if (!main) return []
      const candidates = main.querySelectorAll<HTMLElement>(
        'button, a, input, [role="button"], [tabindex]:not([tabindex="-1"])',
      )
      const visible: string[] = []
      candidates.forEach((el) => {
        // Skip <main> itself (it has tabIndex={0} per 43-11 to satisfy
        // scrollable-region-focusable, but it's not an "interactive" in
        // this sweep's semantics).
        if (el === main) return
        if (el.offsetParent === null) return
        const rect = el.getBoundingClientRect()
        if (rect.width === 0 || rect.height === 0) return
        visible.push(el.outerHTML.slice(0, 200))
      })
      return visible
    })
  const visibleCount = visibleHandles.length

  // Pre-focus the FIRST visible interactive INSIDE <main>, not <main>
  // itself. This guarantees the subsequent Tab walk stays within main on
  // its first press (otherwise the first Tab from <main tabindex=0> jumps
  // OUT to topbar/sidebar and we never re-enter).
  if (visibleCount > 0) {
    await page
      .locator('main')
      .first()
      .locator(
        'button:visible, a:visible, input:visible, [role="button"]:visible, [tabindex]:not([tabindex="-1"]):visible',
      )
      .first()
      .focus()
      .catch(async () => {
        // Fallback: focus <main> if no interactive is locator-visible despite the DOM filter saying yes
        await page.locator('main').first().focus()
      })
  } else {
    // Genuinely empty route content — assertion would be vacuously true; skip with reason
    test.skip(
      true,
      `[${route.name}][${locale}] no visible interactives inside <main> after waitForRouteReady`,
    )
  }
  await page.waitForTimeout(50)

  const reached = new Set<string>()
  for (let i = 0; i < visibleCount + 1; i++) {
    await page.keyboard.press('Tab')
    const focused = await page.evaluate(
      () => document.activeElement?.outerHTML?.slice(0, 200) ?? '',
    )
    reached.add(focused)
  }

  const label = `[${route.name}][${locale}]`
  expect(
    reached.size,
    `${label} unreached interactives: counted=${visibleCount} reached=${reached.size}`,
  ).toBe(visibleCount)
})
```
