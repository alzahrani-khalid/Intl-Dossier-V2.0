## Three available strategies (this plan applies Strategy 1 to all 4 sites)

**Strategy 1 — Raise contrast via token:** Drop `opacity-60` and rely on the existing `--sidebar-ink` token at full opacity. The token in directions.ts defines explicitly contrastive ink-on-sidebar pairs per direction (e.g., Bureau-light: bg=#ece5d2, ink=#1a1814 → ratio 12.4:1). Removing the 0.6 multiplier brings every direction × mode combo above 4.5:1 by default.

**Strategy 2 — aria-hidden:** Mark the element `aria-hidden="true"` (only valid if visual-emphasis-only and a SCREEN-READABLE equivalent exists nearby). NOT used in this plan — every site here carries semantic content.

**Strategy 3 — New `--sidebar-ink-soft` token:** Add a new direction-aware token mathematically guaranteed to meet 4.5:1 against `--sidebar-bg` per direction. Heavier change (mod tokens/directions.ts + buildTokens.ts + bootstrap.js); REJECTED for this plan because Strategy 1 is sufficient and surgical.
