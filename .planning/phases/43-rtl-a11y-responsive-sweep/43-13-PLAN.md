Current rule (frontend/src/styles/list-pages.css:421-433):

```css
.btn {
  display: inline-flex;
  align-items: center;
  gap: 6px;
  padding: 7px 14px;
  border-radius: var(--radius-sm);
  border: 1px solid var(--line);
  background: var(--surface);
  color: var(--ink);
  font-size: 13px;

  transition: all 0.12s;
}
```

Target rule (after this plan):

```css
.btn {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: 6px;
  padding: 7px 14px;
  min-block-size: 2.75rem; /* QA-03: 44px touch-target floor (Gap-1 closure) */
  border-radius: var(--radius-sm);
  border: 1px solid var(--line);
  background: var(--surface);
  color: var(--ink);
  font-size: 13px;
  font-weight: 500;
  transition: all 0.12s;
  box-sizing: border-box;
}
```

`justify-content: center` is added so wider buttons keep label centered
(no visual regression — current `inline-flex` + content-sized box was
implicitly left-aligned but content-width-equals-content). `box-sizing:
border-box` is explicit so the 44px floor includes border + padding;
without it, the 1px border would push total to 46px on min-clamp paths.
