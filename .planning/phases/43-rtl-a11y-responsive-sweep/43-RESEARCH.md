## 1. Executive Summary

The plan is **almost entirely additive** — five new Playwright spec files, two helper files, one ESLint glob edit, one markdown doc, and PNG fixture files. No runtime app code changes are required *except* for fixing a small set of directional icons in v6.0 routes that currently use `rotate-180` instead of the QA-04-mandated `scaleX(-1)` (see §6).

**Critical findings the planner must absorb:**

1. **CI command surface is fragmented** (§2): the project has *two* Playwright configs (root + `frontend/`), the CI runs only the root one via `test:e2e:ci`, and `frontend/tests/e2e/*.spec.ts` is *not* part of CI today. The new sweep specs live under `frontend/tests/e2e/` per CONTEXT, so wiring them as blocking requires either (a) adding a new shard to `e2e.yml` that runs `pnpm -C frontend exec playwright test`, or (b) moving sweep specs to the root `tests/e2e/`. **Decision needed in planning.**
2. **V6_ROUTES needs corrections** (§3): the dashboard route is `/dashboard`, not `/`; both `/dossiers/engagements` and `/engagements` exist and resolve to the same component. The CONTEXT registry is otherwise correct.
3. **5 fix-in-scope items in §6**: `VipVisits`, `EngagementStageGroup`, `PersonsListPage`, and the legacy `PersonDetail/CreatePage` use `rotate-180` for directional flips. QA-04 mandates `scaleX(-1)`. Either (a) fix these in Phase 43 (recommended — small, scoped), or (b) document `rotate-180` as a sanctioned alternative in `docs/rtl-icons.md` (changes scope of QA-04).
4. **`culori` is NOT a transitive dep** — CONTEXT specifics §"Programmatic focus-ring contrast check" is incorrect. Plan must either add `culori` as devDep or use a 30-line manual OKLCH→sRGB→relative-luminance helper.
5. **`window.__design` hatch is intact** but missing `setLocale` and `setClassif` — the focus-outline spec can set direction/mode/hue/density only. For RTL probing in the focus-outline matrix, the spec must also seed `id.locale` via the existing `loginForListPages(page, 'ar')` pattern.

## 5. `window.__design` Test Hatch

**Verified intact** in `frontend/src/design-system/DesignProvider.tsx`.

### Exposed API

```ts
interface DesignTestHatch {
  setDirection: (d: Direction) => void   // 'chancery' | 'situation' | 'ministerial' | 'bureau'
  setMode: (m: Mode) => void              // 'light' | 'dark'
  setHue: (h: Hue) => void                // 0–360 number
  setDensity: (d: Density) => void        // 'comfortable' | 'compact' | 'dense'
}
```

**MISSING from hatch:** `setLocale`, `setClassif`. The `qa-sweep-focus-outline.spec.ts` per D-08 needs to vary direction × mode (4 × 2 = 8) — locale is NOT part of that matrix per CONTEXT, so `setLocale` absence is fine. If RTL focus-ring needs probing (Settings page in AR), use the existing `loginForListPages(page, 'ar')` for that combo.

### Env gating (verified at `DesignProvider.tsx`)

```ts
useEffect(() => {
  if (!(import.meta.env.DEV || import.meta.env.MODE === 'test')) return
  ;(window as unknown as { __design: DesignTestHatch }).__design = { setDirection, setMode, setHue, setDensity }
  return (): void => { try { delete (window as unknown as { __design?: DesignTestHatch }).__design } catch {} }
}, [setDirection, setMode, setHue, setDensity])
```

- Hatch is `import.meta.env.DEV || MODE === 'test'` gated → tree-shakes from production bundle.
- Playwright runs against `pnpm dev` (NODE_ENV=development per `playwright.config.ts:73`) → `import.meta.env.DEV = true` → hatch is exposed.
- **No setup needed for CI**; hatch is automatic.

### Usage pattern (canonical, from `tests/e2e/token-engine-sc/` Phase 33-09)

```ts
await page.goto('/settings')
await loginForListPages(page) // ensure auth
await page.waitForFunction(() => typeof (window as any).__design?.setDirection === 'function')
await page.evaluate(() => (window as any).__design.setDirection('chancery'))
await page.evaluate(() => (window as any).__design.setMode('dark'))
// Settle paint before screenshot
await page.waitForTimeout(50)
```

**Pitfall:** The hatch's setters update React state which persists to localStorage. Successive specs see the previous run's tokens unless reset. Use `test.beforeEach` to reset to `bureau / light / 32 / comfortable` (the v6.0 defaults).

async function settlePage(page: Page, route: string): Promise<void> {
