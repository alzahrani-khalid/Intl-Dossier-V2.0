  <action>
MainLayout.tsx line 92 already provides `p-4 sm:p-6 lg:p-8 xl:mx-auto xl:w-full xl:max-w-[1600px]` on the content wrapper. Pages inside MainLayout must NOT add their own container padding.

**Search pattern:** Find all files under `frontend/src/pages/` containing `container mx-auto` or responsive padding patterns like `px-4 sm:px-6 lg:px-8`.

**For each file, apply this rule:**

1. Remove `container` class (MainLayout handles max-width)
2. Remove `mx-auto` that accompanies `container`
3. Remove responsive padding: `px-4`, `sm:px-6`, `lg:px-8`, `p-4 sm:p-6`, `p-6 lg:p-8` etc. -- any padding that duplicates what MainLayout provides
4. If the page intentionally constrains width narrower than 1600px (e.g., a form page), keep `max-w-4xl mx-auto` (or `max-w-3xl`, `max-w-2xl`) but still remove the responsive padding classes
5. If removing `container mx-auto px-4 sm:px-6 lg:px-8` leaves an empty className, remove the className prop entirely or the wrapper div if it only existed for padding

**Efficient approach:** Use grep/ripgrep to find all occurrences, then process files in batches. The pattern to search:

```
rg -l "container\s+mx-auto" frontend/src/pages/
rg -l "px-4\s+sm:px-6" frontend/src/pages/
rg -l "p-4\s+sm:p-6" frontend/src/pages/
```

**Do NOT touch:**

- Components outside `frontend/src/pages/` (they may render outside MainLayout)
- Pages that render in modals or dialogs (they have their own layout)
- The MainLayout.tsx file itself

| `border-gray-200`, `border-gray-300` | `border` | Standard borders |
| `divide-gray-200` | `divide-border` | List dividers |
| `hover:bg-gray-100`, `hover:bg-gray-50` | `hover:bg-accent` | Interactive hover states |
