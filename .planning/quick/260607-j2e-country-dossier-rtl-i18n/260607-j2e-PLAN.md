## Why M1 is not fixed (deliberate)

`DossierEngagementsTab` reads `related_dossiers` (from `dossier_relationships`) +
past `calendar_events` by design — its own comment says it mirrors
`EngagementHistoryCard`. UAE has zero related engagement dossiers and zero past
events, so the empty tab is **correct** for the current data. Separately,
`host_country_id` is NULL on all 3 `engagement_dossiers` rows system-wide (none
are UAE). There is a latent inconsistency — the engagement-count KPI reads
`host_country_id` while the tab reads `dossier_relationships` — but reconciling
them is a design change with zero current visible effect and real regression risk.
Surfaced, not silently rewritten.
