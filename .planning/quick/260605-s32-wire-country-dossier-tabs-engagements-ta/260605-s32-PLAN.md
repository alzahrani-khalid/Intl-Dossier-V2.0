In `fetchWorkItems` (the `if (taskIds.length > 0)` block, ~lines 599-628):

1. Replace the tasks query `.from('tasks').select('*, assignee:assignee_id(full_name)')` with
   `.from('tasks').select('*')`. The embed is unresolvable — `tasks.assignee_id` FKs to a table
   without a `full_name` column (full_name lives on `public.users`), so PostgREST returns 400 and
   every task is silently dropped from the aggregate. Keep `.in('id', taskIds).limit(limit)`.

2. Restore assignee names via a batched lookup AFTER the tasks resolve, mirroring the working
   pattern in `frontend/src/services/tasks-api.ts` (~lines 313-337): collect the non-null
   `assignee_id`s from the fetched task rows into a deduped array; if non-empty, query
   `supabase.from('users').select('id, full_name').in('id', assigneeIds)`; build a
   `Map<string, string | null>` of id -> full_name. (This `from('users')` read is an established,
   RLS-permitted pattern in tasks-api.) Set each work item's `assignee_name` from the map:
   `assignee_id != null ? (map.get(assignee_id) ?? null) : null`.

3. Replace BOTH reads of `t.deadline` with `t.sla_deadline` — the `isOverdue` check (~line 608)
   and the `deadline:` field mapping (~line 621). The `tasks` table has NO `deadline` column; the
   real column is `sla_deadline`, so deadlines currently never compute. The emitted DossierWorkItem
   `deadline` field stays named `deadline` (that is the work-item contract) — only the SOURCE column
   changes to `t.sla_deadline`.

4. Update the local `TaskRow` interface (~line 147): rename `deadline: string | null` to
   `sla_deadline: string | null`, and remove the now-unused `assignee: { full_name: string } | null`
   member (the embed is gone). Keep it `any`-free; if the Supabase row shape needs a cast, cast via
   `unknown` only as a last resort. Do NOT touch the commitment or intake branches — the intake
   branch's embed works because `intake_tickets.assigned_to_id` FKs to `public.users`.

Do NOT add a DB/integration test for this (it is a PostgREST query-shape fix — covered by type-check
