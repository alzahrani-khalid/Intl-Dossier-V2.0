# Phase quick-260605-s32 Plan 01: Wire Country Dossier Tabs (Engagements, Tasks, Timeline, Docs) Summary

Wired the four placeholder country-dossier detail tabs to real data and fixed the silent PostgREST 400 that dropped every task out of the dossier work-items aggregate (and the non-existent `deadline` column read that meant task deadlines never computed).

## What was built

1. **`fetchWorkItems` tasks-branch fix** (`dossier-overview.service.ts`): the tasks query used `select('*, assignee:assignee_id(full_name)')`, an unresolvable embed (`tasks.assignee_id` does not FK to a table exposing `full_name`), so PostgREST returned 400 and every task was silently dropped from the work-items aggregate. Replaced with `select('*')` plus a batched `public.users` lookup (the RLS-permitted pattern used in `tasks-api.ts`) to restore assignee names, and switched both the overdue check and the emitted `deadline` field to read the real `sla_deadline` column. `TaskRow` no longer declares `deadline`/`assignee`; added a small `AssigneeUserRow` shape for the lookup.

2. **Three reusable, type-agnostic tab components** under `frontend/src/components/dossier/tabs/` — each takes only `{ dossierId }` and fetches `fetchDossierOverview` under a dedicated per-tab query key (`['dossier-tab', <section>, dossierId]`) scoped via `include_sections`, with `staleTime: STALE_TIME.NORMAL`:
   - `DossierWorkItemsTab` — thin pass-through to `WorkItemsSection` (`include_sections: ['work_items']`).
   - `DossierDocumentsTab` — thin pass-through to `DocumentsSection` (`include_sections: ['documents']`).
   - `DossierEngagementsTab` — combines related engagement dossiers + past calendar events, newest-first and **uncapped** (mirrors `EngagementHistoryCard`'s mapping without the recent-entries cap); engagement-dossier rows link to their detail page via `getDossierDetailPath(id, 'engagement')`, event rows render as plain title rows; dates via `formatDayFirst`; working "No engagements" empty state.

3. **Four country route files rewired** (`timeline`/`tasks`/`docs`/`engagements`.tsx) from the static "Content coming soon" placeholder to the canonical `lazy(() => import(...))` + `Suspense` pattern from `positions.tsx`, each with a `TabSkeleton type="list"` fallback. Timeline loads the existing `DossierActivityTimeline`; the other three load the new tab components.
