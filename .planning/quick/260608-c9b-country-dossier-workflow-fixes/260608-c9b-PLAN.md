# Quick Task: Country Dossier workflow inspection fixes (round 3)

## Context

Third inspection-report pass over the Country Dossier workflow (browse → open →
work items), tested in EN (LTR) and AR (RTL). All six reported issues were
verified VALID against the live code and the staging DB
(`zkrcjzdemdmwhearhfgg`) before planning. Prior rounds (`260607-h9z`,
`260607-j2e`) fixed the shell/header chrome and dossier work-item Arabic titles;
this round closes the remaining drill-in, overview-i18n, recent-activity,
dashboard-title, analytics-alignment, and date-format gaps.

## Tasks (atomic commits)

1. **[HIGH] Work-item cards not interactive** — `WorkItemsSection.tsx`.
   `WorkItemCard` looks clickable (chevron + hover) but has no handler. Wire
   source-based drill-in mirroring `OpenCommitmentsSection`:
   - commitment → `useCommitmentDrawer().openCommitment(id)` (drawer is globally
     mounted at `routes/_protected.tsx`, keeps dossier context)
   - task → `navigate({ to: '/tasks/$id', params: { id } })`
   - intake → `navigate({ to: '/intake/tickets/$id', params: { id } })`
     Card becomes a full-width `<button>` with a focus-visible ring; dispatch held
     in `WorkItemList`.

2. **[MEDIUM] Overview cards English-only in AR** — `i18n/{en,ar}/dossier.json`.
   The six country overview cards call `t('overview.*')` / `t('analytics.*')` in
   the `dossier` namespace, but only a _nested_ `dossier.overview` exists — the
   top-level keys are missing, so the English `defaultValue` renders in both
   languages. Add top-level `overview` + `analytics` objects to both files.

3. **[MEDIUM] Recent Activity empty titles** — view + edge fn + frontend.
   Root cause (confirmed in DB): `dossier_activity_timeline` LEFT JOINs the
   empty legacy `commitments` table, so commitment rows return NULL
   title/status/priority/assignee. `CREATE OR REPLACE VIEW` re-pointing the
   commitment join to `aa_commitments` (+ append `activity_title_ar`), applied to
   staging via MCP and committed as a migration. Edge fn passes the new column
   through; `SharedRecentActivityCard` renders the AR title in AR mode.

4. **[HIGH] Dashboard overdue widget English titles in AR** — `query-columns.ts`
   `COMMITMENTS.LIST` omits `title_ar`; `usePersonalCommitments` always reads
   `c.title`. Add `title_ar` to LIST + the `Commitment` type; pick locale-aware
   title in the hook (mirrors the SUMMARY fix from `260607-j2e`).

5. **[LOW] Overview analytics not aligned with Tasks** —
   `useAnalyticsForDossier.ts` country case ignores `work_items_count` (=2 for
   UAE). Add it to `OverviewStats` + a third country metric
   (`analytics.workItemsCount`, key added in task 2).

6. **[INFO] Date format inconsistency** — route three inline
   `toLocaleDateString(...)` calls through the shared `formatDayFirst` util
   (`Tue 28 Apr` + Arabic-Indic digits): `WorkItemsSection.tsx`,
   `DossierFirstSearchResults.tsx`, `Countries.tsx`.
