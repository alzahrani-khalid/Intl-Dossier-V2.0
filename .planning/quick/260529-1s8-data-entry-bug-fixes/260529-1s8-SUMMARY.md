# Quick Task 260529-1s8 — Data-entry bug fixes (SUMMARY)

DB-schema migrations are the agreed next checkpoint.

## What was fixed (6 commits on `fix/data-entry-bugs`)

1. **`fix(dossiers-create)`** — engagement extension now writes to
   `engagement_dossiers` (was the legacy `engagements` table → every engagement
   create rolled back); `abbreviation` is now persisted (was silently dropped).
   Edge function **redeployed** to staging (`zkrcjzdemdmwhearhfgg`).
2. **`fix(dossier-wizard)`** — forum creation unblocked (removed orphaned
   `forum_type`, which has no DB column and silently dropped input); working-group
   creation unblocked (`status` → `wg_status`, enum aligned to
   `{active,suspended,disbanded}` + ar `disbanded` key); required-field gaps closed
   (topic `theme_category`, country `iso_code_2/3`, organization `org_type`);
   `tags` text↔array transform.
3. **`fix(intake)`** — IntakeQuickForm routed through `useCreateTicket` (was a 404
   to a nonexistent endpoint with a camelCase body); response mapped snake→camel so
   the success dialog shows the ticket number (was "undefined"); IntakeForm
   `title_ar`/`description_ar` made required (NOT NULL).
4. **`fix(forms)`** — `FormMessage` now translates messages, and all schema
   messages point to the `validation` namespace (+ new en/ar keys), ending raw-key
   and hardcoded-English validation leaks across TaskEditDialog, Task/Commitment
   quick forms, CommitmentForm, InteractionNoteForm, LegislationForm. TaskEditDialog
   priority/stage labels repointed to the `unified-kanban` namespace (has Arabic).
5. **`fix(writes)`** — useForums rolls back + throws on extension failure (was
   silent orphan); preference-sync surfaces remote failure (was false success);
   PushOptInBanner uses a targeted update keyed on `user_id` (was a partial upsert
   that clobbered theme/language) + new `dismissSyncFailed` key.
6. **`docs`** — CLAUDE.md glossary "Source-Specific Column Carve-Outs": intake
   `status`/`urgency`, commitment `due_date`/`owner_*`, task `sla_deadline`
   legitimately differ from the unified glossary and must NOT be renamed.
