# Quick Task 260624-d09 — SUMMARY

## Goal

Make the Phase-72 copilot drawer actually reach the runtime and answer with Gemma 4 12B
on the Mac-local stack. Started as "serve `/chat` (not just `/health`)" — driving the e2e
for the first time revealed the copilot was **never wired end-to-end**: three distinct,
previously-untested layers each had to be fixed.

## What was wrong (three layers, all pre-existing)

1. **Serving.** `Dockerfile.prod` runs `node dist/index.js` → the bootstrap only
   `import`ed the Mastra config (constructing the route graph) then started a bare
   **Express server exposing only `/health`**. The Mastra HTTP server that serves `/chat`
   was never started (`@mastra/deployer` / `mastra` CLI absent). → `POST /api/copilot/chat`
   = **404** (`Cannot POST /chat`).
2. **Protocol.** The frontend (`useCopilotRuntime.ts`) uses `@ag-ui/client` `HttpAgent` —
   it POSTs an **AG-UI `RunAgentInput`** (no `method` field). But the runtime registered
   `registerCopilotKit`, which mounts the **CopilotKit v2** endpoint that requires a
   `method` field. → every turn **400** (`Missing method field`) before the model.
3. **Model.** The Mac stand-in was `qwen3:30b`; switched to the real **`gemma4:12b`**
   (Gemma 4 12B, GA 2026-06-03 — the model the spec aliases as `gemma-4-12b`).

## Changes

- **`deploy/docker-compose.mac.yml`** — `VLLM_MODEL: qwen3:30b` → `gemma4:12b` (+ comment).
  Tool-calling verified on Ollama `/v1` before the swap.
- **`agent-runtime/src/index.ts`** — replaced the Express `/health` stub with a Hono
  server that mounts `mastra.getServer().apiRoutes` (`/chat` + `/health`), applies the
  `ALLOWED_ORIGINS` CORS, sets `c.set('mastra', …)`, and serves via `@hono/node-server`.
  Disabled CopilotKit anonymous telemetry (air-gap).
- **`agent-runtime/src/mastra/index.ts`** — replaced `registerCopilotKit` with a real
  **AG-UI SSE endpoint**: parse `RunAgentInput`, build the keystone `RequestContext`
  (JWT + `x-language` + JWT-derived `resourceId`, identical to the old `setContext`),
  run via `MastraAgent.getLocalAgent(...).run()`, SSE-encode each AG-UI `BaseEvent` with
  `@ag-ui/encoder`. Keystone (caller-JWT RLS) + owner-only threads (D-08) preserved.
- **`agent-runtime/package.json`** — `+hono@4.12.26`, `+@hono/node-server@1.19.14`,
  `+@ag-ui/encoder@^0.0.57` (declared what the code imports; one deduped hono);
  `-express`, `-@types/express` (now unused).

Each approach was proven in-container before coding (handler mount → `/chat` 400 not 404;
`getLocalAgent().run()` → 816 `BaseEvent`s to completion). `type-check` + `lint` clean.
