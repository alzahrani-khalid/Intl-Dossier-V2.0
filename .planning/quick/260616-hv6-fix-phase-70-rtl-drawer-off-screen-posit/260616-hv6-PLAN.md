<objective>
Fix two confirmed RTL/visual bugs in Phase 70 (Digests & Alerts), reproduced live in Arabic on staging. Both root causes are already diagnosed; this plan encodes the exact surgical edits only.

- Bug 1: The "Add alert" (AlertRuleForm) and "Subscribe to digest" (DigestSubscribeDrawer) drawers render jammed half off-screen in the top-left corner. Cause: the base `DialogContent` centers via Tailwind v4 `translate-x-[-50%] translate-y-[-50%]`, which compile to the standalone CSS `translate` property (not `transform`). The drawers' inline `transform: 'none'` does NOT clear `translate`, so the panel is shoved by (-50%,-50%) of its own size.
- Bug 2: In Arabic RTL, primary/accent/danger CTA buttons render in a Latin font (Inter, no Arabic glyphs). Cause: `.dir-bureau .btn-primary { font-family: var(--font-body) }` in list-pages.css (specificity 0,2,0) beats the universal `html[dir='rtl'] * { font-family:'Tajawal'... }` rule (0,1,1). Outline/ghost buttons set no font-family, so they inherit Tajawal correctly — which is why only "some" buttons are affected.

Purpose: Restore correct RTL drawer positioning and Arabic font rendering for Phase 70 CTAs.
Output: One-line addition to each of two drawer files (Bug 1) and a new scoped `!important` rule in index.css (Bug 2).
</objective>
