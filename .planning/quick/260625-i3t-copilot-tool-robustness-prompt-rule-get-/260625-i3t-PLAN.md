# Quick Task 260625-i3t: Copilot tool-robustness — stop the model fuzzing UUID args

## Problem (live probes 2026-06-25)

"Draft a brief for the G20 Data Gaps Initiative dossier" / "Publish a weekly digest for the
G20 …" both produced a Zod failure on a READ tool: `get_dossier: dossierId: Invalid UUID` — the
model passed the dossier NAME/title, not the UUID. The model retries and often recovers, but the
failed calls add latency + fragility. Same root class as the work_item bug: a non-UUID where a
`.uuid()` arg is required.

## Fix (agent-runtime ONLY)

### Task 1 — prompt rule (EN + AR)

`agent-runtime/src/mastra/agents/copilot.ts`: add ONE terse line (existing voice) to BOTH the EN
and AR "When answering" lists — when a tool needs an id/UUID (dossierId, signalId, work-item ids),
ALWAYS pass the exact UUID returned by a prior lookup (list_dossiers / hybrid_rag_search /
read_signals / get_dossier); NEVER a name, title, or placeholder; if you don't have the UUID yet,
call the lookup tool first.

### Task 2 — get_dossier name-OR-UUID resolution (low-effort, doing it)

`agent-runtime/src/mastra/tools/dossier-lookups.ts` (get_dossier):

- loosen `dossierId` to `z.string()` (drop `.uuid()`); describe it as a UUID or name.
- in `execute()`: add a module UUID regex; if `dossierId` is NOT a UUID, resolve it to an id via
  a name/title ILIKE lookup (`name_en`/`name_ar`, best single match, is_active) before the by-id
  query; if no match, return the neutral `{ dossier: null }`. Sanitize the term for the
  PostgREST or-filter (strip `,()*`). Indistinguishable-empty preserved on every failure.

### Task 3 — tests

`agent-runtime/src/mastra/tools/tools.test.ts`: the schema loosened, so add coverage —

- extend the fake Supabase builder with `or`/`ilike` (chain) so the name path is exercised.
- assert a non-UUID `dossierId` ("G20 Data Gaps Initiative") still builds the client from the JWT
  and returns the neutral `{ dossier: null }` on an empty match (indistinguishable-empty); a UUID
  still takes the by-id path. Existing get_dossier keystone/empty cases stay green.

## Scope guard

Touch ONLY copilot.ts + dossier-lookups.ts + tools.test.ts (agent-runtime). NO frontend.
