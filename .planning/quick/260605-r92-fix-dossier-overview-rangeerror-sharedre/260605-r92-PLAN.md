<objective>
Fix the `RangeError: Invalid time value` that crashes the dossier Overview tab for any
dossier with at least one activity. Root cause is already verified (do NOT re-investigate):
the `dossier_activity_timeline` view + `dossier-activity-timeline` edge function expose each
activity with `link_id` and `activity_timestamp` and emit NO `created_at` / `id` / `due_date`
/ `is_overdue` / `updated_at`. The frontend `TimelineActivity` interface fictionally declares
those fields, so TypeScript never caught the mismatch. `SharedRecentActivityCard` reads
`activity.created_at` (always `undefined`) -> `new Date(undefined)` -> Invalid Date ->
`formatDistanceToNow` THROWS.

Purpose: Restore the Overview tab for all dossiers, make the data contract honest so the
compiler can catch this class of bug, and harden the same-root-cause latent twin
(`ActivityTimelineItem`) in one pass to avoid a follow-up.

Output: An honest `TimelineActivity` interface, a defensively-formatted Recent Activity card
keyed on the real `link_id`, a fixed detail-timeline twin, and a regression test that goes
RED before the fix and GREEN after.
</objective>
