# Phase quick-260605-r92: Fix Dossier Overview RangeError Summary

**Eliminated `RangeError: Invalid time value` crashing the dossier Overview tab by making the `TimelineActivity` interface match the edge-function contract (`link_id` + `activity_timestamp` real; `created_at`/`id`/`is_overdue` optional) and formatting timestamps defensively.**

## Performance

- **Duration:** ~6 min
- **Started:** 2026-06-05T16:45:21Z
- **Completed:** 2026-06-05T16:51:09Z
- **Tasks:** 2 completed
- **Files modified:** 4 source files + 1 new test

## Accomplishments

- Reproduced the production crash with a RED regression test, then drove it GREEN (TDD).
- Made `TimelineActivity` honest: real fields the `dossier-activity-timeline` edge function always emits are required (`link_id`, `activity_timestamp`, ...); the display-only fields it never emits (`created_at`, `id`, `is_overdue`, `due_date`, `sla_deadline`, `updated_at`, `inherited_from_*`) are now optional with a clarifying comment, so the compiler catches blind reads.
- `SharedRecentActivityCard` now keys rows on `link_id` and formats `activity_timestamp` through a defensive `formatActivityTime` helper (guards nullish + `Number.isNaN(getTime())` → em-dash `'—'`, never throws).
- Hardened the latent same-root-cause twin `ActivityTimelineItem` in the same pass: derives `isOverdue` from a status fallback and renders the real `activity_timestamp` for meta time instead of the absent `updated_at`.
- `DossierActivityTimeline` list now keys on `link_id` instead of the absent `id`.
