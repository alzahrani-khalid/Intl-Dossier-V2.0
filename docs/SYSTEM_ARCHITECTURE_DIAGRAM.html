    Enter Base Fields

    check_dossier_name_similarity()
  4. 4
    Type-Specific Fields
    e.g. iso_code for countries, person_subtype for persons
  5. 5
    AI Pre-Fill (optional)
    AnythingLLM suggests field values from context
  6. 6
    Database Transaction
    INSERT dossiers + extension table + dossier_owners + auto user_profile
    dossiers [extension] dossier_owners

    status='todo', workflow_stage='todo'

5. Context Inheritance — How Work Items Find Their Dossier

Work items automatically inherit dossier context through entity chains, tracked in work_item_dossiers.

Inheritance Source Chain Example
direct
Task/Commitment → Dossier
User creates task directly from dossier page
engagement
Task → Engagement → Dossier
Task inherits from engagement's parent dossier
after_action
Commitment → After-Action → Engagement → Dossier
Commitment from after-action inherits 3 levels up
position
Work Item → Position → Dossier(s)
Inherits from position's linked dossiers
mou
Work Item → MOU → Organization(s)
Inherits from MOU party organizations
6. After-Action Flow — The Most Complex Creation Chain

After-action records are created after an engagement and produce commitments, decisions, and risks — all auto-linked to the parent dossier.

7. Work Item Lifecycles
📋 Task Lifecycle (Kanban)
todo → in_progress → review → done
Can also be cancelled at any stage
🤝 Commitment Lifecycle
pending → in_progress → completed
Can become overdue if past deadline • cancelled
🎫 Intake Ticket Lifecycle
draft → submitted → triaged → assigned → converted or closed
Can be merged with duplicate ticket
📜 Position Lifecycle (Approval)
draft → stage 1 review → stage N review → approved → published
Each approval creates a version snapshot • Can be rejected
📝 MOU Lifecycle
draft → internal_review → external_review → negotiation → signed → active
Auto-renewal tracking at 90/60/30 days before expiry

9. Junction Tables — The Glue Holding Everything Together
work_item_dossiers
Purpose: Links any work item type to any dossier
Columns:
• work_item_type (task | commitment | intake)
• work_item_id (UUID)
• dossier_id (UUID)
• inheritance_source (direct | engagement | ...)
• inherited_from_type, inherited_from_id
• inheritance_path (JSONB chain)
• is_primary (boolean)
dossier_relationships
Purpose: Graph edges between any two dossiers
Columns:
• source_dossier_id (UUID)
• target_dossier_id (UUID)
• relationship_type (bilateral | membership | ...)
• status (active | historical | terminated)
• effective_from, effective_to
• notes_en, notes_ar
position_dossier_links
Purpose: Many-to-many positions ↔ dossiers
Columns:
• position_id (UUID)
• dossier_id (UUID)
• link_type (applies_to | related_to | endorsed_by | opposed_by)
intake_entity_links
Purpose: AI-assisted links from intake to any entity
Columns:
• intake_ticket_id (UUID)
• entity_type (polymorphic)
• entity_id (UUID)
• is_ai_suggested (boolean)
• confidence_score (float)
• proposal_id (UUID, nullable)
dossier_owners
Purpose: Staff responsible for dossier maintenance
Columns:
• dossier_id (UUID)
• user_id (UUID)
• role_type (owner | co-owner | reviewer)
event_participants
Purpose: Links people to calendar events
Columns:
• event_id (UUID)
• user_id (UUID, nullable)
• person_dossier_id (UUID, nullable)
• rsvp_status (accepted | declined | tentative)
